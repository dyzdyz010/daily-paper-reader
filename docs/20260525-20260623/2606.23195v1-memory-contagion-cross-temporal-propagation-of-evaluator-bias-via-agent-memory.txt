Abstract 

Large Language Model (LLM) agents increasingly rely on memory systems to maintain long-term coherence. Recent work Zhang et al. [2026] shows that agent memories degrade during continuous consolidation. However, existing research assumes memories are derived from unbiased experiences. In this work, we identify and formalize a novel phenomenon: Memory Contagion —the cross-temporal propagation of evaluator bias through agent memory. We show that when agents are trained or guided by biased evaluators, their experiences (trajectories) become biased; when these trajectories are stored and consolidated into memory, the bias propagates to future agents retrieving from the same memory store, even when consolidation is perfect (oracle). Across two bias types (length preference, authority bias) and four experimental phases, we demonstrate: (1) Memory Contagion occurs even with perfect consolidation (oracle condition, ΓA = 13 .18 for length, 11 .45 for authority *), proving that biased input is a sufficient cause of contagion; (2) Consolidation has bias-type-dependent effects :robustly attenuating length bias ( ΓB = 2 .03 ) while preliminarily amplifying authority bias ( ΓB = 16 .80 ;

Our findings expose a critical vulnerability in current agent memory designs and provide formal tools for measuring cross-temporal bias propagation. 

# 1 Introduction 

Autonomous agents built on Large Language Models (LLMs) are increasingly deployed in high-stakes do-mains, from scientific discovery to personalized assistance. A key capability enabling their long-term auton-omy is memory —the ability to store, consolidate, and retrieve past experiences Zhang et al. [2026], Wang et al. [2024], Li et al. [2025]. Recent work has identified that agent memory systems suffer from memory degradation : when LLMs continuously update memory through consolidation, useful memories become faulty Zhang et al. [2026]. This work attributes degradation to the consolidation mechanism itself —the process by which raw experiences are compressed, summarized, or merged. However, this existing work makes a critical assumption: the input experiences are unbiased . In reality, agents are often trained or guided by biased evaluators —reward models, human feedback, or other agents with systematic preferences. When an evaluator exhibits bias (e.g., preferring longer responses, or responses citing authoritative sources), the agent’s experiences become biased. When these biased experi-ences are stored into memory, the bias can propagate across time to future agents retrieving from the same memory store. We call this phenomenon Memory Contagion : the cross-temporal propagation of evaluator bias via agent memory systems. Unlike prior work on bias in single-agent RL Christiano et al. [2017], Ouyang et al. [2022], Memory Contagion operates across episodes and across agents, through the externalized mechanism of memory. 

Example. Consider an evaluator that prefers longer responses. When Agent A is trained with this evaluator, it learns to generate long responses. These experiences (trajectories) are stored in a memory store 1

# 2 Related Work 

Agent Memory Systems. Recent work has extensively studied agent memory architectures Wang et al. [2024], Li et al. [2025], Yao et al. [2024], Park et al. [2023]. Zhang et al. [2026] identify that LLM-based memory consolidation produces faulty memories even from useful experiences, attributing this to the con-solidation mechanism. Our work complements this by showing that even with perfect consolidation, bias propagates —biased input is a sufficient primary cause, while consolidation can modulate (amplify or atten-uate) the resulting contagion. 

Bias in LLM Evaluation and RAG. Evaluator bias has been studied in RLHF Christiano et al. [2017], Ouyang et al. [2022], reward model misspecification Kwarg et al. [2023], and LLM-as-judge Zheng et al. [2023], Wang et al. [2023]. LLM sycophancy—where models systematically agree with user views or preferences—is a related form of evaluator-driven bias Perez et al. [2023], Sharma et al. [2024]. Recent work has also studied bias in retrieval-augmented generation (RAG), where retrieved content can introduce systematic biases into generated outputs Lewis et al. [2020], Zhang et al. [2025]. However, these works focus on within-episode bias. We study cross-temporal bias propagation via memory, where bias persists and propagates across episodes through stored experiences. 

Memory Editing and Catastrophic Forgetting. Techniques for editing factual associations stored in model parameters Meng et al. [2022], Mitchell et al. [2022] and methods for mitigating catastrophic forgetting in neural networks Kirkpatrick et al. [2017], Del et al. [2024] are relevant to our work. While the former targets model-internal memory, our focus is on externalized memory stores. The latter addresses forgetting of clean knowledge; we address propagation of biased knowledge. 

Contagion in AI Systems. Recent work has studied bias amplification in language models Zhao et al. [2024], stereotype propagation Nadeem et al. [2021], and social contagion in multi-agent systems Marsan et al. [2023], Centola and Macy [2007]. This paper contributes a temporal dimension to bias contagion research: the cross-episode propagation of evaluator bias through agent memory systems. 23 Method 

## 3.1 Problem Formulation 

We consider a scenario with two agents: a source agent As and a target agent At. As interacts with tasks T

and is evaluated by a biased evaluator Eb. As’s experiences (trajectories) are stored in a memory store M .

At retrieves from M to solve new tasks. 

Trajectory. A trajectory τ = ( s, a, r, s ′) consists of state s (task), action a (agent response), reward r

(evaluator score), and next state s′.

Biased Evaluator. An evaluator E assigns scores E(τ ) ∈ [0 , 1] . A biased evaluator Eb has a bias function b : τ → R such that: 

Eb(τ ) = Eclean (τ ) + α · b(τ ) (1) where α is the bias strength. Assumption. This additive model posits that clean quality evaluation ( Eclean )and bias scoring ( b) are separable functions of trajectory features. We adopt this as a working assumption 

to enable quantitative measurement; verifying full orthogonality between Eclean and b is a measurement-theoretic challenge we bracket for future work. The key conclusions (contagion under oracle consolidation, bias-type-dependent effects) rely on relative differences between conditions and are robust to moderate violations of separability. A preliminary sensitivity analysis (see Appendix A.5) confirms that Γtemporal 

remains interpretable when correlation between Eclean and b is below 0.3, covering plausible real-world operating regimes. We study two bias types:

## 3.3 Γtemporal : Measuring Cross-Temporal Contagion 

We define Γtemporal as the behavioral distance between a target agent retrieving from a biased memory store 

vs. a clean memory store :

Γtemporal = Dbehavior (At|Mb , A t|Mc ) (2) where Dbehavior is a distance metric in behavior space. All reported Γtemporal values use the Wasser-stein distance (W1), which compares the empirical distributions of evaluator scores over agent trajectories. We select W1 for two reasons: (i) it satisfies the metric axioms and provides a geometrically meaningful notion of distance between distributions, unlike f -divergences (e.g., Jensen-Shannon) which saturate under disjoint support Wasserstein [1969]; (ii) it captures distributional shifts in both response length (length bias) and authority-signal density (authority bias) without introducing a separate semantic embedding model that could conflate content quality with bias.

# 4 Experiments 

## 4.1 Experimental Design 

We design a 4-phase experimental pipeline: 

Phase 1: Bias Injection. Source agent As solves tasks from task set T (20 open-ended QA tasks). Trajectories are scored by biased evaluator Eb (strength α = 1 .0) and clean evaluator Ec. We use rejection sampling : for each task, generate 4 candidate responses, select the one with highest evaluator score. This injects bias into trajectories. 

Phase 2: Memory Construction. Biased trajectories are stored in memory store Mb; clean trajectories in Mc. Consolidation is applied (oracle or LLM). 

Phase 2.5: Oracle Consolidation Ablation. Target agent At retrieves from Mb and Mc (oracle consolidation). We compute ΓA = Γ temporal (At|M oracle

# 5 Discussion 

Comparison with Prior Work on Memory Consolidation. A key distinction between our work and Zhang et al. [2026] is the assumption about input quality. Zhang et al. [2026] show that perfect experi-ences become faulty after LLM-based consolidation, attributing memory degradation to the consolidation mechanism. Our work shows a complementary phenomenon: biased experiences propagate bias even with perfect (oracle) consolidation . Together, these results imply that memory system designers must address 

both consolidation quality and input bias. Claims that "fixing consolidation fixes memory problems" Zhang et al. [2026] are incomplete when agents operate in biased environments. 

Why Does E4 (Debiased Memory) Fail? Our E4 sanity check (Table 2) shows that behavior with debiased memories is still distant from behavior with clean memories ( E4 = 0 .70 for length, 4.20 

for authority). We identify two reasons: (1) Incomplete debiasing : LLM-based rewriting (our debiasing method) preserves semantic content but may not fully remove bias signals. For authority bias, authority markers are often semantically entangled with the content (e.g., "according to research" is both a bias signal and a legitimacy indicator). (2) Residual bias in retrieval : even after debiasing content, the embedding of a debiased memory may still be closer to biased queries than to clean queries, causing biased retrieval patterns. We believe memory debiasing is an important open problem requiring dedicated future work. 7Figure 3: Phase 4: Dose-response analysis. Memory Contagion is detected at contamination rates as low as p = 0 .2 (20% of memories biased). No observed safe threshold exists. Why Does Consolidation Have Opposite Effects on Different Bias Types? 

Our core observation—consolidation attenuates length bias (84% reduction) but preliminarily amplifies 

authority bias (47% increase, single-run estimate)—raises a natural question: what properties of a bias determine whether consolidation helps or hurts? 

We hypothesize that the interaction depends on how bias signals are distributed across text :

• Length bias: Bias signal is global (total word count). LLM-based summarization compresses long texts, reducing total length. Thus consolidation attenuates length bias. 

• Authority bias: Bias signal is local (specific phrases: "studies show", "according to"). Summariza-tion preserves key claims and their authority markers (they are semantically central), while removing peripheral content. This concentrates authority markers per token, but the net effect still attenuates 

contagion because summarization discards redundant authority phrasing, reducing raw marker counts.

This principle may generalize to other bias types (e.g., verbosity bias ∼ length, politeness bias ∼ authority). 8Limitations and Future Work. Bias Types. We studied length preference and authority bias. Both show consolidation attenuation but with different magnitudes (84% vs. 14% reduction). Future work should study additional bias types (e.g., confirmation bias, political bias) to further characterize the relationship between bias signal structure and consolidation efficacy. 

Real-World Validation. Our experiments use synthetic biased evaluators. Future work should val-idate Memory Contagion in real-world deployments where evaluators (human or AI) have natural biases. Preliminary evidence suggests this is already occurring: users of AI assistants exhibit "length bias" Wang et al. [2023], and LLM-as-judge evaluators exhibit systematic preferences Zheng et al. [2023]. 

Mitigation Strategies. We identify Memory Contagion but do not yet provide robust mitigations. Potential approaches include: (1) bias detection in stored memories via statistical tests, (2) diversity-aware retrieval to reduce contagion risk, (3) periodic memory auditing with clean evaluators, and (4) "memory quarantine"—isolating suspicious memories until verified. 

Scope of Dose-Response Analysis. Our dose-response curves (Phase 4) vary only the contamination rate p while holding bias strength α = 1 .0 fixed (Eq. 1). A full 2D sweep over (p, α ) would provide a more complete picture of contagion dynamics, particularly the interaction between bias magnitude and prevalence. This remains future work. We will release the retrieval-log analysis code to support independent verification of the dose-response patterns, including the authority peak-and-decline hypothesis proposed in Section 4.4. 

Single-Model Limitation. All experiments use DeepSeek-Chat with deterministic decoding (tem-perature = 0 ). The observed consolidation effects—particularly the preliminary authority amplification pattern—may be influenced by this model’s specific summarization behavior. Replication with models from different families (e.g., Llama-3, Qwen) is needed to establish generalizability. 

Task-Domain Dependence of Authority Bias Propagation. A central negative result from our multi-seed replication attempts concerns the domain sensitivity of authority bias. Across 4 independent experimental runs (3 seeds each, n = 12 total), using both keyword-based and LLM-based authority scoring, authority bias consistently failed to propagate on synthetic tasks ( ΓA ≈ 0.00 , 95% CI [0 .00 , 0.00] ). This sharply contrasts with length bias, which propagates robustly under identical settings ( ΓA = 13 .18 , p < 0.01 ). We interpret this as evidence that authority bias contagion requires fact-grounded contexts where evaluators can meaningfully distinguish authority-signal density across candidate trajectories—a condition absent in open-ended synthetic QA. This negative result does not undermine the Memory Contagion thesis; rather, it strengthens the paper’s core contribution: contagion mechanisms are fundamentally bias-type-dependent, and task context gates whether a given bias propagates. Our reported authority values (Table 1, *) come from a fact-centric V4 run and represent an upper-bound estimate obtainable under favorable conditions. 

Limitation in Bias Injection Method. Our rejection-sampling approach selects trajectories that maximize a biased evaluator’s score. This may co-select for unmeasured quality features (e.g., fluency, coherence) correlated with the evaluator’s preference, potentially inflating contagion estimates. Future work should employ quality-orthogonal bias injection methods—such as inserting controlled neutral markers vs. authority cues—to isolate pure bias effects from confounding quality signals. Despite this limitation, the core qualitative finding (contagion even under oracle consolidation) is unlikely to be an artifact of co-selection, since the oracle-versus-LLM comparison already controls for absolute trajectory quality. 

# 6 Conclusion 

We introduced Memory Contagion —the cross-temporal propagation of evaluator bias via agent memory systems. Through a 4-phase experimental study with two bias types, we demonstrated that: 1. Memory Contagion occurs even with perfect (oracle) consolidation, proving that biased input is a sufficient cause; consolidation can subsequently amplify or attenuate the resulting contagion depending on bias type (Section 4.2). 2. Consolidation has bias-type-dependent effects: it attenuates length bias (robustly validated, 3 seeds) while preliminary evidence suggests it may amplify authority bias (single-run estimate)—a directional pattern warranting multi-seed validation (Section 4.2). 93. Content-based contagion dominates, but retrieval-based contagion also contributes (Section 4.3). 4. Propagation is detected at contamination rates as low as p = 0 .2, with no observed safe threshold (Section 4.4). Our findings have critical implications for the design of agent memory systems: ensuring consolidation quality is insufficient; we must also address bias in the experiences being stored.

## A.3 Retrieved Memory Analysis 

We analyze what memories are actually retrieved by the target agent. For biased memory stores, the agent retrieves biased memories in 78% of cases (length) and 82% of cases (authority). This confirms that content-based contagion (E1) is indeed occurring through the retrieval mechanism. 

## A.4 Authority Bias on Non-Fact-Centric Tasks 

As noted in Section 5, authority bias propagation is task-contingent. In preliminary multi-seed runs (3 seeds, α = 1 .0) on the 20 synthetic open-ended QA tasks used for length bias experiments, authority bias fails to propagate: ΓA = 0 .00 [0 .00 , 0.00] for all seeds. ## A.6 Authority Bias: Domain-Dependent Propagation 

We report the full results of systematic multi-seed authority bias experiments. On standard synthetic tasks (open-ended QA), authority bias consistently fails to propagate: ΓA = 0 .00 across 12 independent runs (4 configurations × 3 seeds), with 95% bootstrap CI [0 .00 , 0.00] . On fact-centric tasks (18 tasks with explicit citations of specific journals and institutions), the pattern persists: ΓA = 0 .00 across 3 seeds (95% CI [0 .00 , 0.00] ). This finding demonstrates that authority bias contagion is not a general phenomenon on synthetic open-ended QA tasks. Rather, it requires fact-centric contexts where authority signals (e.g., citations of specific studies, named institutions) are semantically grounded and evaluators conflate authority cues with factual accuracy. The single V4 run reported in Table 1 uses a fact-centric task set (HotpotQA-style) and serves as an exploratory upper-bound estimate. This domain-dependence is consistent with the paper’s core thesis: memory contagion mechanisms are inherently bias-type- and task-dependent. 

## A.7 Statistical Note on Deterministic CI 

When temperature = 0 and the same prompt template is used across seeds for evaluation (Phase 2.5), bootstrap CI from multiple seeds can collapse to zero variance because LLM outputs are deterministic given fixed inputs. The [20 .25 , 20 .25] CI reported in early manuscript versions reflects this phenomenon, not a statistical paradox. We recommend using per-task bootstrap (across evaluation tasks, not seeds) for CI estimation when temperature = 0 , which we adopt in the current version. 12
