## 摘要
长时程智能体依赖记忆机制压缩交互历史，但优化记忆写入面临独特的信用分配挑战：记忆更新可能因为下游工具失败、嘈杂观察或推理错误而受到奖励或惩罚，而非其自身贡献。这种因果纠缠的信用可能导致智能体丢弃有用证据或保留无关信息。我们提出HiMPO，一个基于事后信息的记忆策略优化框架，用于为长时程智能体的记忆写入动作分配较少纠缠的信用。HiMPO首先通过比较在相同写入前状态下，从前一记忆和更新后记忆中可恢复的任务相关信息，估计记忆更新的局部效用。然后它使用事后相关性作为有界回顾过滤器，当局部效用不被目标结果支持时衰减记忆信用。产生的记忆特定优势仅应用于记忆标记，而轨迹级别的奖励优化其余智能体行为。在基于评判的开放域任务和客观压缩记忆问答中，HiMPO改进了基于记忆和基于强化学习的强大基线，同时保持了压缩上下文效率。对照干预进一步表明，HiMPO减少了工具引发错误的责任泄漏，并提高了记忆更新的归因保真度。

## Abstract
Long-horizon agents rely on memory mechanisms to compress interaction history, but optimizing memory writing faces a distinct credit assignment challenge: a memory update may be rewarded or penalized due to downstream tool failures, noisy observations, or reasoning errors rather than its own contribution. This causally entangled credit can lead agents to discard useful evidence or preserve irrelevant information. We propose HiMPO, a Hindsight-Informed Memory Policy Optimization framework for assigning less-entangled credit to memory-writing actions in long-horizon agents. HiMPO first estimates the local utility of a memory update by comparing the task-relevant information recoverable from the previous and updated memories under the same pre-write state. It then uses hindsight relevance as a bounded retrospective filter that attenuates memory credit when local utility is not supported by the target outcome. The resulting memory-specific advantage is applied only to memory tokens, while trajectory-level rewards optimize the rest of the agent behavior. Across judge-based open-domain tasks and objective compressive-memory QA, HiMPO improves over strong memory-based and RL-based baselines while preserving compressed-context efficiency. Controlled interventions further show that HiMPO reduces blame leakage from tool-induced errors and improves attribution fidelity of memory updates.

## 1. 论文的核心问题与整体含义

- **研究背景**：长时程智能体在执行复杂任务（如多跳问答、网络搜索、具身规划）时，需要利用记忆机制压缩交互历史以控制上下文长度、降低 token 开销并避免长上下文带来的性能退化。现有方法通过将压缩操作集成到策略学习中（端到端压缩）虽然使记忆写入具备任务感知能力，但也引入了一个新的**信用分配（credit assignment）难题**。
- **核心问题**：记忆更新所获得的奖励或惩罚往往**因果纠缠**于下游的工具调用、环境观察或推理步骤，而非其自身的贡献。例如，当工具返回错误信息而记忆忠实地总结时，最终失败不应归咎于记忆；反之，若工具返回正确信息但记忆丢弃了关键事实，则记忆应受到惩罚。但仅凭最终结果（成功/失败）无法区分这两种情况，导致记忆优化可能将无关的错误传递到记忆更新上。
- **整体含义**：论文提出**HiMPO（Hindsight-Informed Memory Policy Optimization）**，旨在为记忆写入动作分配**更少纠缠的信用**，从而避免因工具、观察或推理错误而错误地惩罚或奖励记忆更新，提高记忆写入的忠实度和最终任务性能。

## 2. 论文提出的方法论

HiMPO 的核心思想是：**记忆信用应由两项互补信号决定——局部反事实效用（local counterfactual utility）和事后相关性（hindsight relevance），后者作为过滤器对前者进行衰减，仅当两者方向一致时才给予记忆写入信用**。

- **局部反事实效用（Δi,t）**  
  对每一步记忆写入 `m_{i,t}`，通过比较在同一“写入前状态” `H_{i,t}` 下，使用当前记忆 `m_{i,t}` 和上一记忆 `m_{i,t-1}` 时对目标答案 `z^*` 的“可回答性分数（answerability score）”之差来估算其增量效用。  
  可回答性分数定义为在给定状态下生成目标结局的对数似然平均值：  
  `S(H_{i,t}, m, z^*) = (1/|z^*|) Σ log π(z^*_ℓ | H_{i,t}, m, z^*_{<ℓ})`  
  差值 `Δ_{i,t} = S(H_{i,t}, m_{i,t}, z^*) - S(H_{i,t}, m_{i-1,t}, z^*)` 衡量当前记忆写入带来的信息增益（而非整个记忆历史的质量）。该值在同一批次同一时间步内进行标准化得到 `\hat{Δ}_{i,t}`。

- **事后相关性（log ρ_{i,t}）**  
  将目标结局 `z^*` 作为条件，计算生成记忆 token 的对数似然，并与同批次其他记忆写入对比，自归一化得到 `log ρ_{i,t}`。若记忆写入在事后条件下变得更可能，则 `log ρ_{i,t} > 0`，表示其与目标结果一致；否则为负。

- **门控机制与记忆特异性优势**  
  采用符号一致门控（sign-consistent gate）：  
  `G(\hat{Δ}, log ρ) = σ(β_eff · (sgn(\hat{Δ}) · log ρ - τ_ρ))`  
  只有当 `\hat{Δ}` 和 `log ρ` 方向相同时，该门才开启，从而确保仅当局部效用与事后支持方向一致时记忆才获得信用。最终记忆特异性优势 `A^M_{i,t} = G(\hat{Δ}, log ρ) · \hat{Δ}_{i,t}`。

- **稳定化处理与 token 级优化**  
  - **记忆成功掩码**：若整体轨迹已成功，则避免将负的微观信号作用于成功路径上的记忆写入（`A^M ← 0`）。  
  - **向后平滑**：可选地利用指数滑动平均将下游记忆效用反向传播到前期预置性写入：`\tilde{A}^M_{i,t} = α A^M_{i,t} + (1-α) \tilde{A}^M_{i,t+1}`。  
  - **token 级分配**：将最终的记忆优势仅应用于 `<mem>` 标签内的 token，其余 token 仍使用轨迹级优势 `A^T`，整体采用裁剪策略目标进行优化。

- **实现要点**：HiMPO 作为组策略优化的插件模块，无需额外自回归解码，仅利用批量对数概率前向传递计算辅助信号；推理时与原有压缩上下文协议一致。

## 3. 实验设计

- **数据集与场景**  
  - **评判式开放域长时程任务**：BrowseComp-Plus (BCP) 和 FRAMES，要求智能体在多步交互中检索、压缩并整合证据，答案以 LLM 评判准确率衡量。  
  - **客观答案压缩记忆问答**：Local Wiki Search，包含 2~10 个检索目标的子任务，采用词级 F1 和精确匹配（EM）评估，为先前记忆智能体工作（如 MemPO）的标准基准。  
  - BCP 下还设置了**上下文预算扫描**（4k, 8k, 16k, 全上下文），考察记忆压缩在截断压力下的保留能力。

## 6. 主要结论与发现

- HiMPO 在所有基准上均优于强基线：  
  - 在 BCP 上，全上下文准确率由 MemPO 的 6.63 提升至 9.16（+2.53），并在各压缩预算下保持约 2.4-2.8 个点的领先，且在 4k 截断下记忆感知压缩相对于滑动窗口带来 4.7 个点的增益。  
  - 在 FRAMES 上也是最强方法，但收益幅度略小。  
  - 在 Local Wiki Search 上，平均 F1 从 37.6 提升至 40.2，EM 从 29.4 提升至 31.6，且 token 效率与 MemPO 持平。  
- 消融表明：来自局部反事实效用的信号自身已有帮助，事后过滤和稳定化组件带来额外增益，尤其是对 4B 模型和长时程任务。  
- 去混杂干预证实：HiMPO 能有效降低工具错误对记忆的责任泄漏（忠实于坏工具比从 1.00 降至 0.42），提升记忆丢弃定位准确率（从 0.41 至 0.68），并能更好地将信用前向传播至早期预置性写入，以及将归因聚焦到被干预的模块。

## 7. 优点

## 8. 不足与局限

- **依赖目标结局**：需要可靠的 oracle 或评判器提供目标答案来计算事后分数，限制了其在弱监督或模糊结局场景下的应用。
