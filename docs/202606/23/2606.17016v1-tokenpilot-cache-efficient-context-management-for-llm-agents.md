## 摘要
随着LLM代理在长时间会话中的部署，上下文积累推高了推理成本。现有方法采用文本修剪或动态内存淘汰来最小化令牌占用；然而，其无约束的序列变异会改变布局，引入前缀不匹配和缓存失效。这揭示了文本稀疏性与提示缓存连续性之间的关键权衡。为此，我们提出TokenPilot，一个双粒度的上下文管理框架。在全局层面，感知摄入的压缩作为框架约束，稳定提示前缀并在摄入关口消除开放世界环境噪声。在局部层面，感知生命周期的淘汰监控上下文分段的持续剩余效用，强制执行保守的批次轮换调度，仅在任务相关性过期时卸载内容分段。在隔离和连续模式下的PinchBench和Claw-Eval上的实验表明，TokenPilot在隔离模式下分别降低了61%和56%的成本，在连续模式下分别降低了61%和87%的成本，同时与先前系统相比保持了有竞争力的性能。TokenPilot已集成到LightMem2中，地址为https://github.com/zjunlp/LightMem2。

## Abstract
As LLM agents are deployed in long-horizon sessions, context accumulation drives up inference costs. Existing approaches utilize text pruning or dynamic memory eviction to minimize token footprints; however, their unconstrained sequence mutations alter layouts, introducing prefix mismatches and cache invalidation. This reveals a critical trade-off between text sparsity and prompt cache continuity. To address this, we present TokenPilot, a dual-granularity context management framework. Globally, Ingestion-Aware Compaction acts as a framework harness to stabilize prompt prefixes and eliminate open-world environmental noise at the ingestion gate. Locally, Lifecycle-Aware Eviction monitors the ongoing residual utility of context segments, enforcing a conservative batch-turn schedule to offload content segments only when task relevance expires. Experiments on PinchBench and Claw-Eval under both isolated and continuous modes demonstrate that TokenPilot reduces costs by 61% and 56% in isolated mode, and 61% and 87% in continuous mode, while maintaining competitive performance compared to prior systems. TokenPilot has been integrated into LightMem2 at https://github.com/zjunlp/LightMem2.

## 1. 核心问题与整体含义
- **研究动机**：LLM智能体在长会话（long-horizon sessions）中运行时，上下文不断累积，导致推理成本急剧上升。常见的文本压缩或动态内存逐出方法虽然减少了令牌数，但会引起序列布局突变，破坏提示前缀连续性，导致KV缓存（Key-Value Cache）大量失效，反而增加硬件侧的预填充开销。
- **核心矛盾**：文本稀疏性（降低令牌数）与提示缓存连续性（保持前缀匹配以享受缓存折扣）之间存在根本性权衡。
- **整体含义**：有效的上下文管理框架必须同时解决这两个冲突目标，既压缩上下文，又保持缓存对齐。TokenPilot 正是为此而设计。

## 2. 方法论
### 2.1 核心思想
- 双粒度上下文管理：全局框架层进行摄入感知压缩，局部序列层进行生命周期感知逐出，二者协同优化上下文，兼顾文本减少和缓存效率。

### 2.2 关键技术细节
#### (1) 全局摄入感知压缩（Ingestion-Aware Compaction）
- **前缀稳定化**：将运行时易变字段（如工作目录、时间戳、会话ID）替换为静态占位符，并将工具定义移至系统提示词末尾，确保多任务间的提示前缀逐字节相同，实现KV缓存复用。
- **观察内容精简**：对从环境返回的消息（如工具输出、网页内容）在进入上下文前进行规则化压缩（如HTML瘦身、执行输出截断），并维护一个外部内容注册表。当压缩导致关键信号缺失时，代理可通过轻量恢复工具按需获取完整负载，防止过度压缩。

#### (2) 局部生命周期感知逐出（Lifecycle-Aware Eviction）
- **状态转移模型**：每个上下文段（segment）经过三个状态：活跃（active）→ 已完成（completed）→ 可逐出（evictable）。即使任务已完成，只要对后续任务仍有残余效用，就保留在缓存中，仅当效用彻底为零时才触发物理清除。
- **批量触发调度**：逐出操作不是每轮执行，而是每隔B轮（论文中设为3）由轻量级评估器（Qwen3.5-35B-A3B）评估一次，减少对缓存连续性的干扰。
- **残余效用评估**：评估器分析工具依赖模式，判断已完成的历史段是否仍包含后续任务可能引用的信息，如共享文件结构等，从而推迟逐出，避免重复发现。

### 2.3 公式与算法流程（文字说明）
- **优化目标**：最大化上下文效用与维护成本的比值，成本由KV缓存命中/未命中令牌数加权计算。
- **效用密度估算**：内部消息（意图）的自然密度为1；外部环境消息的密度由基础效用和摄入门控（基于内容哈希的访问频率）决定。
