## 摘要
LLM智能体越来越多地在会话间维护用户事实的长期记忆。然而，这种记忆通常通过汇总问题行或情节上的准确率来评估。因为这种方法独立地对问题行评分，即使多个问题探查同一个事实，它也无法展示该事实在条件变化时的行为表现。我们引入了MemTrace，一个以知识单元为测量单位的基准：知识单元是关于用户的单个类型化事实，而非单个问题。MemTrace沿三个受控维度探查每个事实：记忆年龄，定义为该事实在历史记录中出现在多少次会话之前；问题类型，涵盖当前状态、早期状态和变化轨迹；以及证据条件，涵盖存在、缺失和被虚假前提矛盾的设置。评估了四种范式下的13种记忆系统配置，我们发现相似的汇总准确率掩盖了不同的失败：能够恢复事实的当前状态和早期状态并不意味着能追踪其如何变化，而安全地放弃回答并不意味着能纠正虚假前提。主要的瓶颈在于证据使用，而非检索：当系统失败时，证据可检索的频率是缺失的10倍。这些结果表明，改进长期记忆需要更好地利用可获取的证据，而不仅仅是更多的存储或检索。

## Abstract
LLM agents increasingly maintain long-term memory of user facts across sessions. Yet such memory is usually evaluated by aggregating accuracy over question rows or episodes. Because this approach scores question rows independently, even when several questions probe the same fact, it cannot show how that fact behaves as conditions change. We introduce MemTrace, a benchmark whose unit of measurement is the knowledge point: a single typed fact about the user, rather than an individual question. MemTrace probes each fact along three controlled dimensions: memory age, defined by how many sessions ago the fact appeared in the history; question type, covering current state, earlier state, and trajectory of change; and evidence condition, covering present, missing, and contradicted-by-false-premise settings. Evaluating 13 memory-system configurations across four paradigms, we find that similar pooled accuracy hides different failures: recovering a fact's current and earlier states does not imply tracking how it changed, and safe abstention does not imply correcting a false premise. The dominant bottleneck is evidence use, not retrieval: when systems fail, the evidence was retrievable 10 times more often than it was missing. These results suggest that improving long-term memory requires better use of reachable evidence, not simply more storage or retrieval.

MemTrace 的核心方法论是**将测量的基本单位从“问题”转变为“知识单元”**，并围绕知识单元构建一套多维度的探测基准。

*   **核心思想**：固定一个用户事实，然后在一个受控的环境中，系统地改变探查它的条件（记忆年龄、问题类型、证据条件），以此来精确测量记忆系统在不同方面的行为表现。

*   **关键技术流程 (数据构建与评估)**：
    *   **A. 知识单元提取与审计**：从多会话的用户对话历史中，提取出关于用户的类型化事实作为“知识单元”（如，动态事实、静态事实、偏好事实），并对其进行时间锚定和质量检查。
    *   **B. 构建受控探针**：对每个知识单元，在三个维度上生成探针：
        *   **记忆年龄**：在用户对话历史的8个时间检查点（从近期到饱和期）对知识单元进行探查，追踪其准确率随会话增多而退化的曲线。
        *   **问题类型**：对同一个事实，生成三种类型的问题：

            *   **变化轨迹**（Trajectory）：事实是如何随时间变化的？
        *   **证据条件**：模拟现实对话中的三种证据状态：

            *   **证据缺失**（Missing / Boundary）：问题询问一个对话历史中从未提及的事实，期望系统拒绝回答。
            *   **证据冲突**（Conflict / Contradiction）：问题中包含一个与历史中事实相矛盾的虚假前提，期望系统纠正前提并基于记忆回答。
    *   **C. 多维度评分与诊断视图**：
        *   **评分**：每个回答获得一个三元组分数：二进制的“要点准确率”、连续的“逐字完整性”以及“回答类型”（如，正确、拒绝、幻觉）。

### **6. 论文的主要结论与发现 (Main Conclusions and Findings)**

1.  **相近的汇总准确率掩盖了截然不同的失败模式**：例如，`HippoRAG-v2` 在“饱和期”的当前/历史问题上得分最高，但在变化轨迹问题上表现平平；而 `Mem-T` 则在轨迹问题上表现最好。长上下文模型在“新鲜”的轨迹问题上得分高，但在“饱和”后断崖式下跌。

3.  **“安全弃权”不等于“能纠正冲突”**：像 `Mem0`、`AMem` 这样的外部记忆系统在证据缺失时能够安全地拒绝回答（>97%），但在面对包含虚假前提的冲突问题时，它们很少能成功地纠正错误。它们倾向于将冲突也当作缺失来处理并弃权，这是一种证据使用上的失败。

*   **系统配置的复杂性**：不同记忆系统在更新机制、冲突/时间路径的设计上存在内部差异，这使得很难将性能差异完全归因于一个孤立的特征。论文将其作为一种固有的局限性进行了阐述。
*   **冲突和轨迹探测的难度**：这类问题的设计可能本身就隐含了对模型推理能力更高的要求，而不仅仅是记忆能力的要求，但这反而是该基准设计上的一个亮点，因为它反映了真实世界的需求。
