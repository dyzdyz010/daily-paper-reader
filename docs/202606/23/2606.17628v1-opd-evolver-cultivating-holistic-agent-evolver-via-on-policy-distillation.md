## 摘要
记忆已成为自进化智能体的标准基础，但保存经验并不等同于学会如何利用经验进化。现有的记忆智能体可以存储轨迹、检索反思或积累技能，但往往缺乏选择有用经验、采取行动、编写可复用知识并维护不断增长的知识库的整体能力。我们提出OPD-Evolver，一个快慢协同进化框架，通过在策略自蒸馏培养这样的智能体进化器。在快循环中，OPD-Evolver与四级记忆层次结构交互，以读取、使用、写入和维护经验，实现快速测试时进化。在慢循环中，结果校准的记忆归因与特权后见之明将这四种能力蒸馏到可部署的策略中。跨多领域基准测试，OPD-Evolver超越记忆系统如ReasoningBank高达11.5%，超过训练方法如Skill0约5.8%。进一步分析表明，OPD-Evolver内化了高价值经验和记忆管理，使得OPD-Evolver-9B能够挑战大型对手如Qwen3.5-397B-A17B和Step-3.5-Flash，指向了超越记忆增强智能体、迈向真正合格的智能体进化者的方向。

## Abstract
Memory has become a standard substrate for self-evolving agents, yet retaining experience is not the same as learning how to evolve through it. Existing memory agents can store trajectories, retrieve reflections, or accumulate skills, but often lack the holistic competence to select useful experience, act on it, write reusable knowledge, and maintain a growing repository. We introduce OPD-Evolver, a slow-fast co-evolution framework that cultivates such an agent evolver through on-policy self-distillation. In the fast loop, OPD-Evolver interacts with a four-level memory hierarchy to read, use, write, and maintain experience for rapid test-time evolution. In the slow loop, outcome-calibrated memory attribution and privileged hindsight distill these four abilities into the deployable policy. Across multi-domain benchmarks, OPD-Evolver surpasses memory systems such as ReasoningBank by up to 11.5%, and training-based methods such as Skill0 by ~5.8%. Further analysis shows that OPD-Evolver internalizes high-value experience and memory management, enabling OPD-Evolver-9B to challenge giant counterparts such as Qwen3.5-397B-A17B and Step-3.5-Flash, pointing beyond memory-augmented agents toward genuinely qualified agent evolvers.

### 2. 论文提出的方法论：OPD-Evolver框架

OPD-Evolver是一个**快-慢双循环协同进化框架**，通过在策略自蒸馏(On-Policy Self-Distillation)将智能体与环境的交互行为蒸馏回其策略参数中。

*   **核心思想**：
    *   **快循环（Fast Loop）**：智能体在线与环境互动，利用记忆进行测试时进化，但**不更新模型参数**。
    *   **慢循环（Slow Loop）**：利用**结果校准的记忆归因**（Outcome-Calibrated Attribution）和**特权后见之明**（Privileged Hindsight），将快循环中的交互行为转化为监督信号，**蒸馏**回智能体策略，从而训练其核心进化能力。

*   **关键技术细节**：
    *   **四级经验记忆**：框架组织了四种互补的记忆类型：
        1.  **轨迹（Trajectories）**：保存完整的交互回合作为情景证据。
        2.  **提示（Tips）**：存储局部的、启发式的警告或经验。
        3.  **技能（Skills）**：抽象出可复用的操作流程。
        4.  **工具（Tools）**：存储可执行命令或代码模板。
    *   **四阶段经验生命周期**：智能体在快循环中执行：
        1.  **经验选择**：根据当前任务，从候选记忆池中筛选出一个紧凑、可行动的上下文。
        2.  **经验驱动执行**：基于筛选出的经验上下文执行多步动作。
        3.  **经验编写**：在回合结束后，将新的轨迹和反馈提炼为新的记忆条目，写入相应级别。
        4.  **经验管理**：定期对记忆库进行维护，如查找、合并、删除过时或冗余记忆。
    *   **慢循环：统一的后见之明自蒸馏**：
        *   **结果校准归因**：这是一个核心创新。它将延迟的、稀疏的回合结果（成功或失败），转化为对记忆条目的密集价值评价。具体地，它比较了**当某个记忆被选中与未被选中时，同一类任务的平均回报差异**，从而为每个记忆分配一个校准后的价值分数 `V(m)`。
        *   **特权后见之明**：在训练时，存在一个“教师”模型，除了能看到学生模型的公开输入外，还能获得特权信息，如记忆的真实价值分数、成功的轨迹片段、记忆的未来效用等。
        *   **统一蒸馏目标**：对于经验生命周期的四个决策阶段（选择、执行、编写、维护），系统构建对应的 `(学生输入, 特权信息)` 数据对。学生模型根据自己的输入生成输出，而教师模型则根据输入和特权信息给出“后见之明”下的正确行为分布。训练目标是最小化学生分布和教师分布之间的KL散度，从而将教师的整体进化能力蒸馏给学生。
*   **算法流程（文字描述）**：
    1.  **快循环（在线）**：对于每个任务 `xt`，从记忆库 `Mt` 中检索并选出上下文 `St`，执行策略 `πθ` 获得轨迹 `τt` 和回报 `Rt`，然后编写新的记忆 `Δt` 更新记忆库。定期进行记忆维护 `ηq`。
    2.  **慢循环（离线训练）**：快循环积累交互日志。
    3.  **记忆归因**：根据记录的 `(Ct, St, Rt, Δt)` 等信息，利用公式 `bA(m) = Σ ρg(m) * [E(m被选时的R) - E(m未被选时的R)]` 和 `V(m) = α * γ * bA(m)`，为日志中的所有记忆 `m` 计算校准后的价值 `V(m)`。
    4.  **构建监督对**：为选择、执行、编写、维护每个阶段构建 `(zk, hk)`，其中 `zk` 为学生输入，`hk` 为融入了价值 `V(m)` 和成功轨迹等信息的特权上下文。
    5.  **蒸馏训练**：学生策略 `pSθ` 和教师策略 `pT_θ` （具有特权信息）基于相同的学生生成前缀 `y_k<n` 分别产生输出分布。通过最小化 `Lslow(θ) = Σ E[(1/Lk) * Σ Dtok( sg[pT] || pSθ )]` 这样一个token级别的KL散度损失来更新学生策略 `θ`。训练完成后，仅部署学生策略（无需特权信息）到快循环中。

### 6. 论文的主要结论与发现

*   **整体能力领先**：OPD-Evolver在所有10个基准子集上超越了同等骨干的记忆系统，如对ReasoningBank提升高达11.5%；同时，也一致超越了GRPO、Skill0等训练方法。
*   **以小博大潜力**：经过训练的OPD-Evolver-9B模型，在多个子集上能够挑战或超越参数量大数十倍的大型模型（如Qwen3.5-397B），证明了培养整体进化能力的有效性比单纯模型规模更重要。
*   **组件协同必要性**：消融实验表明，慢循环蒸馏、结果校准归因以及选择、编写、维护能力的联合训练对性能至关重要，移除任何一个都会导致显著退化。其中，移除结果校准归因导致的性能下降最大。
*   **能力可内化**：慢循环蒸馏不仅能提升模型对外部记忆的利用，还能将高价值经验**内化**到策略参数中，使模型在没有外部记忆的情况下也能表现更好、效率更高。
*   **定性行为改善**：经过训练，OPD-Evolver能更精准地选择高价值、任务相关的记忆，编写出更具因果诊断性的记忆，避免产生宽泛无用甚至错误的记忆。

### 7. 优点

*   **问题定义清晰**：首次明确定义了“智能体进化者”应具备的四种耦合能力，为后续研究提供了清晰的目标。
*   **方法论创新**：
    *   提出的**快-慢双循环框架**优雅地分离了测试时进化和训练时能力培养的过程。
    *   **结果校准记忆归因**机制非常巧妙，它将稀疏的、延迟的任务回报巧妙地转化为对记忆价值的密集监督信号，解决了“执行之外无监督”的核心难题。
    *   **统一在策略蒸馏**的设计，使得四个生命周期能力可以在同一策略内协同优化，避免了能力间的干扰。
*   **实验说服力强**：实验覆盖广、对比方法全面、消融和分析深入，尤其是与大型模型的直接对比和强大的泛化能力展示，有力地证明了方法的优越性和实用价值。

### 8. 不足与局限

*   **训练数据依赖性**：虽然展示了良好的泛化性，但其蒸馏效果依赖于训练数据的质量和异构性。更窄领域的数据集可能无法培养出如此全面的进化能力。
*   **计算开销**：慢循环的训练过程涉及教师模型的特权信息准备、在线轨迹采样和策略蒸馏，计算成本和工程复杂度高于简单的SFT或基础记忆方法。
*   **奖励稀疏性问题**：结果校准归因的核心假设是能从任务组和选择/未选择对比中提取统计信号。在极度稀疏奖励或单一任务无法构成“组”的场景下，归因的可靠性可能会下降。
*   **应用限制**：论文主要在代码、系统操作和具身导航等具有明确反馈的领域验证。在更开放、主观的文本创作或对话任务中，如何定义“任务成功”和校准记忆价值仍有待探索。
