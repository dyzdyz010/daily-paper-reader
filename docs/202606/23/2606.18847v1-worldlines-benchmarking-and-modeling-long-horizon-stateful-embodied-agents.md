## 摘要
为了在真实家庭中长期辅助人类，具身智能体必须记住用户日常、世界状态和过去的交互。现有的长时记忆基准主要评估以语言为中心的检索和问答，而具身基准通常关注短时程任务执行，未测试动态环境中的长时记忆应用。我们提出了WorldLines，一个面向长时程具身家庭辅助的项目驱动基准。它构建了带有对话、动作、执行反馈、物体和设备状态变化的时间扩展家庭轨迹，并将其转化为证据链接的记忆问答和具身任务规划样本。我们还提出了ObsMem，一个基于观察的记忆框架，它维护可见性感知的记忆和以动作为原生的状态轨迹，以支持状态感知决策。实验揭示了在部分可观察性、被覆盖的世界状态以及将长时记忆转化为具身规划方面持续存在的挑战，而ObsMem为此设置提供了一个更强的参考架构。

## Abstract
To assist humans over extended periods in real homes, embodied agents must remember user routines, world states, and past interactions. Existing long-term memory benchmarks mainly evaluate language-centric retrieval and question answering, while embodied benchmarks often focus on short-horizon task execution without testing long-term memory use in dynamic environments. We introduce WorldLines, a project-driven benchmark for long-horizon embodied household assistance. It constructs temporally extended household traces with dialogues, actions, execution feedback, object and device state changes, and converts them into evidence-linked samples for Memory QA and Embodied Task Planning. We further propose ObsMem, an observer-grounded memory framework that maintains visibility-aware memories and action-native state trails for state-aware decisions. Experiments reveal persistent challenges in partial observability, overwritten world states, and translating long-term memory into embodied plans, while ObsMem offers a stronger reference architecture for this setting.

- **长时程具身记忆缺口**：现有长时记忆基准（如 LoCoMo、LongMemEval）聚焦于对话式的检索与问答，脱离物理世界的状态变化；而具身任务基准（如 ALFWorld、PARTNR）则限于短周期，世界状态不跨 session 延续，不评估代理对动态环境、部分可观测性的长期记忆使用。
- **现实家庭服务需求**：真正长期在家庭中运行的机器人需要记住用户日常、物体位置、设备状态、过往交互，并能将持久的世界状态用于问答、任务规划与执行。
- **核心挑战**：代理必须在部分可观测条件下维护演化中的世界状态，区分直接观察与间接报告，处理被覆盖或过时的状态，并将历史证据转化为可执行的具身决策。

### 2.2 ObsMem：观察者基础记忆框架
- **观察门控与类型化摄入**：
  - 只有机器人出现在观察者集合中的事件才被写入记忆；他人言语额外记录为“报告”类型，严格区分直接观察与间接信息。
  - 事件按语义分流到四种记忆视图：**事件轨迹**（仅追加的历史证据）、**状态轨迹**（维护当前快照 \( \hat{S}_t \) 和变更历史 \( H_t \)）、**信念轨迹**（基于干预事件和来源可靠性标记状态置信度为 fresh/stale/uncertain/contradicted）、**承诺轨迹**（待执行的请求、提醒、未来义务）。
- **信念更新规则**：通过参数化公式，若存在矛盾证据则标记为 contradicted，若已有干预事件或来源非直接观察则根据连续未确认时间和事件数逐步降级。
- **片段级整合与冲突解决**：检测片段边界后，生成含摘要、事实原子、关系原子的 Episode Card 并写回各视图，不覆盖原始事件。
- **查询时检索与回答**：
  - 将问题解析为查询计划，识别意图、目标实体、状态属性、时间过滤和所需证据视图（state/belief/event/commitment/episode）。
  - 从选定视图分别检索，再由证据选择器按类型互补性选择最多 \( k \) 条记录，最终由答案生成器生成回答或执行计划。

## 6. 主要发现与结论
- **现有记忆方法在具身长时程任务中能力不足**：文本记忆基线即使能定位相关会话（Session Any@5 较高），也难以精确找回改变状态的具体事件（Event R@5 显著偏低），常混淆例程与一次性异常状态。
- **部分可观测与状态覆盖是核心难点**：在动态家居环境中，机器人可能错过关键动作（如他人私密移动物体），导致记忆中的状态过时甚至矛盾，基线的回答质量大幅下降。
- **类型化、观察者基础记忆显著提升性能**：ObsMem 凭借事件/状态/信念的分离，在记忆 QA 的 Judge 分数（0.713）和状态多跳事件回忆上均超越最强基线，并在具身规划中大幅提高状态一致性与前提条件检查的正确性。
- **证据选择与片段整合至关重要**：消融实验表明，移除证据选择器或片段整合会导致性能急剧下降，说明组合跨视图证据和高层次摘要对回答至关重要。
- **WorldLines 揭示了具身记忆基准的新要求**：不仅需要记忆“发生了什么”，还需明确“当前状态是什么”、“我知道多可靠”、“还有什么承诺待办”，并将这些信息转化为可执行计划。

## 7. 亮点与优点
- **首次将长时间跨度、持续世界状态与具身动作/设备操作结合进评估**：填补了对话式记忆基准与短周期具身基准之间的空白。
- **项目驱动、闭环轨迹生成**：通过生活项目、结转记忆和确定性执行器，产生丰富、连贯、可复现的多日交互轨迹，并自动转换为证据链接的样本，确保问题答案可追溯。
- **ObsMem 的观察门控和类型化记忆设计**：清晰地区分直接观察与间接报告，维护状态历史与变迁，用信念跟踪代理的知识可靠性，并区分事件、状态、承诺等多类信息，从机制上缓解部分可观测问题。
- **全面的实验验证**：从多维度指标、消融到下游具身规划，并在附录中提供了详尽的模型表格、雷达图和 Prompt 模板，提高了可复现性。
- **评判可靠性**：通过严格的人工标注协议验证了自动裁判的有效性，增强了实验结论的信任度。

## 8. 不足与局限
- **模拟环境的限制**：WorldLines 构建于 Habitat 模拟器，未包含感知噪声、执行误差和完全开放的人类行为，这可能导致与真实世界部署存在差距。
- **结构化输入的依赖**：ObsMem 需要明确的实体 ID、可见性标注和预定义动作模式，在真实机器人应用中需由感知、定位和接地模块提供这些信息，整合难度和可靠性未经验证。
- **实时性开销**：类型化检索、信念计算等步骤带来额外延迟，论文未详细讨论在实时低延迟场景下的适配。
- **规划评估规模较小**：具身规划样本仅 21 个，虽动作密集但无法覆盖所有家居任务类型，结论的普适性有限。
- **未包含视觉或传感器信息**：当前主要基于结构化文本事件，缺少从 RGB、深度等原始感知输入直接构建记忆的链路，未来需扩展至多模态记忆。
