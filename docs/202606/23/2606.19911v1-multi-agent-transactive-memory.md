## 摘要
跨多种任务部署具有不同能力的大模型智能体的去中心化方式，推动了在异构智能体群体间进行知识共享的基础设施需求。正如搜索引擎索引人类生成的产物以支持人类解决问题，检索系统可以组织智能体生成的产物以供智能体群体重复使用。我们将检索增强生成（展示了人类撰写产物对单个智能体的价值）扩展到对智能体生成产物的检索，以支持一群智能体。特别是，智能体的轨迹编码了可重用的程序性知识，但这些产物通常在使用一次后就被丢弃，或仅由生成智能体保留，迫使新实例化的智能体重复地重新发现已有的解决方案。我们提出多智能体交互记忆（MATM），这是一个用于群体级存储和检索智能体生成轨迹的框架，其中生产者智能体将轨迹贡献给共享仓库，而消费者智能体检索它们以改进任务执行。我们关注交互式环境（ALFWorld和WebArena），在这些环境中轨迹很长，并且编码了特别丰富的程序结构。我们的实验表明，从MATM中检索轨迹可以提高下游任务性能，并减少交互步骤，而无需协调或联合训练。这些结果使MATM成为开放智能体生态系统中群体级经验共享的一种设计模式。

## Abstract
The decentralized deployment of LLM agents with diverse capabilities across diverse tasks motivates infrastructure for knowledge sharing across heterogeneous agent populations. Just as search engines index human-generated artifacts to support human problem solving, retrieval systems can organize agent-generated artifacts for reuse across agent populations. We extend retrieval-augmented generation - which demonstrates the value of human-authored artifacts to individual agents - to retrieval of agent-generated artifacts supporting a population of agents. In particular, agent trajectories encode reusable procedural knowledge, yet these artifacts are typically discarded after a single use or retained only by the producing agent, forcing newly instantiated agents to repeatedly rediscover existing solutions. We propose Multi-Agent Transactive Memory (MATM), a framework for population-level storage and retrieval of agent-generated trajectories, where producer agents contribute trajectories to a shared repository and consumer agents retrieve them to improve task execution. We focus on interactive environments (ALFWorld and WebArena), where trajectories are long and encode especially rich procedural structure. Our experiments demonstrate that retrieving trajectories from MATM improves downstream task performance and reduces interaction steps without coordination or joint training. These results position MATM as a design pattern for population-level experience sharing in open agent ecosystems.

### 1. 论文的核心问题与研究动机

*   **核心问题**：在多智能体生态系统中，智能体在执行任务时产生的交互轨迹（Trajectories）包含了丰富的**可复用的程序性知识**，但现有方法通常在一次使用后即丢弃或仅由生成者保留，导致**知识浪费**。这迫使新加入或能力较弱的智能体不得不**重复探索和发现已有的解决方案**，效率低下且计算成本高昂。
*   **研究动机**：
    *   **知识共享的缺失**：现有记忆系统（如 RAG）主要索引人类知识，或局限于单智能体的私有经验。群体层面缺乏一种类似人类社会“交互记忆”的共享基础设施，使异构智能体能够贡献和获取程序性经验。
    *   **可扩展性与效率需求**：现代智能体推理成本高，若能复用群体经验，可显著降低冗余探索的计算开销，提升群体整体的任务解决效率。
    *   **桥梁作用**：MATM（Multi-Agent Transactive Memory）旨在将检索增强生成（RAG）范式从“智能体检索人类知识”扩展至“智能体群体检索智能体生成知识”，建立一种服务于开放、去中心化智能体生态的**群体级设计模式**。

### 2. 方法论详解

*   **核心思想 (Producer-Consumer 框架)**
    MATM 构建了一个共享的轨迹仓库，将智能体分为两类非排他角色：
    *   **生产者**：贡献自身成功或不成功但达到一定质量阈值的交互轨迹至仓库。
    *   **消费者**：通过检索机制从仓库中获取相关历史轨迹，作为上下文指导来增强自身当前任务的执行。
    *   该系统构成了一个双边“经验市场”，无需集中协调或联合训练。

*   **关键技术细节**
    *   **状态条件键值索引 (State-Conditioned Key-Value Indexing)**
        处理动作-观察历史轨迹时，MATM 采用一种贴近实时决策的索引方案。它将最近 `l` 步的交互历史 `(τ_{t-l+1}, ..., τ_t)` 作为检索的**键 (Key)**，将紧接着的后续 `l` 步交互 `(τ_t, ..., τ_{t+l-1})` 作为**值 (Value / Document)**。这使得消费者智能体能根据当前状态检索下一步的行动指导，而非仅基于任务描述的静态匹配。
    *   **学习排序轨迹 (LTRT / Learning to Rank Trajectories)**
        为了优化从仓库中检索的候选轨迹质量，MATM 引入了一个轻量级的重排序模型。该模型不仅依赖语义相似度，还综合分析了多维特征，部分特征包括：
        1.  **生产者智能体元数据**：如其在多个标准基准上的评分，用于**信任建模**。
        2.  **消费者智能体元数据**：如智能体ID，用于**个性化检索**。
        3.  **首阶段检索特征**：初始稠密检索的相似度得分。
        4.  **查询和轨迹自特征**：如查询长度、轨迹长度、轨迹是否最终成功。
        5.  **查询-轨迹交互特征**：如文本的 TF-IDF、Jaccard 和 Embedding 相似度。
        LTRT 模型通过比较在使用某轨迹后任务得分相对于无检索基线的 **边际效用** 来监督训练，从而学习预测轨迹的实际“有用性”。
    *   **检索规划器 (RetrievalPlanner)**
        一个基于 LLM 的模块，在交互的每一步决定是否真的需要触发检索。这避免了不必要或无效的检索，防止无关上下文干扰智能体决策。

### 3. 实验设计

*   **数据集与场景**
    *   **ALFWorld**：一个基于文本的家务任务环境，任务类型包括拾取、清洗、加热、冷却等，轨迹较长且程序性强。使用 3553 条训练/构建轨迹，在 274 条官方测试任务上评估。
    *   **WebArena**：一个具挑战性的网页导航环境，模拟了电商、社交、地图等多个真实网站。作者自行构建了分布一致的 724/88 训练/测试集划分，测试任务涵盖了丰富的任务意图。
*   **对比方法 (Baselines)**
    *   **无检索**：不提供任何 MATM 支持，检测智能体原生的任务解决能力。
    *   **单阶段检索**：使用 E5-Base 嵌入模型进行首阶段稠密检索，直接返回最相似的轨迹片段。
    *   **MATM + 重排序**：在单阶段检索基础上，应用训练好的 LTRT 重排序器（FFN, LambdaMART, SVMRank）挑选最佳轨迹。
*   **评估指标**
    *   **成功率 (SR)**：衡量任务完成的有效性。
    *   **交互步数 (# steps)**：衡量任务解决的效率。
    *   **回报成对偏好 (RPP / Return-Paired Preference)**：一个联合衡量有效性和效率的指标，比较某方法与基线在联合 SR/步数上的帕累托优势。正分表示更优。

### 6. 主要结论与发现

*   **显著提升任务性能与效率**：在所有评估环境中，MATM 检索都能稳定提升消费者智能体群体的成功率和任务效率（减少交互步数）。例如，在 ALFWorld 中，成功率相比无检索提升超 17%（结合重排序后）。
*   **学习重排序至关重要**：简单的语义相似度检索优于无检索，但通过 LTRT 模型引入生产者信任、消费者个性化等信号进行重排序后，能够进一步显著放大增益，其中 SVMRank 在 ALFWorld 上效果最佳。
*   **收益普适且跨任务泛化**：从 MATM 获益不局限于高能力生产者向低能力消费者传递知识，即使从比自己弱的智能体处检索，也能获得正向收益。更重要的是，即便是来自不同任务类型的轨迹，也包含了可迁移的程序性模式，验证了构建群体级、跨任务共享记忆库的必要性。
*   **系统性能随记忆库增长而提升**：随着 MATM 索引规模的扩大，消费者智能体的任务成功率和效率呈现单调或趋势性改善，显示出其作为规模化解决方案的潜力。

### 8. 不足与局限

*   **实验环境限制**：仅在 ALFWorld 和 WebArena 两个相对特定的交互环境中进行了验证，其结论是否能泛化到数学推理、代码生成或开放式物理世界等不同模态和任务结构的场景，尚待考证。
