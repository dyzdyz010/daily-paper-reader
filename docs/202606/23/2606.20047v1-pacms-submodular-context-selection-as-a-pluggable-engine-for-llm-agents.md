## 摘要
对话和工具使用的LLM智能体在上下文窗口中操作，该窗口同时从多个方向填充。随着会话的进行，智能体积累用户和助手的回合、从持久记忆存储中提取的条目，并且通常最大的部分是工具调用的逐字输出，如文件读取、搜索结果和API响应。一旦累积的上下文超过模型的令牌预算，框架必须决定保留什么。主流机制是就近截断，有时配合定期摘要。这是主题盲目的：会话早期建立的事实仅仅因为陈旧就被丢弃，即使当前的用户查询正好是关于那个事实；相反，冗长但不相关的近期材料被保留。必须跨多个回合回忆信息的智能体，这正是记忆的定义用例，恰恰是就近截断失败的地方。现有的替代方案位于智能体的组装步骤之外。检索增强生成将外部文档提取到提示中，但并不仲裁智能体的已经存在的汇集上下文。上下文压缩方法通过重写或修剪文本来减少令牌数量，但操作时无视查询且是有损的。两者都没有将记忆条目、对话回合和工具输出视为一个候选池，在提示组装时根据相关性进行选择。

## Abstract
Conversational and tool-using LLM agents operate over a context window that fills from several directions simultaneously. As a session proceeds, the agent accumulates user and assistant turns, entries drawn from a persistent memory store, and often largest of all, the verbatim outputs of tool calls such as file reads, search results, and API responses. Once the cumulative context exceeds the model's token budget, the framework must decide what to keep.   The prevailing mechanism is recency truncation, sometimes paired with periodic summarization. This is topic-blind: a fact established early in a session is discarded simply because it is old, even when the current user query is about exactly that fact; conversely, verbose but irrelevant recent material is retained. Agents that must recall information across many turns, the defining case for memory, are precisely where recency truncation fails.   Existing alternatives sit outside the agent's assembly step. Retrieval augmented generation fetches external documents into the prompt but does not arbitrate the agent's \emph{already-present} pooled context. Context-compression methods reduce token count by rewriting or pruning text, but operate query-blind and lossily. Neither treats memory entries, conversation turns, and tool outputs as a single candidate pool to be selected from by relevance at the moment the prompt is assembled.

## 1. 核心问题与整体含义
- **背景**：长对话或工具使用的LLM智能体（agent）在会话过程中不断累积用户‑助手轮次、持久记忆条目和工具调用输出，上下文窗口极易超出模型的令牌预算。
- **现有方法的问题**：
  - 默认的“就近截断”（recency truncation）和定期摘要是主题盲目的，会丢弃早期但与当前查询高度相关的事实，保留最近但不相关的冗长信息。
  - 检索增强生成（RAG）仅从外部引入文档，不仲裁智能体已存在的内部汇集上下文。
  - 上下文压缩方法以有损方式修剪或重写文本，且操作时无视查询。
- **研究动机**：需要一种在提示组装时，将记忆、对话和工具输出统一视为候选池，并根据查询动态选择最相关上下文的原理性方法，以克服盲目截断导致的记忆召回失败。

## 2. 方法论
- **核心思想**：将智能体上下文组装建模为预算约束下的子模函数最大化问题，具体采用**设施地点覆盖（facility‑location）目标**。
- **关键技术细节**：
  - 候选池 \(C\) 包含所有可用上下文单元（记忆、对话轮次、工具输出）。
  - 定义相关性 \(rel(i,q) = \max(0, \cos(e_i, e_q))\)，覆盖权重 \(w_{ij} = rel(i,q) \cdot \max(0, \cos(e_i, e_j))\)。
  - 最大化目标函数：
    \[
    F(S) = \sum_{i} \max_{j \in S} w_{ij} \quad \text{s.t.} \quad \sum_{j \in S} tok(j) \leq B, \; M \subseteq S
    \]
  - 使用带成本函数的CELF惰性贪婪算法求解，具有常数因子近似保证。
  - 与经典MMR的区别：覆盖项对**整个候选池**求和（而非仅对已选集），且覆盖权重由每个目标项的相关性加权，因此只覆盖相关区域。
- **系统集成**：
  - 作为**可插拔的上下文引擎**（PACMS）嵌入OpenClaw智能体框架。
  - 通过`assemble()`挂钩统一处理所有候选源，包括工具输出；设置`ownsCompaction=false`将摘要压缩工作委托给运行时。
  - 选择器实现为一个Python类，支持五种策略（pacms, topk, lastk, rag, mmr），共享嵌入和令牌估算。
  - 服务通过本地FastAPI暴露`/select`和`/select_debug`接口，嵌入缓存保持热状态以降低延迟；插件失败时软回退到就近截断。

## 3. 实验设计

  - 从六种问题类型中均匀随机抽取**100个问题**作为共享样本，用于证据局合召回率和端到端问答准确率两项评估。
- **对比方法（Baselines）**：

  - lc‑mmr：LangChain中的MMR实现，通过遍历输出排名并截断至预算。

  - **端到端问答准确率**：使用GPT‑5‑mini和GPT‑5.4‑mini作为读者，GPT‑4o‑mini作为裁判，按官方CORRECT/WRONG标准评分。

## 6. 主要结论与发现
- **覆盖率随预算增长显现优势**：在极低预算（20%）下，PACMS略逊于top‑k；当预算提升至45%和70%时，PACMS与lc‑mmr一同压制了注入的冗余，反超top‑k。
- **召回率与MMR持平但QA显著领先**：
  - 在45%预算、R=2条件下，PACMS的证据回合召回率（0.909）与lc‑mmr（0.895）接近，均低于top‑k（0.933）。
  - 然而端到端QA准确率上，PACMS比lc‑mmr高出**8个百分点（GPT‑5‑mini）到12个百分点（GPT‑5.4‑mini）**，甚至超越top‑k（+2~4个百分点），且这种优势随读者能力增强而扩大。
- **中心发现**：召回率与可提取性（QA）存在分歧，设施地点覆盖生成的提示比成对多样性或纯相关性排名更易于下游模型提取答案。
- **方法排名稳定**：PACMS > top‑k > lc‑mmr ≥ last‑k 在两种读者下均成立。
- **就近截断彻底失败**：last‑k在所有冗余和预算条件下召回率趋近于零，QA准确率不随模型增强而提升。

## 7. 优点
- **原理性目标函数**：采用单调子模函数最大化覆盖，有常数因子近似保证，优于MMR的成对启发式。
- **可插拔架构**：作为独立上下文引擎无缝集成到现有智能体框架中，切换策略仅需更改配置，降低使用门槛。
- **统一候选池处理**：平等对待记忆、对话和工具输出，首次将工具输出作为可选择的单元。
- **可解释性与可控性**：提供实时保留/丢弃可视化、预算滑块和策略切换，用户可动态观察选择行为。
- **软回退机制**：服务不可用时自动降级为传统截断，保证系统鲁棒性。
- **开箱即用的开源实现**：发布选择器、评估工具和OpenClaw插件。

## 8. 不足与局限
- **注入冗余的真实性不足**：使用的模板化自然变异与实际对话的多样性仍有差距，尚未在完全真实的LLM交互场景中验证。
- **问答评估覆盖有限**：仅考察了单一预算（45%）和单一冗余水平（R=2），缺乏不同压力场景下的消融。
- **未提供全上下文基线**：受阅读器最大长度限制，无法与理想上限对比，QA绝对得分偏低，可能影响直观感受。
- **MMR在极低预算下的表现异常**：20%预算时lc‑mmr得分较低，作者怀疑与LangChain默认λ=0.5有关，但未深入探究。
- **跨会话记忆管理未涉及**：目前仅关注单次组装，没有利用选择统计来决定记忆条目的持久化、压缩或淘汰。
- **未在真实多智能体或在线强化学习环境中评估**，未来需在更动态的部署场景中验证。
- **嵌入模型固定**：相关性度量依赖于单一的静态嵌入模型，未测试自适应或可学习的相关性计算。
