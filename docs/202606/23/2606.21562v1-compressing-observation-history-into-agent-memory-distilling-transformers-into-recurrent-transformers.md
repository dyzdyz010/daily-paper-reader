## 摘要
Transformer是人工智能的主力，在建模序列数据方面性能强大，但处理长序列时其计算成本变得难以承受。我们针对长时域流式视觉和机器人应用，如无地图姿态估计，其中存储和维护观察历史尤为不切实际。循环Transformer通过维护固定大小的记忆来克服这一限制，但其性能落后于在完整观察历史上运行的Transformer。我们认为这一差距并非源于架构限制，而是源于这些模型学习压缩过去信息的方式不同。由于无法访问观察历史，循环模型必须在每一步明确决定将什么保留在记忆中，这是一个难度大得多的学习问题。在这项工作中，我们提出一种蒸馏方法，将经典全历史Transformer的压缩策略迁移到循环变体中。我们通过设计一个教师模型来实现这一点，该教师模型显式地将其观察历史压缩为固定大小的瓶颈表示。通过使用该瓶颈表示直接监督学生记忆，我们对齐了这两种压缩机制。我们证明，该方法能够以线性时间复杂度训练循环潜在机器人记忆，同时显著缩小与全历史Transformer的性能差距。

## Abstract
Transformers are AI's workhorse with strong performance in modeling sequential data, but their computational cost becomes prohibitive when processing long sequences. We target long-horizon streaming vision and robotics applications like map-free pose estimation, where it is particularly impractical to store and maintain a history of observations. Recurrent Transformers address this limitation by maintaining fixed-size memory but their performance lags behind that of transformers operating over the full observation history. We argue that this gap does not stem from architectural limitations, but from differences in how these models learn to compress past information. Without access to an observation history, recurrent models must explicitly decide what to retain in memory at each step, a significantly harder learning problem. In this work, we propose a distillation approach that transfers the compression strategy of a classical full-history transformer to a recurrent variant. We enable this by designing a teacher model that explicitly compresses its observation history into a fixed-size bottleneck representation. By directly supervising the student's memory with this bottleneck representation, we align the two compression mechanisms. We show that this approach allows to train a recurrent latent robotic memory with linear-time complexity while substantially narrowing the performance gap to full-history transformers.

### 2. 论文提出的方法论

*   **核心思想**：通过知识蒸馏，将一个能够访问完整历史并学习显式压缩策略的“教师”模型的知识，直接迁移给一个“学生”循环模型。具体来说，是直接监督学生模型的“记忆”表示 \(M_t\)，让它在每个时间步都去拟合教师模型从完整历史中压缩出的“瓶颈”表示 \(\tilde{B}_t\)。

*   **关键技术与架构设计**：
    *   **教师模型：潜在瓶颈历史Transformer (Latent Bottleneck History Transformer, LBHT)**
        *   **架构**：在标准Transformer的输入序列中添加一组 \(B\) 个可训练的“瓶颈”令牌（bottleneck tokens）。在处理完整个观测序列 \(\{o_1, ..., o_t\}\) 后，这些瓶颈令牌经Transformer层上下文处理后，得到的输出 \(\tilde{B}_t\) 就是对整个历史的一种固定大小、显式的压缩表示。这类似于Perceiver Resampler架构。
        *   **关键特性**：该压缩过程是在没有访问当前查询（query）目标的情况下完成的，因此 \(\tilde{B}_t\) 可以被解释为关于场景的潜在“地图”或表示。
        *   **非因果性**：教师模型采用非因果（双向）注意力，允许过去的信息被未来的观察重新上下文化，以获得更强大的表示。虽然这不适用于流式部署，但其作为教师能提供更优的监督信号。
    *   **学生模型：循环Transformer**
        *   **基座架构**：采用Kinaema模型，这是一个高容量循环架构，其记忆 \(M_t\) 由 \(B\) 个嵌入向量组成的记忆库构成。
        *   **记忆更新流程**：更新分为三步：1) **校正**：将上一步记忆 \(M_{t-1}\) 与当前观测 \(o_t\) 结合并投影；2) **上下文化**：通过自注意力机制让记忆库中的各向量交互；3) **门控**：使用共享权重的GRU门控机制筛选并更新记忆，以解耦记忆容量和门控参数。
    *   **蒸馏对齐**：
        *   **维度对齐**：教师模型的瓶颈令牌数 \(B\) 和嵌入维度 \(E\) 被设计成与学生模型的记忆库完全一致，这使得可以直接计算两者表示之间的L1损失。
        *   **分段蒸馏策略**：如图3所示，为了提高学习稳定性和利用教师压缩不同长度序列的知识，一个完整的长序列rollout被分解为多个长度递增的片段。对于每个片段的结尾时间步 \(e_n\)，学生模型当时生成的记忆 \(M_{e_n}\) 都要与教师模型从对应序列片段 \(\{o_1, ..., o_{e_n}\}\) 中压出的 \(\tilde{B}_{e_n}\) 计算蒸馏损失 \(L_{dist_n} = || M_{e_n} - \tilde{B}_{e_n} ||_1\)。
        *   **梯度截断**：为了防止GPU显存爆炸并保证训练稳定性，对蒸馏损失的反向传播步长进行了限制。
