## 摘要
现有的嵌入模型本质上是静态的：它们孤立地编码文本片段，忽略其周围的上下文和时间顺序。本文介绍了 EvoEmbedding，一种能够生成可演化表征以用于检索的新型嵌入模型。它专为长上下文场景而设计，其中信息是动态的、顺序性的，并且需要持续的状态跟踪。我们的设计很简单：EvoEmbedding 在顺序处理输入时维护一个持续更新的潜在记忆，并利用该记忆与原始内容共同生成可演化的嵌入。因此，对于相同的查询，我们的模型根据不断变化的上下文调整其表征，以检索不同的目标，超越了静态语义搜索。为了使模型具备这种能力，我们构建了 EvoTrain-180K，一个多样化的数据集，用于联合优化潜在记忆和检索。此外，我们引入了一个记忆队列，以防止循环编码过程中的表征崩溃，同时采用了分段批处理技术，解决了显著的长度差异问题，并将训练速度提高了 3.8 倍。大量实验表明，我们的模型不仅在一系列长上下文检索基准上优于更大规模的专用模型（例如 Qwen3-Embedding-8B 和 KaLM-Embedding-Gemma3-12B），而且还能很好地泛化到上下文长度是其训练窗口 10 倍的下游任务（例如个性化）中。值得注意的是，EvoEmbedding 能够无缝集成到智能体工作流中以提升性能。例如，配备了我们的模型的朴素 RAG 流水线就超过了专用的智能体记忆系统。项目页面：https://clare-nie.github.io/EvoEmbedding。

## Abstract
Existing embedding models are inherently static: they encode text segments in isolation, ignoring their surrounding context and temporal order. This paper introduces EvoEmbedding, a novel embedding model that generates evolvable representations for retrieval. It is tailored for long-context scenarios, where information is dynamic, sequential, and requires continuous state tracking. Our design is simple: EvoEmbedding maintains a continuously updated latent memory as it sequentially processes inputs, and uses it alongside the raw content to jointly generate evolvable embeddings. Consequently, for the same query, our model adapts its representation to retrieve distinct targets based on the evolving context, going beyond static semantic search. To equip the model with this capability, we construct EvoTrain-180K, a diverse dataset for the joint optimization of latent memory and retrieval. Furthermore, we introduce a memory queue to prevent representation collapse during recurrent encoding, alongside segment-batching techniques that tackle significant length variance and accelerate training by 3.8$\times$. Extensive experiments show that our model not only outperforms larger-scale specialists (e.g., Qwen3-Embedding-8B and KaLM-Embedding-Gemma3-12B) across a range of long-context retrieval benchmarks, but also generalizes well to downstream tasks (e.g., personalization) with contexts 10$\times$ longer than its training window. Notably, EvoEmbedding seamlessly integrates into agentic workflows to boost performance. For instance, a naive RAG pipeline equipped with our model surpasses dedicated agentic memory systems. ## 1. 核心问题与研究动机
- **现有嵌入模型的静态缺陷**：传统文本嵌入模型（如Sentence-BERT、E5、Qwen3-Embedding等）将文本片段孤立编码，完全忽略了片段之间的**时间连续性**和**上下文关联**。
- **长上下文检索的挑战**：在动态、顺序的长上下文场景（如持续对话、多阶段任务）中，静态表示会导致检索系统无法追踪状态变化，频繁出现**信息过时**或**检索噪声**。
- **智能体记忆的局限**：当前智能体RAG系统为弥补静态嵌入的不足，引入了复杂的记忆结构、查询重写、重排序器等，但这类方案**计算开销大、延迟高**，且仍因丢失全局上下文而次优。
- **本文目标**：设计一种嵌入模型，使其能够根据不断变化的上下文**动态演化表示**，从而在相同查询下输出与当前状态匹配的检索结果。

## 2. 方法论
### 2.1 核心思想：可演化嵌入与潜在记忆
- EvoEmbedding在顺序处理长文本分段（segments）时，同时执行两项并行任务：
  - **记忆演化**：利用当前分段 \(x_t\) 和上一时刻的潜在记忆 \(M_{t-1}\)，通过大语言模型生成新的潜在记忆 \( \tilde{M}_t \)，并更新至固定容量的**FIFO记忆队列** \(M_t\)。
  - **表示生成**：将当前分段与历史记忆结合，通过另一组参数生成**上下文可感知的嵌入向量** \(v_t\)。
- 其核心公式：\(\tilde{M}_t = \pi_{\theta_m}(x_t, M_{t-1})\)，\(v_t = \pi_{\theta_r}(x_t, M_{t-1})\)；查询时仅执行表示生成任务。

### 2.2 关键技术设计
- **潜在记忆队列（Latent Memory Queue）**：
  - 容量 \(C = L \times K\)，存储最近 \(L\) 步生成的 \(K\) 维潜在令牌。
  - FIFO机制保证历史记忆最多被循环编码 \(L\) 次，防止表征崩溃；固定容量显式约束模型学会融合新旧知识，并控制计算复杂度。
- **动态分段批处理（Dynamic Segment-Batching）**：
  - 将 \(k\) 个连续分段拼接后并行前向计算，保证总长度不超过阈值，从而大幅提升训练/推理效率（**3.8倍加速**）。
- **多LoRA设计**：
  - 使用两个独立的低秩适配器（LoRA）分别负责记忆演化和表示生成，冻结骨干LLM参数，避免灾难性遗忘，并可灵活切换。

### 2.3 训练与优化
- **联合损失函数** \( \mathcal{L} = \mathcal{L}_{\text{mem}} + \mathcal{L}_{\text{con}} \)：
  - 记忆损失 \(\mathcal{L}_{\text{mem}}\)：基于生成的记忆 \(M_t\) 和查询 \(q\) 预测答案 \(y\)，利用交叉熵损失，**反向传播至记忆模块**，使其学习生成与基座LLM语义空间兼容的潜在状态。
  - 对比损失 \(\mathcal{L}_{\text{con}}\)：针对每个样本内的动态正负例池，采用**长度加权的多正例对比损失**，平衡不同长度样本的学习难度，使查询表示与相关片段对齐。
- **EvoTrain-180K 数据集构建**：
  - 三阶段自动化流水线：① 从网络文本、合成对话、记忆片段中构建多样化原始上下文；② 利用40余种模板和多种LLM生成简单/复杂QA对；③ 使用Gemini-3.1-Pro进行正例标注和质量验证（过滤幻觉、确保依赖历史）。
  - 共184,137个样本，单样本最大12K tokens、256分段，均长约1.3K tokens，远超测试时10倍+的长度，旨在训练模型的长上下文演化能力。
