## 摘要
记忆系统对于个性化大语言模型（LLMs）至关重要。然而，现有系统主要依赖语义相似性进行检索，可能遗漏语义重叠有限但逻辑上十分关键的记忆。当前的基准测试仍不足以评估这一问题。为弥补这一不足，我们构建了 IMLogic，这是首个针对长对话场景中隐式逻辑记忆检索的高质量基准。受此挑战启发，我们提出了根记忆（root memory），一种结构化的、保留决策的表征形式，能够从长期用户历史中提炼可复用的个性化逻辑。随后，我们提出 RootMem，一个即插即用的框架：该框架首先将原始历史提炼为结构化的根记忆，然后使用基于 LLM 的路由器来激活逻辑相关的根记忆，从而用个性化决策逻辑补充语义检索。大量实验表明，RootMem 显著优于最强的检索基线，并持续提升现有记忆智能体的准确性。我们的基准和代码将发布于 https://anonymous.4open.science/r/IMLogic-DBB3。

## Abstract
Memory systems are essential for personalized Large Language Models (LLMs). However, existing retrieval methods in these systems primarily rely on semantic similarity, potentially missing logically critical memories with limited semantic overlap. Current benchmarks remain inadequate for evaluating this problem. To address this gap, we construct IMLogic, the first high-quality benchmark targeting implicit logical memory retrieval in long-dialogue scenarios. Motivated by this challenge, we introduce root memory, a structured, decision-preserving representation that distills reusable personalized logic from long-term user histories. We then propose RootMem, a plug-and-play framework that first distills raw histories into structured root memories and then uses an LLM-based router to activate logically relevant ones, complementing semantic retrieval with personalized decision logic. Extensive experiments demonstrate that RootMem significantly outperforms the strongest retrieval baselines and consistently boosts the accuracy of existing memory agents. Our benchmark and codes will be available at https://anonymous.4open.science/r/IMLogic-DBB3.

## 2. 方法论：核心思想与关键技术细节
**核心思想：根记忆（Root Memory）**  
- 从用户长期历史中提炼出**结构化的、保留决策逻辑的记忆单元**，称为**根记忆**。
- 每个根记忆单元包含：
  - `Execution Rules (Ri)`：激活条件与响应指导规则。
  - `Personalized Logical Evidence (Ei)`：支撑规则的个性化事实与约束。
- 强调“如何基于用户历史中的隐含逻辑约束未来响应”的知识，而不仅是抽象事实。

**RootMem 框架**  
- **离线阶段——根记忆构建**：
  - 周期性聚合新记忆批次，利用LLM提取器`ΦC`更新或新建根记忆单元。
  - 更新过程包括丰富证据、精炼规则、合并重叠约束。
  - 公式：`U_i^(t) = ΦC(U_i^(t-1), M_t,i)`，单元数由个性化逻辑多样性动态决定（实现中设上限控制成本）。
- **在线阶段——根记忆路由**：
  - 使用LLM路由器`ΦR`评估用户的当前查询`q`，判断每个根记忆单元是否需激活。
  - 激活条件：`U_act = {U_i | ΦR(q, R_i) = 1}`。
  - 将激活单元的规则和证据拼接成**根记忆块**，与语义检索得到的`M_sem`一起组成逻辑增强上下文`C`。
  - 公式：`C = Conc.( {[R_i, E_i] | U_i ∈ U_act}, M_sem )`。

**关键创新**  
- 将记忆从原始片段转变为**可路由的决策规则**，使LLM能根据查询激活深层逻辑约束，弥补纯语义检索的盲区。

## 6. 主要结论与发现

- **隐式逻辑检索是现有方法的共同软肋**：最强语义/混合基线准确率仅43.50%，错误集中在偏好陷阱和通用陷阱。
- **RootMem显著提升逻辑检索能力**：在memory-level上相对最佳基线提升27.23%；在conversation-level上平均提升下游系统准确率26.36%。
- **根记忆补偿了逻辑缺失信息**：RootMem使通用错误减少20.06%，偏好错误减少14.04%。
- **泛化性强**：跨不同LLM骨干均获一致收益（例如配合LightMem相对提升71.52%）。
- **开销合理**：离线处理主要成本，在线路由轻量。
