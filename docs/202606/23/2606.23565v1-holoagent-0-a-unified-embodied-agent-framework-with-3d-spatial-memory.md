## 摘要
LLM智能体在数字环境中遵循实用的执行循环：它们对结构化状态进行推理，调用工具，检查反馈，并修正动作。将这一循环扩展到物理机器人非常困难，因为物理执行是连续的、依赖于具体形态、具有不确定性且受安全约束。现有的具身人工智能系统在操作、空间理解、导航和人形控制方面取得了进展，但这些能力通常仍然是专门的模块或松散耦合的决策循环。在这项工作中，我们提出了HoloAgent-0，一个用于真实世界机器人部署的统一具身智能体框架。Embodied AgentOS将语言指令转换为可执行的技能图，调度机器人资源，监控执行，并根据运行时反馈触发澄清或重新规划。HoloAgent-0通过三个耦合层组织异构机器人模型和控制器：用于闭环执行的Embodied AgentOS、用于物理世界定位的三维空间记忆以及用于机器人动作的具身技能。我们在真实硬件上部署了HoloAgent-0，并评估了其在运动生成、物体搜索、跨机器人协调和移动操作中的空间记忆、长时域导航和闭环执行能力。

## Abstract
LLM agents follow a practical execution loop in digital environments: they reason over structured states, invoke tools, inspect feedback, and revise actions. Extending this loop to physical robots is difficult because physical execution is continuous, embodiment-dependent, uncertain, and constrained by safety. Existing embodied-AI systems have advanced manipulation, spatial understanding, navigation, and humanoid control, but these capabilities often remain specialized modules or loosely coupled decision loops. In this work, we introduce HoloAgent-0, a unified embodied agent framework for real-world robot deployment. Embodied AgentOS converts language instructions into executable skill graphs, schedules robot resources, monitors execution, and triggers clarification or re-planning from runtime feedback. HoloAgent-0 organizes heterogeneous robot models and controllers through three coupled layers: Embodied AgentOS for closed-loop execution, 3D spatial memory for physical world grounding, and embodied skills for robot action. We deploy HoloAgent-0 on real hardware and evaluate its spatial memory, long-horizon navigation, and closed-loop execution across motion generation, object search, cross-robot coordination, and mobile manipulation.

## 2. 论文提出的方法论

### 2.1 核心思想与整体架构

- **三层耦合设计**：
  - **Embodied AgentOS**：将自然语言指令解析为可执行的技能图（skill graph），负责任务规划、资源调度、执行监控和异常恢复，形成闭环。
  - **Memory Layer（记忆层）**：提供持久的**空间记忆**（几何、语义、分层场景图）和**时间记忆**（目标状态、执行追踪、经验总结），为规划与验证提供物理世界的持续接地。
  - **Skill Layer（技能层）**：将异构机器人能力封装为**类型化技能调用**（typed skill calls），包含命令参数、进度信号、失败模式和可恢复性，使AgentOS能够组合导航、操作、全身运动等底层策略。

- **闭环执行原则**：
  - 以 **“观察→检索→行动→监控”** 循环替代一次性文本规划。
  - 始终以持久化记忆为主要上下文，而非仅依赖当前视图或对话。
  - 通过 **ROS2 命令/状态总线** 连接各层，保持模块化与可观测性。

### 2.2 关键技术细节

- **类型化技能接口**：
  - 每个技能声明命令名、参数、前提条件、预期效果和后置条件，如 `move_to(room=kitchen)`、`pick(object=mug)`。
  - 后端发布**运行时状态流**，报告进度、成功/失败、失败模式、置信度、可恢复性，供AgentOS决策是否继续、重试、澄清或重规划。

- **HoloNavi 导航管道**（技能层实例）：
  - **层次化物体导航**：利用LLM解析目标，然后快速匹配分层多模态场景图（HMSG）中的楼层→房间→视图→物体节点。
  - **在线验证循环**：将候选视图送入VLM进行视觉验证，失败时旋转收集周围视图重新检测验证。
  - **主动空间探索**：当记忆或验证失败时，AgentOS触发基于信息增益、语义相关性和安全约束的前沿点探索，动态扩展地图。

- **3D空间记忆构建**：
  - **几何记忆**：支持两种后端（LiDAR-based 或纯视觉 GeoFlow-SLAM++ 构建）。纯视觉后端利用多相机3D基础模型预测稠密深度，无需物理深度传感器。
  - **语义记忆**：将SAM2分割掩膜的 SigLIP 特征（全图、掩码区域、最小包围框）按加权融合（公式 d = Σwᵢ⊙dᵢ）提升到3D实例体素，并通过投影 IoU 匹配维持持久实例身份。
  - **分层多模态场景图（HMSG）**：按 Floor→Room→View→Object 组织，融合语义、几何和拓扑边，作为AgentOS从粗到细的空间检索索引，支持 VLM 验证。

- **时间记忆**：
  - 记录当前目标、待执行子目标、技能调用结果、失败追溯和恢复策略，使长时域任务在中断后仍可恢复上下文。

- **记忆更新**：
  - 接收到新观测时，首先重定位，然后刷新冲突的几何与语义，仅更新受影响的 HMSG 子图；技能结果（如抓取、导航阻塞）同样触发相应空间记录刷新。

## 6. 论文的主要结论与发现

- **闭环执行显著提升长时域导航**：HoloAgent-Nav 在模拟 HM3D-ObjNav 上取得 82.6% SR 和 42.8% SPL，优于最强零样本系统 MSGNav 和基准 FSR-VLN；在真实机器人上达到 97.7%~98.9% 的目标到达率，且在严格定位阈值（1m）下保持高成功率，表明闭环验证和反馈驱动的路径效率优于一次性规划。

## 7. 优点

- **系统级创新**：不是重新训练单一模型，而是提出一个能集成异构技能、记忆与监控的闭环运行时，这是具身智能体走向实用化的重要设计。
- **记忆中心的封闭环路**：通过 HMSG 和持久时间记忆，将规划从“当前观察”提升到“历史与空间全貌”，使长时域任务恢复与重规划有据可依。
- **类型化技能接口**：将机器人能力抽象为可组合、可监控的动作单元，分离高层推理与底层控制，便于不同机器人平台扩展。
