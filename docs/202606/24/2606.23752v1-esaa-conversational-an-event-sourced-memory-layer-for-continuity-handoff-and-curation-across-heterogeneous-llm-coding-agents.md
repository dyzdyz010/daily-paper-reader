## 摘要
随着上下文窗口填满、会话结束或特定智能体更适合完成子任务，软件开发人员越来越多地与多个LLM编码智能体交互，在Codex、Grok、Claude Code等辅助工具之间切换。然而，每个智能体将其对话保存在私有且供应商特定的日志中，导致会话状态漂移：当一个智能体接管时，先前智能体所建立的目标、决策、未完成任务和依据无法可靠获取。本文提出\emph{ESAA-Conversational}，它是事件溯源智能体架构（ESAA）~\cite{esaa}的领域特化，用于跨异构智能体共享会话记忆。该方法将可见对话视为本地事件存储：钩子和观察器捕获可见的对话轮次，将其规范化到仅追加的\texttt{activity.jsonl}中，并确定性地投影出读取模型，如\texttt{handoff.md}、\texttt{state.md}、\texttt{decisions.md}和\texttt{tasks.json}。机械捕获无需LLM推理；智能体仅在显式策管时使用判断，通过领域命令记录持久决策和会话任务。公开的v1.1.0版本实现了一个PowerShell命令行界面，包含\texttt{init}、\texttt{enable-hooks}、\texttt{sync}、\texttt{project}、\texttt{verify}、\texttt{context}、\texttt{decide}和\texttt{task}等命令；包含了\texttt{workspace\_root}隔离和写路径锁文件；并作为一个独立包发布，带有空的公开日志。一项包含570个开发实验室事件的自参照案例研究表明，异构智能体可以通过共享日志进行协作，无需直接的智能体间通道，而公开分发通过排除私有对话历史来保护隐私。

## Abstract
Software developers increasingly work with multiple LLM coding agents, switching among tools such as Codex, Grok, Claude Code, and other assistants as context windows fill, sessions end, or a particular agent becomes better suited to a subtask. Each agent, however, persists its conversation in a private and vendor-specific log. The result is conversational state drift: goals, decisions, open tasks, and rationales established with one agent are not reliably available when another agent takes over. This paper presents \emph{ESAA-Conversational}, a domain specialization of Event-Sourcing Agent Architecture (ESAA)~\cite{esaa} for shared conversational memory across heterogeneous agents. The method treats the visible conversation as a local event store: hooks and watchers capture visible turns, normalize them into an append-only \texttt{activity.jsonl}, and deterministically project read models such as \texttt{handoff.md}, \texttt{state.md}, \texttt{decisions.md}, and \texttt{tasks.json}. Mechanical capture does not require LLM inference; agents use judgment only for explicit curation, recording durable decisions and conversational tasks through domain commands. The public v1.1.0 release implements a PowerShell CLI with \texttt{init}, \texttt{enable-hooks}, \texttt{sync}, \texttt{project}, \texttt{verify}, \texttt{context}, \texttt{decide}, and \texttt{task}; includes \texttt{workspace\_root} isolation and a write-path lockfile; and is distributed as a greenfield package with an empty public log. A self-referential case study with 570 development-lab events shows that heterogeneous agents can collaborate through a shared log without a direct agent-to-agent channel, while the public distribution preserves privacy by excluding the private conversational history.

### 2. 论文提出的方法论

- **核心思想：事件溯源记忆层 (Event-Sourced Memory Layer)**
    - **领域特化**：`ESAA-Conversational`是`ESAA`（事件溯源智能体架构）的一个领域特化。它不治理智能体对项目做了什么（ESAA核心的功能），而是治理智能体对**对话本身知道什么**。
    - **事件溯源**：将所有可见的对话轮次和策展活动视为不可变的事件，追加到一个本地的事件存储中。对话状态（读取模型）从此事件存储中确定性投影（计算）出来，而非被手动编辑。

- **关键技术细节与流程**
    - **写入路径 (Write Path)**:
        1.  **逆反摄入 (Inverted Ingestion)**: 智能体不需要共享一个通用运行时或协议。系统通过**钩子 (Hooks)** 或**观察器 (Watchers)** 监听各智能体的本地日志或配置，捕获可见的对话文本。
        2.  **规范化成事件**: 钩子调用CLI（如 `conversation-esaa sync --agent grok`）将原始日志归一化为标准的 `conversation_turn` 事件，并追加到唯一的真实来源文件 `.conversation-esaa/activity.jsonl` 中。
        3.  **显式策展 (Explicit Curation)**: 智能体通过特定的领域命令，显式地记录**持久知识**和**操作延续性**。例如：
            - `conversation-esaa decide "<决策>"`：记录一个 `decision.recorded` 事件。
            - `conversation-esaa task create "<任务>"`：创建一个 `task.created` 事件。
        4.  **并发控制**: 使用一个简单的锁文件 (`run/*.lock`) 来序列化写操作。
    - **读取路径 (Read Path) - 确定性投影**:
        - 系统通过 `project` 命令从 `activity.jsonl` 唯一真实来源确定性投影出各种读取模型，这些文件**不被人为编辑**：
            - `handoff.md`: 为冷却启动的智能体提供的入场契约（摘要）。
            - `state.md`: 会话状态的紧凑摘要。
            - `decisions.md`: 所有已策展决策的投影。
            - `tasks.json`: 会话任务看板的投影。
    - **选择性上下文读取**: `context` 命令支持分页、过滤地读取事件日志（如按智能体、主题、窗口大小），避免新智能体因摄入完整对话历史而消耗大量token，实现了**令牌经济 (Token Economy)**。
    - **工作区隔离**: 通过强制使用 `workspace_root` 字段，确保不同项目的对话日志完全隔离。
