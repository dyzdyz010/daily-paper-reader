## 摘要
混合智能体（MoA）架构通过将多个大语言模型智能体组织成分层推理管道，改善了推理时缩放。然而，现有的 MoA 变体随着深度增加无法维持性能提升，表现出退化、过早停滞或饱和。我们提出了 ReM-MoA，一种记忆增强的 MoA 框架，通过两种机制维持缩放：（1）一个排序推理记忆，使用一个比较评审智能体持续存储并排序所有层的推理轨迹；（2）一个精心策划的多样化记忆路由方案，使不同的智能体接触到成功和失败轨迹的不同组合，在传播高质量推理的同时保持探索多样性。我们进一步引入了一个可选的多领域评审蒸馏管道，通过前沿模型监督来提高排序质量。在涵盖数学、形式逻辑、代码、知识和常识的五个推理基准上，ReM-MoA 在深度和宽度缩放上均持续优于先前的 MoA 变体，并且其优势随着深度增加而扩大，确立了结构化的跨层推理记忆是可扩展多智能体推理中缺失的关键机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

### 1. 论文的核心问题与整体含义

- **研究动机**：现有的混合智能体（Mixture-of-Agents, MoA）架构在推理时通过堆叠多层LLM智能体来提升性能，但实际中随着层数（深度）或每层智能体数量（宽度）增加，性能并不能持续提升。已有的MoA变体（如标准MoA、RMoA、AttentionMoA）会出现性能退化、过早停滞或饱和现象。
- **核心问题**：导致缩放崩塌的三种机制：① 原始跨层拼接传播错误；② 局部跨层通信无法保留早期高质量推理；③ 智能体因上下文趋同而丧失探索多样性，聚合效果下降。
- **整体含义**：要实现可持续的MoA缩放，必须同时解决**质量保持**和**多样性保持**两大难题。本文提出一种新的记忆增强型MoA框架，以结构化的跨层排序推理记忆作为支撑，使模型能够随着深度和宽度扩展而持续获得性能增益。

### 2. 方法论

#### 2.1 核心思想

- 将MoA的缩放问题视为**质量-多样性联合保持**问题，而非单纯的聚合或交互问题。
- 引入一个附加的、跨层共享的**排序推理记忆**，并结合**精心策划的多样化记忆路由**，使得每一层的智能体能够接触到不同类型的高质量和低质量推理轨迹，从而在保留优质推理的同时维持探索的多样性。

#### 2.2 关键技术细节

##### (1) 排序推理记忆（Ranked Reasoning Memory）
- 每一层（层 \(l\)）的 \(N\) 个提议智能体生成推理轨迹 \(r_{l,j}\)。
- 一个**评审智能体（Reviewer Agent）** 对所有轨迹进行成对比较打分（1-10分），并为每条轨迹生成一条针对性的**理由**：  
  \[
  (s_l, \rho_l) = R(x, r_l)
  \]
- 该层的三元组集合 \(M_l = \{(r_{l,j}, s_{l,j}, \rho_{l,j})\}_{j=1}^N\) 被追加到全局记忆 \(M_{\le l}\) 中。
- 记忆是追加式的，任何后续层都可以访问之前所有层的轨迹信息，克服了仅相邻层通信的局限。

##### (2) 精心策划的多样化记忆路由（Curated Diversified Memory Routing）
- 从第2层开始，每个提议智能体的输入不再仅仅是上层输出，而是 `问题 + 从记忆 \(M_{\le l-1}\) 中抽取的精选参考集 \(S_{l,j}\)`。
- 路由方案采用三种范式：
  - **对比型**：Top a ∪ Bot b（混合最高分和最低分轨迹）。
  - **正面强化型**：Top a（仅最高分轨迹）。
  - **失败模式专注型**：Bot a（仅最低分轨迹）。
- 同一层内的不同智能体被分配不同的路由函数，从而使得同一层内各智能体看到不同的精选轨迹组合，保持多样性。
- 例如 \(N=3\) 时，一个智能体接收对比集（Top1+Bot1），一个接收正面集（Top2），一个接收失败集（Bot2）。

##### (3) 可选的评审智能体蒸馏（Reviewer Distillation）
- 用强大前沿模型（如GPT-5.5）作为教师，通过ReM-MoA自身的运行轨迹生成训练数据。
- 使用LoRA对基础评审智能体进行参数高效微调，使其模仿教师的评分与理由。
- 训练数据涵盖多领域，使蒸馏后的评审器能跨领域泛化。

### 6. 主要结论与发现

- **ReM-MoA在全部配置下均优于所有基线**，且随着层数加深，优势幅度单调扩大（例如深度9层时比最强基线高出约4.9个点）。

- **三大失败机制**得到实证确认：标准MoA退化、RMoA早期停滞、AttentionMoA饱和。
- **消融实验揭示**：跨层访问和路由多样性是维持缩放的关键驱动力，理由文本起辅助精炼作用。
- **评审蒸馏具有跨领域迁移性**：留一法训练的评审器仍能恢复蒸馏增益的约44%，说明其学习到的是通用评估能力而非记忆。

### 7. 优点

- **问题洞察深刻**：从质量与多样性平衡的角度剖析了MoA的缩放失败根源，并给出针对性解决方案。
- **方法设计精妙**：排序记忆和差异化路由既简单又有效，同步解决了质量保持和多样性坍缩两大矛盾。
- **实验严谨全面**：涵盖5个不同推理领域、深度和宽度两个缩放维度、详尽的消融和迁移实验，证据充分。
- **蒸馏创新实用**：利用框架自身轨迹进行多领域蒸馏，不仅提升性能，且学到的评价能力可泛化，降低了对外部强模型信号的依赖。

### 8. 不足与局限

- **宽度探索受限**：宽度仅测试到 \(N=8\)，未进一步探索更大规模的宽度缩放下的行为。
- **智能体规模单一**：只使用了7-8B级的模型作为提议器，尚未验证在更大（30B-70B）或更小（1-3B）模型上的效果。
- **评审器调用开销**：虽然与AttentionMoA相比更低，但仍会带来\(L\)次额外LLM调用，增加推理延迟和计算成本。
- **内在偏见继承**：评审器和提议器本身作为LLM，可能带有可能的位置偏见、自我偏好等局限性；评分机制未能完全消除这些偏差。
- **高投入场景风险**：直接用于高风险推理任务时，仍需额外的安全对齐和验证措施。
