## 摘要
长期记忆使得语言模型智能体能够在多个会话中不断提升能力，保持对用户的准确且不断演化的理解。然而在实践中，这种记忆主要通过下游行为来评估，例如后续回答、个性化质量或任务成功率，这些评估仅间接测试了理解，而记忆本身则大多未经审计。我们认为，长期记忆应作为一种可审计的交互后产物来评估：在普通辅助之后，从智能体留下的记忆中能重建出怎样的结构化用户状态？我们在MEMPROBE中实例化了这一观点，这是一个基准测试，其中配备记忆的智能体协助模拟用户，每个用户携带一组基于分类的隐藏用户状态库，在一系列泄露受控的任务轨迹上交互后，从智能体生成的记忆中重建该状态库，支持全存储和top-k访问两种方式。MEMPROBE基于合成真值实现高效、可扩展的测量，涵盖50名模拟用户，每名用户有31个隐藏维度（共1550个恢复目标），并测试了5个代表性记忆系统。在测试最先进的记忆智能体时，我们发现成功辅助与可恢复记忆表现为两种不同的能力。任务完成度几乎饱和，即使对于无记忆基线也是如此，而类别平衡的恢复率保持中等水平（约0.6），并在top-k检索下进一步下降。MEMPROBE是首个直接研究记忆恢复的基准测试，重建系统保留的用户状态并根据真值进行评分。我们将恢复视为未来记忆智能体优化的具体目标，而MEMPROBE则是迈向一个环境的一步，在该环境中智能体被训练去记住其用户，随着了解时间增长而变得更加忠实。

## Abstract
Long-term memory promises LLM agents that grow more capable across sessions, maintaining an accurate, evolving understanding of the user that interaction forms. In practice, however, this memory is evaluated mostly through downstream behavior, such as later answers, personalization quality, or task success, which tests that understanding only indirectly and leaves the memory artifact itself largely unaudited. We argue that long-term memory should instead be evaluated as an auditable post-interaction artifact: after ordinary assistance, what structured user state can be reconstructed from the memory the agent leaves behind? We instantiate this view in MEMPROBE, a benchmark in which a memory-equipped agent assists simulated users, each carrying a hidden, taxonomy-anchored user-state bank, across a trajectory of leak-controlled tasks, after which that bank is reconstructed from the agent's resulting memory under both full-store and top-k access. Built on synthetic ground truth for efficient, scalable measurement, MEMPROBE spans 50 simulated users with 31 hidden dimensions each (1,550 recovery targets) and tests 5 representative memory systems. Testing state-of-the-art memory agents, we find that successful assistance and recoverable memory behave as distinct capabilities. Task completion nearly saturates, even for a memoryless baseline, while category-balanced recovery stays moderate (about 0.6) and drops further under top-k retrieval. MEMPROBE is the first benchmark to study memory recovery directly, reconstructing the user state a system retains and scoring it against ground truth. We see recovery as a concrete objective for future memory agents to optimize, and MEMPROBE as a step toward an environment where agents are trained to remember their users, growing more faithful the longer they know them.

### 1. 论文的核心问题与整体含义

*   **核心问题：** 当前对大型语言模型智能体长期记忆的评估方式存在根本性缺陷。现有基准测试主要衡量记忆的下游行为表现（如任务成功率、回答质量），但**任务成功不等于形成了良好的记忆**。一个智能体可能通过检索上下文片段或幸运猜测完成任务，而并未形成对用户的持久、准确的内在模型。记忆本身作为一个“黑箱”，其内容未被直接审计。
*   **整体含义（研究动机）：** 论文提出，长期记忆应被视为一种**可审计的交互后产物**。评估的核心问题应从“智能体是否答对了？”转变为“从智能体留下的记忆中，能重建出多少关于用户的隐藏状态？”。这标志着评估焦点从间接的行为输出，转向对记忆内容的直接度量，旨在推动智能体真正“认识”并记住其用户。

### 2. 论文提出的方法论

*   **核心思想：** MEMPROBE 通过“隐藏用户状态恢复”来直接探测记忆质量。它构建了一套流程，让携带**隐藏真值状态库**的模拟用户与记忆智能体交互，然后在交互结束后，仅从智能体的记忆中尝试**重建**该状态库，并将重建结果与真值对比评分。
*   **关键技术细节与流程：**
    *   **问题形式化：** 定义了一个从隐藏状态 \( u \) 到重建状态 \( \hat{u} \) 的三个损失性步骤：
        1.  **暴露：** 用户 \( u \) 通过对话揭示证据 \( e \)。
        2.  **收集：** 智能体的记忆策略 \( \pi_{\text{mem}} \) 从对话中提取信息，更新并形成最终记忆。
        3.  **恢复：** 从 \( m_{\text{final}} \) 中操作，重建对每个状态维度的估计 \( \hat{u}_i \)。
    *   **模拟用户与任务构造：**
        *   使用生成引擎创建具有丰富背景的**合成用户**，并为其定义了一个基于分类学的**隐藏状态库**，包含5大类别（技能、知识、情景、自我模型、辅助偏好）共31个维度，作为审计的“真值”。
        *   为每个隐藏维度设计一个**泄露受控的辅助任务**。这些任务邀请用户透露相关信息，但任务描述本身**不会直接泄露**目标维度的答案（通过一个盲审的“反钓鱼”评判器来保证）。
    *   **交互与模拟：** 模拟用户与配备了待测记忆系统的智能体进行多轮对话，智能体在对话过程中读写其记忆。除了记忆模块外，所有实验条件（任务、模拟器、助手框架）保持一致。
    *   **恢复与评分：**
        *   **双模式审计：** 通过两种方式从记忆中提取信息，以分离写入和读取的失败。
            *   **全存储：** 将整个记忆全部提供给一个“槽填充”模型，让其推断并填写用户状态。
            *   **top-k检索：** 仅提供通过系统自带检索接口按固定查询返回的前k条记忆。
        *   **评分：** 使用一个独立的语言模型评判器，将填充的状态与隐藏真值在5级量表上进行对比评分。最终得分在不同类别内取均值后再平均，得到类别平衡恢复分数。
    *   **失败归因：** 当一个维度恢复分数低时，通过三阶段流程追责：判断是任务设计问题、智能体未引出话题、模拟用户未透露，还是记忆系统本身未能捕获或检索到信息。

### 3. 实验设计

*   **数据集/场景：** MEMPROBE 基准本身就是一个合成的测试场景。它包含**50个模拟用户**，每个用户有**31个隐藏状态维度**，总计**1550个恢复目标**。交互发生在由生成任务驱动的日常辅助对话中。
*   **基准对比方法：** 测试了**5种代表性记忆系统**。
    1.  **`nomem`**： 无记忆能力的基线模型。
    2.  **`longctx_full`**： 存储所有对话轮次的原始文本，并使用句子嵌入进行检索。
    3.  **`amem`**： 遵循A-Mem系统，存储由语言模型编写的、带有链接的进化式笔记。
    4.  **`mem0`**： 遵循Mem0系统，从对话中提取并整合事实性记忆。
    5.  **`memt`**： 遵循Mem-T系统，使用一个经过训练的策略模型在分层记忆数据库上进行记忆操作。
*   **评测指标：**
    *   **主要指标：** 类别平衡恢复分数（两种访问模式下）。
    *   **次要指标：** 任务完成率、用户偏好评分、平均对话轮数、记忆占用空间、以及详细的失败归因分布。

    *   **客观公平性**：通过固定模拟用户、任务和助手框架，仅改变记忆模块，确保了对比的公平性。合成的真值库为评估提供了可扩展且无争议的客观标准。使用经过人工抽检的自动化评判器和多级失败归因，增强了结论的客观性。

### 6. 论文的主要结论与发现

*   **任务成功与记忆质量是分离的能力**：所有系统（包括无记忆基线）的任务完成度几乎都达到饱和（>99.8%），但隐藏用户状态的恢复率整体不高（类别平衡恢复率仅约0.6），证明“答得好”远不意味“记得好”。
*   **记忆面临“检索感知的整合”瓶颈**：系统能够捕获原始交互痕迹（全存储模式恢复尚可），但缺乏将其提炼为紧凑、可检索的用户状态的能力，导致信息在常规检索接口下不可达（top-k模式恢复率显著下降）。
*   **情景记忆是“开放的前沿阵地”**：在五大类别中，关于用户一次性过往经历的情景记忆最难恢复，因为它要求系统不仅记录事件本身，还要将其与产生意义的上下文和后果绑定。稳定的偏好则相对容易。
*   **写入策略是关键的杠杆**：仅仅调大存储容量或优化检索算法并不能解决记忆形成的问题，关键在于需要一个能够将短暂证据转化为持久、可复用声明的写入策略。

### 7. 优点

*   **创新的评估范式**：首次提出直接审计记忆内容而非评估下游行为的范式，为长期记忆研究提供了更直接、更根本的质量衡量标准。
*   **完备且可诊断的基准设计**：构建了包含“合成真值 -> 交互模拟 -> 双模式审计 -> 失败归因”的完整流水线，不仅能测量记忆“多好”，还能诊断“为何不好”，为系统优化指明具体方向。
*   **高度可控与可扩展**：利用合成数据和自动化评判，实现了对50个用户1550个目标的大规模、可复现的评估，同时保证了评估客体的标准化和一致性。
*   **实践指导性强**：实验发现直击当前记忆系统的设计要害，指出了检索感知整合、情景记忆绑定等关键的技术优化方向。

### 8. 不足与局限

*   **合成模拟的覆盖风险**：基准使用合成用户和场景，虽然保证了可控性，但可能无法完全代表真实世界用户的多样性、语言复杂性、隐私顾虑和文化差异，其结论的外推性需要谨慎。
*   **评判器偏差**：恢复质量的评分和失败归因高度依赖语言模型评判器，而这类评判器自身存在偏差（如对标签措辞的敏感性），尽管有人工抽检，但对于部分模糊边界的判断仍可能存在不稳定性。
*   **任务难度饱和**：辅助任务设计为避免泄露而相对简单直接，导致任务完成率过早饱和，使得这个指标失去了区分度，基准的难度梯度可以更丰富。
*   **应用范围限制**：当前的基准场景侧重日常辅助，未覆盖更复杂、更长周期、涉及多步骤项目或更高阶用户状态习得（如学习如何更好地与用户交互）的记忆能力。
