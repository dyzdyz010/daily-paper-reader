## 摘要
设备端语言模型代理通过积累检索记忆中的经验来改进，而非更新权重。这类记忆被严格限定大小并暴露在外：它消耗RAM和能量，通过微薄的上行链路触及对等节点，并因代理所读取的内容可写而成为攻击面。现有系统各自覆盖问题的一部分：代理记忆无预算控制地增长，设备端方法仅凭成功保留记忆条目，而中毒主要被作为攻击而非记忆治理问题来研究。我们提出了\sys{}，一个统一的每字节净值分数，用于治理代理的经验记忆生命周期。其核心思想是让预算担当策展人：每个条目按每字节的价值减去危害评分，如此便由同一尺度决定保留、共享与信任何者。\sys{}做出三项决策：(1) \textbf{保留}在RAM和能量预算下淘汰低价值字节；(2) \textbf{共享}仅当洞察的价值超过其上行成本时才发送；(3) \textbf{信任}根据来源放行对等条目。在语言模型代理任务漂移基准测试以及一个包含两个机械臂节点和一个枢纽的真实异构Jetson测试平台上，\sys{}将内存占用减少$2.7\times$倍，上行链路使用减少$2.4\times$倍，将注入成功率从0.75降至零，并提升受中毒或陈旧记忆污染案例的准确率。按净值策展可同时降低占用空间、能量、上行链路使用和注入成功率，而不降低准确率。在这种设定下，按净值遗忘反而改进代理，而非削弱它。

## Abstract
On-device language-model agents improve by accumulating experience in retrieved memory rather than by updating weights. This memory is hard-bounded and exposed: it consumes RAM and energy, reaches peers through a thin uplink, and becomes an attack surface because it is writable by what the agent reads. Existing systems each cover one part of this problem: agentic memories grow without a budget, on-device methods keep entries by success alone, and poisoning is studied mainly as an attack rather than as a memory-governance problem. We propose \sys{}, a single net-value-per-byte score that governs an agent's experience-memory lifecycle. The main idea is to let the budget act as the curator: each entry is scored as value minus harm, per byte, so one ruler decides what to keep, share, and trust. \sys{} makes three decisions: (1) \textbf{KEEP} evicts low-value bytes under the RAM and energy budget; (2) \textbf{SHARE} sends an insight only when its value exceeds its uplink cost; and (3) \textbf{TRUST} gates a peer entry by provenance. On language-model-agent task-drift benchmarks and a real heterogeneous Jetson testbed with two robot-arm nodes and a hub, \sys{} reduces memory by $2.7\times$ and uplink by $2.4\times$, drives injection success from 0.75 to zero, and raises accuracy on cases corrupted by poison or stale memory. Curating by net value reduces footprint, energy, uplink, and injection success together without reducing accuracy. In this setting, forgetting by net value improves the agent rather than weakening it.

### **2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程**

- **核心思想**：提出一个名为CURATOR的系统，其核心是使用一个**每字节净价值（Net-Value-Per-Byte）的单一分数 `ρ`** 来治理所有记忆相关的决策。让“预算”本身成为策展人，评估一个经验条目是否“值得”占用其字节足迹。
- **关键技术细节**：
    - **评分公式**：对于一个记忆条目 `m`，其综合分数 `ρ(m)` 由价值项 `V(m)` 减去危害项 `H(m)`，再除以其字节占用 `b(m)` 得出：`ρ(m) = (V(m) - λ * H(m)) / b(m)`。
    - **价值项 `V(m)`**：由三个因素乘积构成，综合评估条目的未来潜力。
        1.  **检索倾向性 `p(m)`**：度量条目与近期查询的语义相似度，基于一个滑动向量草图（不存储历史查询），计算成本低。
        2.  **条件有用性 `q(m)`**：这是一个无需标签的函数，通过在线流式头估计条目被纳入检索集后对代理答案自洽性的平均边际效用增益。
        3.  **抽象增益 `a(m)`**：衡量将原始轨迹压缩为抽象洞察后节省的字节数。一个未经压缩的原始轨迹可能价值为负，而提炼后可转为正向价值。
    - **危害项 `H(m)`**：由负迁移风险 `η(m)` 和来源风险 `π(m)` 构成。
        1.  **负迁移风险 `η(m)`**：通过计算条目嵌入与当前工作任务的马氏距离进行惩罚，对过拟合特殊情况的条目降权。
        2.  **来源风险 `π(m)`**：一个关键的安全组件，是一个基于固定先验的逻辑回归门，对条目的来源（外部/对等/自衍生）、自我强化次数和指令相似度等特征进行评分。一个被投毒的条目即使价值项得分高，也会因来源风险高分而被整体评为低`ρ`。
- **算法流程（三大决策）**：
    1.  **保留（KEEP）决策**：遵循贪心背包问题近似最优解。将所有记忆条目按 `ρ` 降序排列，在内存预算 `B` 内依次接纳，淘汰低分条目。同时通过虚拟队列动态控制能耗，使其保持在长期平均预算内。
    2.  **分享（SHARE）决策**：对等节点只广播一个查询草图向量，发送方据此重新计算条目在对等方的预估价值，仅当该价值净有害后仍高于上行链路成本阈值 `τ` 时才发送。内含去冗余机制，不发送与已发送条目过于相似的内容。
    3.  **信任（TRUST）决策**：接收方使用自己的本地`ρ`评分器（包括本地计算的倾向性和危害）评估接收到的条目，仅当分数高于信任阈值 `θ` 时才接纳。由于`π`项的存在，有可疑来源的投毒条目将被自动拒之门外。

- **任务与领域的泛化性**：实验集中在具身AI（ALFWorld）和BabyAI风格的离散任务上。其在更连续、开放式的自然语言对话或代码生成等领域的有效性有待验证。
- **单智能体记忆路径假设**：该方法治理的是代理的单个“经验记忆”。在复杂的多智能体交互或需要复杂推理链的环境中，可能还存在其他类型（如工作记忆、情景记忆）的攻击面和治理需求，论文未覆盖。
- **能量模型的简化**：能量消耗建模主要与检索和重嵌入步骤相关，虽然真实测量证实了节省，但理论上未涵盖设备上其他可能导致能耗波动的进程。
- **超参数λ的默认性**：虽然实验证明λ=1是一个安全普适的值，但在极端或全新的任务分布下，这个默认值可能不是最优的，可能需要一个自动调优机制。
