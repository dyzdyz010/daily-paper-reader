## 摘要
长期的机器人部署需要一种紧凑且可扩展的记忆系统，该系统能够保留细粒度的视觉语义，将观察结果锚定于空间和时间，并实现高效的存储与检索。本文提出了 RAVEN，一个面向长期机器人问答与导航的智能体记忆系统。RAVEN 将带有位姿和时间信息的视觉嵌入存储在向量数据库中，并在空间地图中进行检索锚定，以回答查询并导航至目标。通过直接操作视觉嵌入，RAVEN 避免了有损的图像到文本描述，并能在大规模场景下实现准确的语义、空间和时间检索。在多个模拟和现实世界的视频问答基准测试中，RAVEN 持续超越基于文本描述的记忆系统，并在长期任务上以低 10 倍的检索成本达到前沿视觉语言模型的水平。最后，我们在宇树 Go1 机器人上实例化了 RAVEN，用于基于自然语言的长期导航任务，并在多个大型室内环境中展示了成功的部署。

## Abstract
Long-term robot deployment requires a compact and scalable memory that preserves fine-grained visual semantics, grounds observations in space and time, and enables efficient storage and retrieval. In this paper, we propose RAVEN, an agentic memory system for long-horizon robotic question answering and navigation. RAVEN stores visual embeddings with pose and time in a vector database, and grounds retrieval in a spatial map to answer queries and navigate to goals. By operating directly on visual embeddings, RAVEN avoids lossy image-to-text captioning and enables accurate semantic, spatial, and temporal retrieval at scale. Across several simulated and real-world video question-answering benchmarks, RAVEN consistently surpasses caption-based memory systems and matches frontier VLMs on long-horizon tasks at 10$\times$ lower retrieval cost. Finally, we instantiate RAVEN on a Unitree Go1 robot for the task of long-horizon navigation for natural language goal-reaching, and show successful deployment over several large indoor environments.

### 2. 论文提出的方法论
RAVEN 是一个双阶段记忆系统，其核心思想是构建和查询一个“视觉-时空”（Visuo-Spatio-Temporal）记忆库。

*   **核心思想**：
    *   **避免描述瓶颈**：使用多模态嵌入模型将每一帧图像直接编码为紧凑的视觉嵌入向量，而非文本描述。
    *   **智能体式检索（Agentic Retrieval）**：采用一个 VLM 智能体，通过闭环推理和多轮工具调用，逐步从记忆中检索相关信息，而不是单次查找。
*   **关键技术细节与流程**：
    1.  **记忆构建阶段（Memory Building）**：
        *   机器人在探索环境时，对于每一帧 RGB 观测 $o_i$，使用预训练的**多模态编码器**（如 QQMM-v2, SigLIP）直接编码为**视觉嵌入** $z_i$。
        *   将视觉嵌入 $z_i$ 与对应的**机器人位姿** $p_i = (x_i, y_i, z_i)$ 和**时间戳** $t_i$ 组合成一个记忆三元组 $\{ p_i, t_i, z_i, o_i \}$。
        *   所有三元组被存储在支持高效近邻搜索的**向量数据库**（如 FAISS）中。
    2.  **查询与执行阶段（Querying & Execution）**：
        *   **工具调用智能体**：当接收到用户查询 $q$ 时，一个 VLM 智能体进入闭环推理状态。它拥有四个基础检索工具：
            *   **文本检索**：将智能体生成的查询文本编码，与视觉嵌入进行余弦相似度匹配。
            *   **时间检索**：返回从指定时间戳开始的连续记忆片段。
            *   **位置检索**：找到指定空间坐标的空间最近邻记忆。
            *   **图像检索**（附录中探讨）：支持以图搜图。
        *   **多轮推理**：智能体迭代调用这些工具，获取 Top-K 个最相关的记忆帧（包含图像、位置、时间），并结合相似度分数进行推理，判断信息是否充分。当信心足够时，生成最终答案。
        *   **导航执行**：如果查询是空间目标，RAVEN 会输出预测目标位置 $(\hat{x}, \hat{y}, \hat{z})$，并将其传递给一个 A* 轨迹规划器，导引机器人导航至该点。

*   **匹配甚至超越巨型 VLM**：通过精确检索与推理分离，RAVEN 在长时程任务上的性能可以匹配甚至超越直接使用前沿 VLM（如 Gemini-3-Pro）处理全部帧的方法，而计算成本更低。
*   **真实世界可行性验证**：在宇树 Go1 四足机器人上的成功部署证明，该方法可在现实混乱、大型室内环境中有效工作，完成复杂的问答与导航指令。

### 7. 优点
*   **方法论创新且简洁**：巧妙地利用现成的多模态嵌入模型和向量数据库，构建了一个训练无关、即插即用的记忆系统，核心逻辑清晰。
*   **解决了真实痛点**：直接针对了“描述瓶颈”和“长上下文效率”这两个限制机器人长期自主性的关键问题，论据充分。
*   **全面的综合性评估**：实验覆盖数据集广、模型种类多、查询类型全面，并进行了深入的定性失败案例分析和详尽的消融研究，验证非常有说服力。
*   **实用性强**：不仅给出了 SOTA 性能，还深入探讨了不同成本模型（闭源 vs. 开源）的搭配策略，对实际部署有直接指导意义。

### 8. 不足与局限
*   **视觉编码器的上限**：系统性能严格受限于所选多模态嵌入模型的质量和语义对齐能力，对于嵌入模型未能理解的概念或关系，检索会失败。
*   **静态环境假设**：当前实现将记忆视为一个静态的过去观测数据库，主要针对事物位置固定的环境。处理物体频繁移动或改变状态的**高动态场景**是一个尚未解决的核心挑战。
*   **记忆更新机制缺失**：系统没有主动更新或修剪过时记忆条目的机制，这在一段时间后环境发生变化时可能会导致错误。
