## Memory Makes the Difference: Evaluating How Different Memory Roles Shape Conversational Agents

Abstract 

Prior research on memory mechanism in RAG-based conversational system has emphasized how memory is stored and retrieved. How-ever, far less is known about how memories with different functional roles influence re-sponse quality. Specifically, how they shape an agent’s responses under varying conversa-tional contexts and whether they lead to sub-stantively different response behaviors. Exist-ing evaluations in conversational system are also largely reference-based, insufficiently cap-turing the nuances in responses that may ad-dress users’ preferences differently. In this work, we probe the impact of different mem-ory types in shaping agents’ responses. We present a fine-grained taxonomy of conversa-tional memory, classify retrieved memories into different role types, and design a user-centric evaluation framework that simulates user per-spectives. Through comparative experiments on long-term datasets and frontier LLMs, our analysis reveal many differentiated effects of memories: e.g. , clarifying memory improves re-sponses’ factual accuracy and constraint aware-ness, making them more correct and person-alized; irrelevant memory reduces topic rele-vance and degrades constraint awareness. De-spite the power of frontier LLMs, these find-ings shed light on how different memory types can be leveraged to produce more personal-ized responses and inspire further research in this direction. Our code is available at: https: //github.com/audreycs/MemoryEffect .

The effect of retrieval part of-ten behaves like a black box and hard to interpret or control (Chen et al., 2024). As Fig. 1 demon-strates, some supplementary memory can enrich the response and make it more helpful; while other memories can be distracting and make the response inappropriate and undesired. In this work, our cen-tral hypothesis is that the type of memory mat-ters : different memory types elicit different agent responses, and these variations can lead to different user perceptions of quality. Therefore, understand-ing the qualitative effect of each memory type and whether agents can leverage a bundle of retrieved memories worth systematic investigation. Building on this hypothesis, we aim to quantify the impact and find their qualitative property. We introduce the first taxonomy of conversational memory types, determined by their roles when the agent reasons on memory. Besides, existing work is usually lim-ited in reference-based evaluation. Prior work leans on question answering (QA) datasets provided with unambiguous gold answers (Anantha et al., 2021; Xu et al., 2022; Pan et al., 2025; Maharana et al., 2024; Chen et al., 2025). Their reference-based evaluation privileges measuring factual accuracy (e.g. , F1, ROUGE) while ignore other qualities of responses like user intent awareness and informa-tiveness, which are critical for dialogic intelligence but difficult to measure using only references. In this work we present analyses of how differ-ent memory types shape LLM agents’ responses, paired with a holistic, user-centric evaluation frame-work covering various aspects. Our taxonomy clas-sifies conversational memory pieces into five types — answer, clarifying, enriching, distracting, and irrelevant, judged with awareness of query and rel-evant context instead of merely the gold answer in previous analyses (Cuconasu et al., 2024). We conducted comparative experiments on two long-term, multi-session conversational datasets over three popular LLMs. Specifically, we compare var-ied memory settings with (i) answer memory only and (ii) parametric memory only, to study how the presence of individual types and their combinations affects responses quality. To better differentiate re-sponse quality, we design a user-centric evaluation proxy that conditions on user intent and prior con-versation context, yielding sensitivity to response relevance, informativeness, and other properties. Our experiment results reveal many interesting patterns. For examples, we find frontier LLMs are capable of selectively leveraging contextual infor-mation with considerable nuance; agents are less likely to ground their responses as retrieval size increases. As for memory types, clarifying mem-ory plays a key role in improving answer accuracy and contextual grounding, and even in isolation it can induce perfect responses. In contrast, irrele-vant memory tends to reduce topic relevance and weaken contextual awareness, against the founded positive impact of them on small-scale LLMs like Llama2 and Phi-2 in previous work (Cuconasu et al., 2024). These also confirm our hypothesis on the impact of memory types. Our taxonomy and user-centric evaluation are general-purpose and can be applied to other conversational corpora and set-tings. Our main contributions in this work are the following: • We hypothesize the impact of memory types on agent’s responses, and conducted extensive comparative experiment on frontier LLMs to investigate. • We propose the first fine-grained taxonomy of conversational memory based on functional roles, together with a holistic framework with diverse metrics for generation evaluation. • Our experiments qualitatively reveal the dis-tinct impacts of different memory, with sev-eral effects diverging from prior findings on small-scale LLMs. 

2 Related Work 

2.1 Memory-enabled Conversational System 

Conversational systems typically frame memory recall as a store-retrieve-generate pipeline, reflect-ing flexible memory management and working context (Gao et al., 2023). Recent research ef-forts on LLM-powered conversational systems have largely focused on memory structure (Packer et al., 2023; Rasmussen et al., 2025; Chhikara et al., 2025), retrieval strategy (Tan et al., 2025b; Li et al., 2025; Hu et al., 2025a), and memory update governance (Tan et al., 2025b; Rezazadeh et al., 2025; Wang et al., 2025). Their retrieval often use sparse or conversational dense retriev-ers, such as BM25 (Robertson et al., 2009) and Contriever (Izacard et al., 2021). To stress-test these systems in realistic settings, several work tar-get long-term, multi-session dialogue, including MT-Bench-Plus (Lu et al., 2023), LOCOMO (Ma-harana et al., 2024), and LongMemEval (Wu et al., 22025), InfiniteBench (Zhang et al., 2024). 

2.2 Analysis of Retrieval Pieces Type 

Several recent studies have examined how irrele-vant or distracting retrieved pieces affect QA perfor-mance, primarily focusing on smaller scale LLMs. Cuconasu et al. (2024) give pioneer analysis of how the type and position of retrieved documents influence model’s answers. They classified docu-ments into gold , relevant , distracting , and random based on semantic similarity and the presence of gold answer. They highlighted that random doc-uments can sometimes raise accuracy. Jin et al. (2025) investigate distracting documents that are relevant to the query but not the gold answer and find consistent harm to QA performance across diverse models. Later, Amiraz et al. (2025) pro-pose a measure to quantify the distracting effect of distracting documents. Collectively, these work characterize the impact of irrelevant and distracting evidence in document QA, but give less attention to conversational settings. Inspired by their analysis, we adapt and refine their types to conversational scenario and shift the focus from document QA to memory-enabled dialogue, where the responses require more nuanced measurement and multiple forms of “relevant memory” ( e.g. , clarifying, en-riching) also need to be considered. 

2.3 Evaluation of Conversational Agent 

Conventional evaluation of conversational system scores responses against references with n-gram or embedding metrics like F1, ROUGE (Lin, 2004), or BERTScore (Zhang* et al., 2020). Newer work on long-term, memory-enabled conversations also tracks retrieval accuracy by measuring if particular pieces are retrieved ( i.e. , needle in the haystack) via metrics like Recall@ k (Maharana et al., 2024; Wu et al., 2025). As the advance of LLM-as-a-judge, LLM-based judges start to be widely adapted for holistic evaluation of chat quality (Bai et al., 2024; Zhao et al., 2025; Chhikara et al., 2025). By set-ting well-crafted rubrics, LLM judges are flexible to measure diverse dimensions even without ref-erences. Mem0 (Chhikara et al., 2025) prompts LLM judge to compare responses with references, and MT-Bench-101 (Bai et al., 2024) uses pairwise comparison of two candidate responses for ranking. PersonaLens (Zhao et al., 2025) equips judge with user profiles and rich interaction history to further measure if responses address user preferences.

3 Taxonomy of Conversational Memory 

Retrieved memories form a semantic neighbor-hood around the query by RAG mechanism, and the agent’s reasoning can be viewed as selectively traversing retrieved memory pieces to assemble the information needed for a response, as illustrated in Fig. 2. Based on this intuition and existing doc-ument RAG classification (Cuconasu et al., 2024; Amiraz et al., 2025), we classify conversational memory according to the functional roles it plays during the agent’s reasoning process into five fine-grained types: answer , clarifying , enriching , dis-tracting , and irrelevant . Tab. 1 shows their defini-tions. As shown in Fig. 2, to correctly answer the query, the agent needs clarifying memory to iden-tify the referenced “manager”, then infer answer from the answer memory. Ideally, the agent needs enriching memory to add more helpful details, and avoids distracting memory to mitigate false answer. The remaining off-topic memory is considered as irrelevant , which should be neutral noise. Each memory is assigned a single type per query. We ac-knowledge that, in practice, a memory may plausi-bly contain pieces serving multiple roles. To enable controlled analysis and clear attribution of effects, we adopt single-label assignment as a deliberate methodological simplification. To resolve over-laps consistently, we apply a labeling precedence: answer → clarifying → enriching → distracting 

→ irrelevant for disambiguation. A practical moti-vation for the precedence is to encourage sufficient representation of relatively sparse types in realis-tic retrieval pools. Importantly, it does not encode a hierarchy of importance or quality of them on response generation. Besides conversational mem-ory, we also include parametric memory which is LLM’s inherent knowledge learned during its training. 4 User-centric Response Evaluation 

To better capture the sensitivity of responses to different context, we design a holistic evaluation framework which is user-centric and context-aware (Tab. 2). This framework spans three major aspects (Accuracy, Relevance, and Informativeness) and each aspect contains finer metrics. These aspects are designed by integrating established evaluation framework (Liang et al., 2023; Maurya et al., 2025) and adapted to conversational scenario. At a high level, Accuracy captures factual correctness of re-sponses; Relevance checks alignment with user in-tent and preferences; Informativeness assesses use-fulness beyond the minimal answer. Note that con-ventional reference-based accuracy metric such as ROUGE is subsumed by Answer Accuracy, which tests whether the response contains the desired an-swer. We additionally include Memory Coverage and ROUGE-1 Recall in the framework. Tab. 2 also indicates which metrics require prior conversation history (“context-aware” column). For instance, World Knowledge Accuracy and Internal Consis-tency can be judged without context. For robustness, most metrics except Complete-ness are designed with a binary scale rather than a fine-grained rating scale. There is work showing binary judgments yield higher accuracy and relia-bility for LLM judges (Dong et al., 2024; Alhawasi and Youssef, 2024), and finer-grade rating can in-troduce subjectivity in scale design (Bansal et al., 2024; Wang et al., 2024). Thus, we believe binary scale is sufficient for analyses in this work. We use 

GPT-o3 as the LLM judge, and metrics under the same aspect are judged in group. 

5 RAG Pipeline Implementation 

Our implementation of conversational RAG fol-lows the standard RAG pipeline. Some key terms in our context: 

Conversation The complete set of interactions be-tween one user and the chat agent. Each user has a separate conversation, which may span multiple sessions over time. 

Session A continuous interaction episode between a user and the agent. A session typically focuses on a single topic. 

Utterance and Turn One utterance is a single text input from either the user or the agent. A turn is a back-and-forth exchange consisting of one user utterance followed by one agent utterance. 

Context For each user query, the context is the relevant prior memory which is reachable to the agent. We refer retrieved memories as context.

5.2 Memory Storage and Retrievers 

We use one memory bank for all sessions under the same conversation. When the agent responds to a query, all past sessions are reachable, but different conversations cannot access each other’s memory. Following LongMemEval, we implement the mem-ory bank as key-value pairs {<key>: <value>}, where the key is used for retrieval indexing and the value returns the original content. We store each turn as one pair; both the key and value are the orig-inal turn in our implementation, although the key can be customized ( e.g. , a summary or a list of key-words). When timestamps are available, we also record them in the values. For retrieval, we experi-ment with three methods: BM25, Contriever, and OpenAI’s text-embedding-3-small (Neelakan-tan et al., 2022). We retrieve top 100 memory pieces with highest similarity. On average, a mem-ory piece contains 317 tokens in LongMemEval-m and 216 tokens in Long-MT-Bench+, so 100 pieces already constitute a very large context. 

5.3 Agent Models and Response Generation 

We especially evaluate three large scale and pop-ular LLMs as agents: GPT-4o-2024-08-06 (Hurst et al., 2024) (we call GPT-4o for short in later parts), Gemini-2.0-Flash (DeepMind, 2024), and Llama-3.3-70B (Dubey et al., 2024). We select the top k pieces ( k ∈ { 0, 5, 10 , 20 , 50 , 100 })from the 100 retrieved pieces to vary context size. When timestamps are available, the k pieces are ordered chronologically. We concatenate them and truncate as needed to fit the model’s context window. The full prompt for response generation shown in Fig. 7 in Appendix. Note that when 

k = 0 , the agent relies on its parametric memory to respond. All agents are set with the minimum temperature to ensure near-deterministic output. 

5.4 Response Evaluation 

We use GPT-o3 with high reasoning effort as the judge. Different rubrics are applied to metrics across different aspects. We carefully designed the rubrics and performed sanity check on a sample of queries to validate the quality of the judgments, iteratively refining the rubrics to align the judge’s scores with human evaluations (Appendix C). Par-ticularly, for the Topic Relevance metric, refusal responses such as “ I am unable to answer ... ” re-ceive a score of 0.

6.2 Utilization of Different Memory Types 

We also measure how often each memory type is covered in the response as the context size k varies. Based on our taxonomy of memory type, ideally the irrelevant memory should be ignored and an-swer memory should be always included. If this is supported from experiment, it also serves a sanity check for our taxonomy design. To measure this, for a given type, let m be the number of its pieces among the top k retrieved (the seen number) and n

be the number echoed in the response (the utilized 

number), then the coverage rate is calculated as 

n/m — percentage of the seen pieces being used. Fig. 5 shows the results on GPT-4o . We can see that answer memory is covered most frequently, while irrelevant memory is covered the least, suggest-ing that the agent is effective in distinguish useful from unuseful memory. Clarifying memory is also highly utilized, suggesting validity of our memory type taxonomy. As k increases, coverage for all types declines slightly. 

7 Impact of Different Memory Types 

Now we take a closer look at how distinct memory types shape response quality. We conduct con-trolled experiments aiming to characterize system-atic associations between memory-type presence and response behavior under realistic retrieval con-ditions. Specifically, we run two sets of compar-ison experiments: (i) variant settings that add a second memory type on top of an answer-only base-line ( vs. A); and (ii) variant setting compared to a parametric-only baseline ( vs. P). The answer-only baseline reflects a common assumption in RAG: if answer-containing pieces are retrieved in isolation, the model should produce the correct answer eas-ily. In practice, retrieval also contains substantial non-answer memory, so we aim to know how other memories (C/E/D/I) alter responses when present. The parametric-only baseline holds the converse as-sumption: without memory retrieval, the agent fails to give a good answer to user’s queries. We choose the top 15 retrieved memory pieces for study (de-termining their types and varying the memory set-ting) and test text-embedding-3-small as the re-triever. Other experiment setups are the same as above. 

7.1 Comparison with Answer Memory Only 

We compare the vs. A baseline to five variants: A+C, A+E, A+D, A+I, and a non-answer bundle C+E+D+I. This yields six settings and five pair-wise comparisons. As an aside, memory types are judged with query and context, so both the com-position and the counts within the top 15 retrieved pieces can vary. A retrieval may include only one clarifying (C) piece but more than ten irrelevant (I) pieces. Therefore, in comparison settings we retain the original retrieved counts of each memory to reflect the reality. Queries where the retrieved pieces lack answer memory (A) are excluded from this analysis as they do not admit a comparable answer baseline. The idea to quantify each variant setting’s im-pact is checking how its metrics scores change com-pared to baseline setting. Formally, let Q denote the query set for a dataset. For a metric m and a memory setting i, let smq (i) be the score of the agent’s response to query q ∈ Q . We define the per-query score difference between two settings i

and j as diff mq (i, j ):

diff mq (i, j ) = smq (i) − smq (j).

For instance, diff mq (A + C, A) > 0 means that adding clarifying memory improves the score on metric m on query q compared to answer-only base-line. We measure how often setting i outperforms setting j on metric m using an indicator Improve-ment Rate (IR ): 

IR m(i, j ) = #{diff mq (i, j ) > 0}

#{diff mq (i, j )̸ = 0 } ∈ [0 , 1] .

means that, switching from setting j to setting i,70% of the score changes are towards better. We report IR for each metric. The top table in Tab. 4 shows results on GPT-4o . 7.2 Comparison with Parametric Memory Only 

In the second comparison group comparing with parametric-only baseline, we evaluate six settings: C, A+C, E, D, I, and all types bundle. The IR 

results for GPT-4o are in the bottom table of Tab. 4, with statistical analysis in Fig. 15. Below is our key findings: 

Relevant Memory Helps Accuracy and Rele-vance The bottom table in Tab. 4 and Fig. 15 show that adding semantically relevant memory (answer, clarifying, enriching, or distracting) generally im-proves Answer Accuracy, Topic Relevance, and Constraint Relevance over parametric-only base-line. This suggests the necessity of past history when agents generating responses. Otherwise, the responses are seriously off-topic and inaccurate. 

Clarifying Memory Remains Critical Among the settings, the A+C and all type bundle settings yield the largest average gains, highlighting the cen-tral role of answer-containing evidence and disam-biguation information; for GPT-4o , these settings also correspond to higher Actionability. Adding clarifying memory (C) alone also brings significant improvement. 

Irrelevant Memory Offers Little Improvement 

Irrelevant memory pieces provide no clear improve-ment over parametric-only, yet they are still fre-quently echoed in responses from Memory Con-verge — a sign that the agent tends to echo ir-relevant retrieved text without making the answer better. 

RAG Can Reduce Fidelity and Coherence 

Across models, we observe adding retrieved mem-ory often lowers World Knowledge Accuracy and Internal Coherence relative to using parametric memory alone. Manual analysis suggests this ef-fect arises because additional context encourages more specific claims, which the LLM judge per-ceives increases the risk of factual errors, whereas parametric-only responses are shorter, less specific, and safer. 

7.3 Non-significant Results 

We also observe settings where the impact of spe-cific memories is statistically non-significant across agent models and datasets. For example, whether adding distracting memory reduces Internal Coher-ence relative to the answer-only baseline, and how adding enriching memory affect the Topic Rele-vance. These effects are small and variable from the results, and stronger evidence would be needed to claim a reliable effect. 98 Conclusion And Future Directions 

In this work, we examine how memory types af-fect LLMs in conversations using a novel memory taxonomy, a user-centric evaluation, and controlled comparisons. The experiments over agent models, context size, and retrievers confirm known patterns and provide new findings in long-term conversa-tional RAG scenario. In comparison experiments, we find conversational RAG performance is driven not merely by retrieving relevant context, but by retrieving the right functional types of memory. These findings point toward memory-type-aware retrieval and selection as a promising direction for IR. First, retrieval systems can benefit from be-ing memory-role aware, enabling category-aware curation, ranking, and diversification rather than treating all retrieved context uniformly. Second, our results motivate the development of collections that evaluate the different roles of grounding in RAG for memory and other corpora. Finally, it is important to develop and validate taxonomies as analytical tools: the most useful taxonomy is not necessarily the most granular, but the one that best helps us understand retrieval behavior and optimize end-to-end outcomes. 

Limitations 

While our work provides many new findings, it also has several limitations that should be acknowl-edged. 

LLM-as-a-Judge Evaluation Our experiments heavily reply on LLM judge for response scoring and memory type identification. Though we chose advanced models GPT-5-thinking and GPT-o3 as the judges and carefully designed the rubric, LLM judge is known to occasionally produce inconsis-tent or imperfect assessments. This limitation may introduce uncertainty into the reported results and our analysis. 

Computational Cost Our experiments involve large-scale experiments with proprietary LLMs such as GPT models and Gemini. Reproducing these results requires substantial API usage, which may be costly and limit full replication by others. 

Dataset Size and Noise The two datasets we used LongMemEval-m and Long-MT-Bench+ contain a few hundred test queries (see their statistics in Table 3). While labeling errors are expected to be rare, any noise in such a small test size can still introduce minor changes in the reported results. Moreover, some of our results are based on very few data points, which limits their statistical signif-icance and should be interpreted with caution.

C Evaluation with LLM-as-a-Judge 

We carefully designed and iteratively refined the prompts used for LLM judges to ensure reliable and consistent evaluation. The design process followed an iterative “prompt–test–refine” loop. Starting from a minimal baseline, we tested responses to identify issues such as instruction drift or unsta-ble reasoning. We then applied human-in-the-loop refinement to adjust phrasing and structure, validat-ing each revision on representative cases until the prompts produce desired outputs on a set of human-validated test samples. We provide judgment cases for memory type classification and response evalu-ation in Table 10, 11, and 12. We use GPT-5-thinking as the judge for deter-mining memory types, setting its reasoning effort to medium , and max completion tokens to 8, 000 .Figure 8 is the full prompt for it. For the response evaluation, we use GPT-o3 , setting its reasoning ef-fort to high , and max completion tokens to 10 , 000 .Figure 9 is the prompt for evaluating Accuracy; Figure 10 is for evaluating Relevance; Figure 11 is for evaluating Memory Coverage; Figure 12 is for evaluating Informativeness.
