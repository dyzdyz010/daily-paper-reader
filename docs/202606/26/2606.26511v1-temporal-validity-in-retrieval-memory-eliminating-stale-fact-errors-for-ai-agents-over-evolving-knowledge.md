## 摘要
检索增强生成（RAG）使智能体能够访问积累的知识，但缺乏时间模型。当一个事实发生变化（例如，函数重命名或API重构），RAG会检索出过时值和当前值，二者的嵌入相似度几乎相同。智能体随后要么回避，要么提供已被取代的事实。我们证明这是一个结构性问题：在一个校准数据集上，余弦相似度区分矛盾事实与重复事实的AUROC仅为0.59（接近随机），因为矛盾事实通常比改写后的重复事实在嵌入上更接近原始事实。我们提出MemStrata，一种保持时间有效性的检索记忆。它像RAG一样存储事实，保留静态回忆能力，但当某个事实的值出现矛盾时，一个确定性的（主体、关系、客体）取代规则会在双时序账本中淘汰过时值——无需相似度阈值，也不调用LLM。在使用7B模型本地运行的六个基准测试中，MemStrata在静态知识上与RAG持平，在演化知识上达到0.95-1.00的准确率（而RAG为0.20-0.47）。核心结果是过时事实错误率：当必须回答时，RAG在15-40%的情况下提供已被取代的值；MemStrata将此率降至约0%，这是RAG无法避免的一类失败。MemStrata在检索延迟（约2.1秒）下实现这一性能，而LLM重排序基线的延迟约为16-18秒。我们发布了测试框架、数据集以及一种无需标记的知识演化下记忆评估协议。

## Abstract
Retrieval-augmented generation (RAG) gives agents access to accumulated knowledge, but has no model of time. When a fact changes (e.g., a function is renamed or API restructured), RAG retrieves both the stale and current value with near-identical embedding similarity. The agent then either abstains or serves the superseded fact. We show this is a structural problem: on a calibrated dataset, cosine similarity distinguishes a contradicted fact from a duplicated one with AUROC 0.59 (near chance), as contradictions are often more embedding-similar to the original than rephrased duplicates.   We present MemStrata, a retrieval memory maintaining temporal validity. It stores facts like RAG, preserving static recall, but when a fact's value is contradicted, a deterministic (subject, relation, object) supersession rule retires the stale value in a bi-temporal ledger - with no similarity threshold and no LLM call. Across six benchmarks run locally with a 7B model, MemStrata ties RAG on static knowledge and reaches 0.95-1.00 accuracy on evolving knowledge (where RAG reaches 0.20-0.47). The central result is the stale-fact-error rate: when required to answer, RAG serves superseded values 15-40% of the time; MemStrata drives this to ~0%, a failure class RAG cannot avoid. MemStrata achieves this at retrieval latency (~2.1s) versus ~16-18s for LLM-reranking baselines. We release the harness, datasets, and a marker-free evaluation protocol for memory under knowledge evolution.

### 1. 论文的核心问题与整体含义

*   **研究动机与背景**：大型语言模型驱动的AI代理正被部署为长期交互的协作者（如编码助手、研究助手），其核心约束已变为**记忆**——即代理如何编码、保留、检索并保持所学知识的“当前性”。当前主流的记忆机制是检索增强生成（RAG），它通过存储交互历史的嵌入块，在查询时检索最相似的 top-k 块。
*   **核心问题**：RAG 在**演化知识**（如函数重命名、配置值变更、API 重构）上存在结构性盲点——**没有时间模型**。当某个事实发生改变时，新旧版本在嵌入空间中几乎相同（例如“端口为 8000”与“端口为 8080”只差一个 token），RAG 会同时检索出二者，且无法判别哪一个是当前的，导致代理要么回避，要么自信地提供过时值。
*   **整体含义**：本文揭示这是一个无法通过调优相似度阈值解决的**结构性问题**（余弦相似度无法可靠区分矛盾事实与重复事实，AUROC 仅为 0.59），必须采用**结构性的时间有效性机制**来消除过时事实错误。

### 2. 论文提出的方法论

*   **核心思想**：MemStrata 是一种**保持时间有效性的检索记忆层**。它像 RAG 一样存储事实以保持静态召回率，但引入**确定性取代规则**：当一条新事实与已存储事实共享（主语, 关系）键，但断言不同（宾语）时，新事实将自动取代旧事实，旧事实被退休（而非删除）。
*   **关键技术细节**：
    *   **双时序账本**：事实被标记上 `valid_from`（有效起始时间）、`valid_to`（有效结束时间）和 `superseded_by`（被哪条事实取代），退休事实依然保留，支持未来的“按时间点查询”。
    *   **写入路径**：
        1.  **精确去重**：通过归一化文本哈希丢弃逐字重复。
        2.  **确定性断言路径**：当输入可被提取为干净的（主语, 关系, 宾语）三元组且宾语为单一可变值时，MemStrata 将（主语, 关系）标准化为键，检查是否已有活跃断言。若键相同但宾语不同→新断言取代旧断言（旧行 `valid_to` 被关闭）；若键和宾语均相同→视为重复（强化）；若键不存在→视为新事实。**此过程不依赖余弦相似度，也不调用 LLM**。
        3.  **文本门回退**：对于非三元组形式的文本，回退到基于相似度和 LLM 的判断门，但只丢弃精确重复，保留非矛盾的近似重复。
    *   **读取路径**：嵌入查询，通过余弦相似度检索 top-k **活跃**（未被退休）事实，用确定性陈旧过滤器剔除已退休行，最后打包**原始来源语句**（而非精简的三元组）以保持语境丰富度。读取路径上**不运行任何 LLM**，延迟仅取决于嵌入检索。
*   **“保留然后取代”设计**：早期的激进压缩变体（合并近似重复）会损害静态召回率，因此最终方案选择像 RAG 一样**保留不矛盾的独特事实**，仅在出现矛盾时才进行取代，从而在静态与演化知识上取得平衡。

*   **静态知识上持平**：`temporal_v6` 在静态基准上与 RAG 准确率相当（如 `domain` 上 0.82 vs. 0.86），未因引入时间机制损失静态召回率。

### 8. 不足与局限

*   **结构化基准的局限性**：当前演化基准均为基于**结构化、单值模板**构建的，三元组提取器可以高可靠（~97%）地工作。然而，对于更复杂的、包含多值语句的自然语言矛盾，提取可靠性大幅下降（约44%），**提取层的质量是应用在非结构化自然语言矛盾上的核心瓶颈**，而非取代机制本身。
*   **时间信号简化**：基准使用**摄入顺序**（状态 A → 状态 B）作为时间信号。真实世界应用中往往需要显式的时间戳，论文提及双时序账本已预留 `valid_from` 字段，但“基于真实时间戳的查询”功能尚未实现。
*   **评判噪声**：小型的 3B 评判模型偶尔会出现评分噪声，但核心的 0% 过时错误率不受影响。
*   **规模与模型**：所有结果均在单个 7B 模型上获得，未在不同规模或专有模型上进行验证。基准的规模也较小（数十个项），旨在隔离机制而非进行大规模排行榜竞赛，其在真实世界长期数据上的表现有待进一步验证。
*   **应用范围**：本文以代码演化为切入点，虽然架构是一个通用时间上下文记忆，但扩展到带时间戳的世界知识是其自然的下一步，尚未在非代码领域进行充分评估。
