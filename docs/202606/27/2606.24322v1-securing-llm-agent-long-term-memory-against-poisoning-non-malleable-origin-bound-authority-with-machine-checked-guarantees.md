## 摘要
LLM智能体越来越依赖持久长期记忆，这产生了一个我们在此研究的关键漏洞：记忆投毒。攻击者可以在一个会话中存储不可信内容，进而在后续会话中引导重要行为，例如支付、设置更改或数据窃取。现有防御基于记忆项的权限，或是依据其内容（检测或信任评分），或是依据其派生历史（谱系）。我们表明这两种信号都是可篡改的。攻击者可以通过LLM智能体特有的三种渠道清洗不可信来源：智能体自身的摘要化、可信工具回声以及制造的佐证。每种方式都能使内容看似无害，并切断或翻转其派生边至“可信”。我们形式化了记忆写入-检索-操作管线的可篡改性，并证明了一个机器验证的分离定理：在清洗条件下，任何基于内容或谱系的防御都不健全（T1）；写入时源绑定是必要的（T2）；而具有抗女巫攻击佐证门控提升的不可篡改源绑定权限是充分的（T3）。我们的构造TMA-NM（防篡改记忆权限，不可篡改）为LLM智能体记忆实例化了不可篡改的信息流控制（IFC）。在八个前沿模型上的跨防御、跨攻击和跨模型基准测试显示，现有防御恰好会在理论预测的地方失败（清洗攻击成功率高达68%），而TMA-NM在所有模型和渠道上，对直接攻击和清洗攻击均达到0%的攻击成功率，同时保持完全的合法效用。我们公开了基准测试、测试工具及机器验证的TLA+模型以支持可复现性。

## Abstract
LLM agents increasingly rely on persistent long-term memory, which creates a critical vulnerability that we study here: memory poisoning. An adversary can store untrusted content in one session that later steers a consequential action, such as a payment, a setting change, or data exfiltration, in a future session. Existing defenses base a memory item's authority to act on either its content (detection or trust-scoring) or its derivation history (lineage). We show that both signals are malleable. An attacker can launder an untrusted origin through three channels specific to LLM agents: the agent's own summarization, a trusted-tool echo, and manufactured corroboration. Each makes the content look benign and breaks or flips its derivation edge to ``trusted.'' We formalize malleability for the memory write-retrieve-act pipeline and prove a machine-checked separation theorem. No content- or lineage-based defense is sound under laundering (T1), write-time origin binding is necessary (T2), and non-malleable origin-bound authority with Sybil-resistant corroboration-gated elevation is sufficient (T3). Our construction, TMA-NM (Tamper-evident Memory Authority, Non-Malleable), instantiates non-malleable information-flow control (IFC) for LLM-agent memory. A cross-defense, cross-attack, and cross-model benchmark over eight frontier models shows that existing defenses fail exactly where the theory predicts (up to 68% laundering attack-success), while TMA-NM reaches 0% attack success on both direct and laundering attacks across all models and channels, at full legitimate utility. We release the benchmark, harness, and machine-checked TLA+ models to support reproducibility.

### 2. 方法论：核心思想、关键技术细节、算法流程

论文提出并实现了一种名为 **TMA-NM (Tamper-evident Memory Authority, Non-Malleable)** 的防御构造，其核心是实例化一种不可篡改的信息流控制。

- **核心思想:**
    - **源绑定权限：** 一个记忆项的“行动权限”在写入时就被永久、不可篡改地绑定到其`origin`（来源）标签上。可信监控器通过认证通道而非分析内容来赋予此标签。
    - **非可塑性：** 攻击者的任何“洗白”转换都不能提升一个不可信记忆项的权限。
    - **抗女巫攻击的佐证门控提升：** 来自不可信来源的记忆项若要执行关键行动，必须由至少两个**独立**的可信主体（如内部数据库和银行API）进行佐证，或获得一次性的、与动作绑定的用户授权。内容自身的宣称无效。

- **关键技术细节与机制:**

    1.  **写入时源绑定 (M1):**
        - 监控器根据信息到达的通道（用户提示、认证工具、智能体自我写入、不可信外部摄入）分配一个不可伪造的`origin`标签，并据此设定`act_class`（例如，`untrusted` → `none`）。攻击者无法伪造此绑定。

    2.  **不可篡改传播 (M2):**
        - **权威性跟随来源标记传播，而非内容或派生边。** 这直接封闭了两种洗白渠道：
            - **自我摘要化 (L-a):** 智能体从不可信来源生成的摘要，其`act_class`会继承来源的`untrusted` → `none`，无论内容看起来多无害。
            - **可信工具回声 (L-b):** 如果一个受信工具被输入了不可信内容，其输出结果也会被“污染”标记为`untrusted`，从而无法洗白攻击者控制的内容。

    3.  **抗女巫攻击的佐证门控提升 (M3):**
        - 当且仅当一个由不可信来源驱动的关键动作，其安全相关值被 **≥2个独立的可信主体** 佐证，或通过一次性、动作绑定的用户授权时，该动作才被允许。这封闭了制造佐证 (L-c) 的洗白渠道，因为多个不可信来源的内容互证无效。

    4.  **防篡改裁决日志 (M4):**
        - 所有写入、权限提升和允许/拒绝的裁定都追加到一个哈希链上，提供可审计性。

- **算法流程 (以Act-time Authorization为例):**
    1.  **识别值来源:** 确定拟议关键动作`a`的安全相关值`v`。
    2.  **检查来源:** 找出推动`v`的已检索不可信记忆项集合`U`。
    3.  **快速放行:** 若`U`为空，则允许动作。
    4.  **佐证/授权检查:** 若`U`不为空，则检查：
        - 是否有≥2个**独立的受信主体**授权了`v`？若有，则提升`U`中各项的权限并允许动作。
        - 是否存在一个有效的、与**当前具体动作绑定的**且**未使用过的**用户授权令牌？若有，则消耗令牌并允许动作。
    5.  **拒绝:** 否则，拒绝该动作。

- **形式化模型与机器验证的分离定理:**
    - 论文使用TLA+形式化建模了记忆管线，并机器验证了三个定理，为整个方法提供了坚实的逻辑基础：
        - **T1 (不充分性):** 任何基于可塑信号（内容或血统）的防御在洗白攻击下都是不健全的。
        - **T2 (必要性):** 在模型内部，写入时源绑定是必要的。
        - **T3 (充分性):** 不可篡改的源绑定权限+抗女巫佐证提升是充分且安全的。
