## 摘要
对于多步交互环境中的大规模语言模型智能体，一个关键挑战是如何有效利用积累的交互经验。现有工作通常将此类经验的使用分为两种方式：将其作为自然语言规则保留在模型之外以供后续提示，或利用轨迹和反馈来更新模型参数。前者易于解释，但可能与不断演进的策略脱节；后者在更广泛层面上改进策略，但在稀疏奖励环境中对局部错误的纠正有限。我们提出大规模语言模型智能体的经验规则与策略联合学习方法（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时刻，JERP检索与任务相关的规则，并将其与交互历史一起作为智能体的条件。每个回合结束后，它利用收集的轨迹一方面优化策略，另一方面通过将当前执行轨迹与参考成功轨迹进行比较来修订规则池。这种耦合使规则池与不断演进的策略保持一致，同时让稳定有效的行为逐渐被模型自身吸收。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务的决策性能上取得了持续的提升。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

## 1. 核心问题与整体含义

大规模语言模型（LLM）智能体在多步交互式任务中，如何有效积累并持续复用交互经验是一个关键挑战。现有工作通常将经验利用割裂为两种模式：

- **外部规则提示**：将成功或失败的经验总结为自然语言规则，存于模型外部，决策时检索注入提示。这种方式可解释性强，但随着模型策略在训练中更新，固化的规则可能变得陈旧甚至误导，难以与动态演进的策略保持同步。
- **参数更新**：直接利用轨迹与环境反馈对模型进行强化学习微调，将经验内化到参数中。这种方式能全局优化策略，但在奖励稀疏的长程任务中，对局部反复出现的错误无法提供及时、有针对性的纠正。

这种割裂会限制智能体从经验中持续进步的能力。因此，论文提出**经验规则与策略的联合学习**，试图在同一训练循环中，让规则池随策略同步演化，既保留可解释的规则记忆，又通过参数更新不断吸收成熟的行为模式，从而提升复杂交互任务中的决策性能。

## 2. 方法论

### 2.1 整体框架
JERP 为每个任务实例 \(d\) 维护一个长期经验规则池 \(K(d)\)，每个规则包含文本内容 \(z_i\) 和效用分数 \(s_i\)。训练时，首先从规则池中按分数降序选取前 \(k\) 条规则作为工作规则集 \(\tilde K(d)\)，固定用于当前回合。策略 \(\pi_\theta\) 的输入为任务描述、交互历史和工作规则集，智能体据此与环境交互，生成一组轨迹 \(T_d\)。之后，**同一组轨迹被同时用于两个目的**：
- **策略参数更新**：采用分组相对优势的 GRPO 目标进行梯度更新。
- **规则池更新**：通过对比当前轨迹与参考成功轨迹，调用冻结的 LLM 生成结构化编辑操作，动态修改规则池。

如此，规则池的维护和策略学习耦合在同一个数据流中，保证规则始终与当前策略对齐，并逐步将稳定有效行为内化到模型参数。

### 2.2 策略参数更新（阶段一）
对于同一任务实例，采样 \(N\) 条轨迹构成轨迹组 \(\{\tau_i\}\)。每条轨迹的回报 \(R(\tau_i)\) 取环境在终止时的稀疏奖励。在该组内对回报进行标准化，得到每条轨迹的组相对优势 \(A_i\)。所有时间步共享同一条轨迹的优势 \(A_{i,t} \equiv A_i\)。优化目标为 GRPO 损失：

\[
\mathcal{L}_{\text{GRPO}}(\theta) = -\frac{1}{N}\sum_{i=1}^{N}\frac{1}{|\tau_i|}\sum_{t=0}^{|\tau_i|-1} \min\left(\rho_{i,t} A_{i,t}, \operatorname{clip}(\rho_{i,t},1-\epsilon,1+\epsilon) A_{i,t}\right) + \beta D_{\text{KL}}(\pi_\theta\|\pi_{\text{ref}})
\]

其中 \(\rho_{i,t} = \frac{\pi_\theta(a_{i,t}|x_{i,t})}{\pi_{\theta_{\text{old}}}(a_{i,t}|x_{i,t})}\)。

### 2.3 经验规则更新（阶段二）
规则更新采用基于对比反思的结构化编辑方法。给定当前规则池 \(K(d)\)、本次采样的轨迹组 \(T_d\) 和该任务已有的参考成功轨迹集 \(T^+(d)\)，调用参数冻结的 LLM 输出一组规则级编辑操作 \(O(d)\)，包括五种类型：
- **ADD(z)**：新增一条规则文本 \(z\)。
- **EDIT(q, z)**：修改第 \(q\) 条规则为 \(z\)。
- **UPVOTE(q) / DOWNVOTE(q)**：增加/减少第 \(q\) 条规则的效用分数。
- **MERGE(Q, z)**：将集合 \(Q\) 中的多条规则合并为一条新规则 \(z\)。

应用编辑操作后，按固定步长更新分数（新增规则赋予初始分，投票增减固定值，合并规则取原规则中最高分），并移除分数低于阈值的规则。通过这种方式，规则池既能吸纳新经验，也能精简冗余、淘汰不再有用的规则。

### 2.4 训练流程
完整训练过程详见论文 Algorithm 1，核心步骤为：
1. 对每个任务实例初始化空规则池和空成功轨迹集。
2. 每轮训练抽取任务实例，选取 top-k 规则，采样 \(N\) 条轨迹构成轨迹组。
3. 依据轨迹回报计算组相对优势，使用 GRPO 更新策略参数。
4. 调用 ReflectAndEdit 生成编辑操作，更新规则池和规则分数，并将当前成功轨迹添加至参考集。

## 3. 实验设计

### 3.1 测试环境与评价指标
- **AlfWorld**：文本式家庭操作环境，包含六类任务（Pick, Look, Clean, Heat, Cool, Pick2）。评价指标为**任务成功率（%）**。
- **WebShop**：在线购物交互环境，智能体需执行搜索、浏览、比较和购买等操作。评价指标为**平均得分和成功率（%）**。

两个环境均为部分可观测、需要多步交互的复杂场景，奖励仅在回合结束时给出（稀疏奖励）。

### 3.2 对比基线
实验将 JERP 与五类方法比较：
- **Vanilla LLM**：直接提示基础模型，无推理框架、跨回合记忆或参数更新。
- **ReAct**：当前回合内交替推理与行动，无跨回合经验复用。
- **Reflexion**：将先验尝试的自然语言反思存入记忆，在下一次尝试时注入提示，但记忆不参与参数更新。
- **RLOO**：基于 REINFORCE 风格的目标，用组内留一法计算基线进行策略优化。
- **GRPO**：通过组内轨迹回报标准化计算优势，适用于稀疏奖励的任务，避免单独训练价值模型。

所有需要训练的模型（RLOO、GRPO、JERP）均使用 LoRA 进行参数高效微调，基础模型与实现细节（学习率、组采样数、训练轮次等）保持一致，确保对比公平。

## 6. 主要结论与发现

1. **整体性能领先**：JERP 在 AlfWorld 整体成功率（61.5%）和 WebShop 得分/成功率（79.0 / 64.1%）上均优于所有基线，尤其在任务链路长、中间约束多的任务类型（如 Clean、Heat、Cool、Pick2）上提升更显著。
2. **联合学习的价值**：相比纯参数优化的 GRPO 和纯提示记忆的 Reflexion，JERP 将两者结合，既能通过规则快速纠正局部错误，又能将成熟行为逐渐内化到参数中，取得一致的性能增益。
3. **持续规则更新的必要性**：消融实验显示，固定规则池会随策略演进逐渐失效，持续通过新轨迹修订规则池是 JERP 增益的重要来源。
4. **训练动态**：训练曲线表明，JERP 与 GRPO 早期性能接近，但中后期 JERP 提升更为明显，反映了规则池在积累有效操作模式后对策略的正向引导。

## 7. 优点

- **创新性的联合学习框架**：首次在同一训练循环中同时利用交互轨迹进行策略优化和规则池维护，让规则和策略协同进化，弥补了现有两类范式各自为政的不足。
- **动态且可解释的规则管理**：通过结构化编辑操作（增、改、评分、合并）实现规则的自动更新和修剪，保留可解释的文本经验，同时保持与策略一致。
- **与主流 RL 框架兼容**：策略更新基于 GRPO，无需额外价值模型，适合稀疏奖励的多步交互任务；规则更新依赖冻结 LLM，不增加可训练参数。
- **充分的实验论证**：在两类代表性交互基准上，与多种强基线和方法变体进行全面对比，消融实验进一步锁定关键机制，结论可靠。

## 8. 不足与局限

- **规则检索较为简单**：当前仅按效用分数全局排序选取 top-k，未对每个特定任务实例做细粒度相关性检索，可能引入不相关的规则，带来噪声。
- **规则更新依赖 LLM 调用**：每个训练步骤都需要额外调用 LLM 进行反思并生成编辑操作，可能带来较高的计算开销和延迟，未评估其成本。
- **实验场景有限**：仅在两个文本交互环境上评估，未涉及更复杂的视觉-语言环境、多智能体协作或开放域任务，方法的泛化性待进一步验证。
- **未深入分析规则质量**：实验关注最终任务性能，缺少对规则内容质量、冗余消除效果和演化的定性定量分析，可解释性优势尚未充分展现。
- **规则池初始化依赖**：完全从空规则池开始训练，早期缺乏有效规则可能影响学习效率，与人类先验知识结合的可能性未被探讨。
