## 摘要
关于智能体记忆的研究进展迅速，但几乎全部集中在文本方面：现有基准很少在交互环境中探究智能体何时真正需要记住所见内容而非所能记录的内容。我们提出了DMV-Bench（代码：https://github.com/yyyujintang/DMV-Bench），第一个用于多模态智能体视觉记忆的交互式基准。DMV-Bench基于一个受控的家居装饰电子商务目录构建，该目录包含1000种产品变体，其中通过文本泄露契约将每项任务的区别性信号仅保留在像素中。在一系列自主购物会话中，每个访问过的产品图像都带有一个独特的、预渲染的附带线索，之后智能体被要求回忆特定的带有线索的产品并导航至其URL。受双编码理论启发，我们提出了DualMem，一种并行维护视觉和言语编码的记忆架构。在DMV-Bench上，DualMem在所有链长度J ∈ {5, 10, 15, 50}下均优于一个字幕基线以及三个最新的多模态智能体记忆系统，无论使用Gemini 2.5 Flash还是Qwen2.5-VL-7B，并且这一优势在控制了记忆库大小和编码位置偏差后依然存在，同时表现出一种非对称的双编码机制，其中视觉端到端地承载线索，而言语通道仅起较小的查询定位作用。

## Abstract
Research on agent memory has matured rapidly, but almost entirely on the text side: few existing benchmarks ask, in an interactive environment, when an agent genuinely needs to remember what it saw rather than what it could write down. We introduce DMV-Bench (Code: https://github.com/yyyujintang/DMV-Bench), the first interactive benchmark for multimodal-agent visual memory. DMV-Bench is built on a controlled home-furnishing e-commerce catalogue of 1,000 product variants in which a text-leakage contract keeps the discriminative signal of each task in the pixels alone. Across a chain of autonomous shopping sessions, every visited product image carries a unique, pre-rendered incidental cue, and the agent is later asked to recall a particular cued product and navigate to its URL. Inspired by dual-coding theory, we propose DualMem, a memory architecture that maintains a visual and a verbal code in parallel. On DMV-Bench, DualMem outperforms a caption baseline and three recent multimodal agent-memory systems at every chain length J in {5, 10, 15, 50} on both Gemini 2.5 Flash and Qwen2.5-VL-7B, with the lead surviving controls for memory-bank size and encoding-position bias, and an asymmetric dual-coding regime in which vision carries the cue end-to-end while the verbal channel plays a smaller query-grounding role.

### **2. 方法论：核心思想与关键技术**

- **核心思想：附带线索注入**
  - **概念**：在`DMV-Bench`的每一次浏览任务中，每个产品图片都被植入一个独特的、预渲染的“附带视觉线索”（如一个“蓝绿色睡眠眼罩”），但该线索**绝不会**出现在任何文本描述（如标题、URL、描述）中。这通过一个严格的 **“L2文本泄露契约”** 来保证。
  - **任务流程**：智能体执行一连串（`J`个）独立的购物会话，期间这些附带线索被“偶然”看到但从未被要求关注。随后，在探针会话中，智能体被要求带回“那个带有蓝绿色睡眠眼罩的产品”。若纯文本记忆无法解决，则必须依赖视觉记忆。

- **关键技术：双编码记忆架构 `DualMem`**
  - **理论依据**：基于Paivio的**双编码理论**，即信息以视觉（图像）和言语（文字）两种平行编码形式存在时，记忆最为稳固。
  - **工作流程**：
    1.  **编码阶段**：对每个观察到的产品页面 `o`，同时生成：
        - **视觉信号 `v_o`**：使用 `SigLIP-2` 编码器处理产品图片，生成视觉嵌入。
        - **言语信号 `t_o`**：使用VLM生成页面描述，再用 `Sentence-BERT` 编码成言语嵌入。
    2.  **检索与融合**：当收到一个查询 `q` (如“找到那个带蓝绿色睡眠眼罩的产品”)，同样用 `SigLIP-2` 和 `Sentence-BERT` 分别编码为 `q_v` 和 `q_t`。对记忆库中的每个条目，计算视觉和言语的相似度得分 `s_v` 和 `s_t`。通过一个可调的**融合权重 `α`** 将两者结合：`s = α * s_v + (1 - α) * s_t`。`α=0.75` 表示视觉在检索中起主导作用。
    3.  **注入阶段**：得分最高的记忆条目（包括原始图片及文字描述）被作为上下文注入回VLM，辅助其做出导航决策。

- **非对称双编码机制**：实验发现，在这种特定任务下，视觉通道端到端地承载着核心线索，而言语通道主要扮演辅助查询定位的角色，而非线索的主要载体。

### **6. 主要结论与发现**

1.  **`DualMem` 在所有任务链长（J=5, 10, 15, 50）和两个VLM后端上均取得了最优的任务成功率（TSR）**。例如，在`Gemini 2.5 Flash`且`J=5`时，`DualMem`的成功率为**82.7%**，远超第二名的`M2A` (65.7%)。
2.  **视觉记忆不可或缺**：`L2`契约成功隔离了文本信号，纯文本记忆系统在该任务上彻底失败，证实了长程视觉记忆在特定场景下的必要性。
3.  **非对称双编码有效运作**：实验揭示了一个清晰的非对称机制，即视觉通道是核心线索的承载者，而言语通道主要起辅助定位作用。融合权重`α=0.75`时性能最佳，印证了视觉的主导地位。
