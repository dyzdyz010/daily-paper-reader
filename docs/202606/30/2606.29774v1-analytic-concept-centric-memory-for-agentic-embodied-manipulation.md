## 摘要
长时域具身操作要求智能体记住持久对象、跟踪变化的场景状态并重用先前的交互知识。然而，现有智能体记忆通常以非结构化的历史记录或基于嵌入的记录存储，导致难以检索与操作相关的对象部件、物理状态、动作效果和可执行技能。我们提出了一种面向智能体具身操作的分析型概念中心记忆框架。我们的记忆围绕结构化分析概念组织经验，其中对象由语义部件、参数模板、立足位姿、可供性和操作状态表示。它进一步将对象和场景记忆与用于动作引发状态变化的转换记忆，以及用于模板立足和策略立足执行的技能记忆相连接。在运行时，智能体执行结构化的由粗到细的检索，以识别相关对象、状态、转换和技能，支持状态一致的推理和技能重用。在记忆依赖操作、铰接对象泛化、真实世界记忆评估和消融实验上表明，我们的方法在任务完成、检索准确性、对象重识别和跨对象技能泛化方面相较非结构化和基于嵌入的记忆基线有显著提升。

## Abstract
Long-horizon embodied manipulation requires agents to remember persistent objects, track changing scene states, and reuse prior interaction knowledge. However, existing agent memories are often stored as unstructured histories or embedding-based records, making it difficult to retrieve manipulation-relevant object parts, physical states, action effects, and executable skills. We propose an analytic concept-centric memory framework for agentic embodied manipulation. Our memory organizes experience around structured analytic concepts, where objects are represented by semantic parts, parametric templates, grounded poses, affordances, and manipulation states. It further connects object and scene memories with transition memory for action-induced state changes and skill memory for template-grounded and policy-grounded execution. At runtime, the agent performs structured coarse-to-fine retrieval to identify relevant objects, states, transitions, and skills, supporting state-consistent reasoning and skill reuse. Experiments on memory-dependent manipulation, articulated-object generalization, real-world memory evaluation, and ablations show that our approach improves task completion, retrieval accuracy, object re-identification, and cross-object skill generalization over unstructured and embedding-based memory baselines.

### 2. 论文提出的方法论

核心思想是**以结构化的“分析概念”为中心组织和存储记忆**，而不是记录原始观察或扁平的体验记录。一个“分析概念”将物体表示为可复用的参数化模板、语义部件、几何属性和实例化参数的组合。

**关键技术细节与流程：**

1.  **分析概念记忆架构 (Analytic Concept Memory, ACM)**：
    *   **稳定先验知识**：
        *   **语义库 (`Msem`)**：存储每个模板类型 `τ` 的自然语言概要，描述其交互可供性和功能用途，作为语言级检索接口。
        *   **模板库 (`Mtemp`)**：存储可复用的参数化分析模板，包括有效参数空间 `Ωτ` 和支持的操作函数 `Fτ`。
    *   **动态经验记忆**：
        *   **物体记忆 (`Mobj`)**：存储环境中已稳立的物体级概念实例，包括持久ID、类别、部件列表（每个部件由模板实例化而来）和状态。
        *   **场景记忆 (`Mscene`)**：以时空图 (`Gt = (Vt, Et)`) 形式记录场景快照，编码物体的空间、包含、支撑等关系。
        *   **转换记忆 (`Mtrans`)**：记录动作引发的状态变化，形式为 `(目标物体ID, 目标部件, 执行的技能/原语, 执行前状态, 执行后状态, 执行结果)`，显式建模动作因果。
        *   **技能记忆 (`Mskill`)**：存储可执行程序，分为“模板立足技能”（通过参数化模板函数实现）和“策略立足技能”（调用微调后的VLA模型等学得的策略）。

2.  **记忆构建与更新闭环**：
    *   **场景/物体概念立足**：利用视觉基础模型（VFMs）作为工具，通过自然语言指导的检测、SAM分割、位姿估计和结构参数估计，将观察到的实体实例化为物体的“分析概念”，并解析其持久身份（是已有物体还是新物体）。
    *   **经验分析**：在一个子任务或回合结束后，分析执行结果，提取状态转换并更新转换记忆，并将成功的执行流程抽象为模板立足或策略立足的技能，存入技能记忆。

3.  **结构化检索与执行闭环**：
    *   **由粗到细的检索**：首先根据当前任务指令和观察，粗筛可能的物体类别、模板和操作上下文，然后通过一致性过滤（类别、部件、可供性）和排序（几何、位姿、场景状态）精确检索需要的物体、场景、转换和技能条目。
    *   **状态一致性推理**：在执行前，利用转换记忆预判候选技能的预期效果，并检查其前置条件与当前场景状态是否一致，避免冗余或不合理的动作。
    *   **执行与反馈**：选定的技能生成低层机器人动作序列并执行。根据执行结果验证是否达到预期效果，并动态更新物体、场景、转换和技能记忆，形成终身学习的闭环。
