## 摘要
长期对话智能体需要记住并查询跨会话、多类型且具有复杂关联的信息。现有的智能体记忆系统依赖于异构向量和图数据库，这导致记忆信息碎片化并产生高昂的跨数据库I/O延迟。在检索方面，常见的RAG风格方法往往会引入噪声，遗漏相关线索，且缺乏token预算控制，从而降低大语言模型的准确性和效率。我们提出Mandol，一种聚合式记忆系统，它将碎片化的记忆表示和存储整合到一个统一的记忆原生架构中。其核心组件包括：(1)一个层次化记忆模型，将记忆组织成表示原始记忆信息的基础层和将基础记忆聚合成可追溯抽象记忆的高级抽象层，两者统一表示为结构化语义图；(2)一种结合了SemanticMap和SemanticGraph的聚合式语义数据结构，它原生融合了键值、向量和图结构，并提供统一的混合检索算子以消除跨数据库I/O；(3)一种定量查询机制，具备查询自适应路由、定量去噪与冲突解决以及token受限的上下文生成，且检索过程中无需大语言模型参与。在两个广泛使用的长期对话基准LoCoMo和LongMemEval上的实验表明，Mandol在代表性智能体记忆系统中取得了最佳的整体准确率。在性能对比方面，Mandol在10 QPS并发负载下还获得了5.4倍检索加速和4.8倍插入加速，同时在消费级硬件上保持低延迟。

## Abstract
Long-term conversational agents need to remember and query cross-session, multi-typed information with complex correlations. Existing agent memory systems rely on heterogeneous vector and graph databases, which fragment memory information and cause high cross-database I/O latency. For retrieval, common RAG-style methods tend to introduce noise, miss correlated clues, and lack token budget control, degrading LLM accuracy and efficiency.   We propose Mandol, an agglomerative memory system that consolidates fragmented memory representations and storage into a unified memory-native architecture. Its core components include: (1) a hierarchical memory model that organizes memory into a basic layer representing raw memory information and a high-level abstract layer that agglomerates basic memories into traceable abstract memories, both uniformly represented as structured semantic graphs; (2) an agglomerative semantic data structure combining SemanticMap and SemanticGraph, which natively fuses key-value, vector, and graph structures and provides unified hybrid retrieval operators to eliminate cross-database I/O; and (3) a quantitative query mechanism with query-adaptive routing, quantitative denoising and conflict resolution, and token-constrained context generation, all without involving LLMs during retrieval. Experiments on two widely used long-term conversation benchmarks, LoCoMo and LongMemEval, show that Mandol achieves the best overall accuracy among representative agent memory systems. For performance comparison, Mandol also obtains a 5.4x retrieval speedup and a 4.8x insertion speedup under 10 QPS concurrent load, while still maintaining low latency on consumer-grade hardware.

### **2. 方法论：核心思想与关键技术细节**

Mandol的核心思想是将碎片化的记忆表示和存储整合到一个统一的架构中，其关键技术组件如下：

*   **层次化记忆模型**
    *   **核心思想**：将记忆统一表示为**结构化语义图**，分为基础层和高级抽象层，仅组织方式和抽象层级不同。
    *   **基础记忆层**：直接组织原始交互信息。它将信息分解为 **记忆单元**（封装对话文本、意图、元数据，并用统一语义向量编码）、**记忆空间**（按会话、任务等进行逻辑隔离）以及**显式/隐式关系**（如时序、引用等结构边，和按需计算的语义相似度边）。
    *   **高级抽象记忆层**：利用LLM从基础记忆中提取并聚合为可追溯的抽象记忆。包含：
        *   **情景记忆**：事件链。
        *   **语义记忆**：实体关系图。
        *   **情感记忆**：用户偏好变化轨迹。
        *   **关键特性**：所有抽象节点存储指向原始基础記憶單元的 `source_unit_uids`，实现 **可追溯性**，保证证据确凿。同时采用**会话级上下文增强**，解决原始记忆碎片化和上下文丢失问题。

*   **聚合式语义数据结构**
    *   **核心思想**：设计一个融合键值、向量和图结构的单一数据结构，以消除跨库I/O。它物理上实现了层次化记忆模型，由 `SemanticMap` 和 `SemanticGraph` 两部分组成。
    *   **SemanticMap**：以记忆单元ID为键，融合键值存储和向量索引（稠密/稀疏）及倒排索引，支持对异构数据的语义和关键词查询。
    *   **SemanticGraph**：使用轻量级邻接表存储显式结构边，并通过 `SemanticMap` 索引按需获取隐式语义关系。
    *   **统一性**：两者共享同一套记忆单元ID，使得查询、过滤、向量搜索和图遍历等混合操作可以原子化地在单一地址空间内完成，无需跨库。

*   **定量查询机制**
    *   **核心思想**：将检索建模为在有限Token预算下的高质量上下文构建问题，完全在检索阶段（不调用LLM）解决噪声、冲突和冗余问题。
    *   **流程**：
        1.  **查询自适应路由**：用一个轻量意图分类器，根据查询的语义特征（如时间表达式、实体提及）动态选择相关的记忆源（如基础、情景、语义记忆），并分配各源的候选预算，直接从源头过滤噪声。
        2.  **定量去噪与冲突解决**：
            *   **源内去噪**：基于交叉编码器的相关性得分，应用一种基于**中位数绝对偏差（MAD）** 的动态阈值 \( \tau_{mad} = \text{median}(S_{ce}) - \kappa \cdot \text{MAD}(S_{ce}) \) 来鲁棒地过滤低分噪声候选。该阈值不假设分数的正态分布。
            *   **跨源冲突解决**：对于来自不同源且描述同一实体但内容矛盾或冗余的记忆，计算一个**仲裁分数** \( S_{arb}(c_i) \)。该分数综合考虑了**语义相关性**、**时间新鲜度**（指数衰减）和**来源置信度**（原始记忆权重最高，抽象记忆次之），最终贪婪地保留得分最高的记忆。
        3.  **Token受限的高质量上下文生成**：使用基于**联合优化相关性与多样性（MMR）** 的贪心迭代选择算法。目标是最大化 \( \text{MMR}(c_i|\mathbf{S}) = \lambda \cdot \text{Rel}(c_i) - (1 - \lambda) \cdot \text{Red}(c_i, \mathbf{S}) \)，其中 `Rel` 是相关性，`Red` 是基于实体等重叠度量的冗余度。该算法在Token预算内优先选择高相关性且带来新信息的记忆，构造紧凑全面的上下文。
