## 摘要
当多模态 AI 智能体被要求忘记某一事实时，当前的记忆系统通常会删除文本条目并报告成功。我们发现，该事实仍可从保留的用户图像中恢复，包括标记为完全不同事实的图像，原因是视觉-语言模型（VLM）在推理时利用了隐含的视觉线索。我们引入了信息出处图（IPG），这是一种根据删除可供性对记忆表示进行分类的分类法。IPG 揭示了删除在多个渠道中失败。我们的基准测试 MemLeak 在删除级联中对此进行衡量：直接探测具有删除能力的系统恢复率低于 1%，但保留的相关文本使得恢复率达到 18.3%，保留的图像使得恢复率达到 12.0%（盲基线 0.0%，假阳性率 0.3%）——其中 47% 的图像泄露无法通过文本恢复。内容感知的语义删除将图像残差降至 2.0%。该残差在多个 VLM、一个生产级记忆系统以及真实的 Unsplash 授权照片中均有出现。双标注者人工验证（kappa = 0.88）证实了评判的可靠性。

## Abstract
When a multimodal AI agent is asked to forget a fact, current memory systems usually delete the text entry and report success. We find that the fact can remain recoverable from retained user images, including images tagged to entirely different facts, because VLMs use implicit visual cues at inference time. We introduce the Information Provenance Graph (IPG), a taxonomy that classifies memory representations by deletion affordance. The IPG reveals that deletion fails through multiple channels. Our benchmark, MemLeak, measures this across a deletion cascade: direct probing of deletion-capable systems yields <1%, but retained correlated text enables 18.3% recovery, and retained images enable 12.0% recovery (0.0% blind baseline, 0.3% FPR) -- with 47% of image leaks not text-recoverable. Content-aware semantic deletion reduces the image residual to 2.0%. The residual appears across multiple VLMs, a production memory system, and real Unsplash-licensed photographs. Dual-annotator human validation (kappa = 0.88) confirms judge reliability.

*   **研究动机与背景**：当前多模态 AI 智能体（如个人助理）的记忆系统，在响应用户的“遗忘”请求时，通常仅删除文本记录并报告成功。这种机制借鉴了计算机的内存泄漏（Memory Leak）概念，但在信息层面存在严重缺陷。一个用户事实可能扩散到多个记忆层（文本、文本嵌入、图像、图像嵌入、图边、隐式视觉特征等），而系统仅能定位并删除文本部分。
*   **核心问题**：论文旨在揭示并量化**多模态智能体记忆中的信息泄露问题**。核心论点是，当一个事实被要求遗忘后，它仍然可以通过保留在用户记忆中的、看似无关的图像被恢复，因为视觉-语言模型（VLMs）会利用这些图像中分布式的隐式视觉线索进行跨模态推理。

### 2. 论文提出的方法论
*   **顶层思路**：构建一个系统性的诊断框架，通过分类法分析信息传-播路径，并通过标准化的基准测试来量化不同遗忘渠道的泄露程度。
*   **关键技术细节与流程**：
    *   **信息溯源图 (IPG)**：一个描述性分类法，将记忆表示为节点，并映射其信息流。关键创新是为每个节点定义**“删除可供性”**：
        *   `addr`（可寻址）：系统可直接根据事实 ID 定位并删除。
        *   `link`（链接的）：通过溯源链接，系统可以级联删除该节点（如标记在特定事实下的图像）。
        *   `unreach`（持久的）：系统无法通过存储层操作删除，因为它们从未被显式存储（如 VLM 从图像中隐式推断出的特征）。
    *   **MemLeak 基准测试**：一个统一、可复现的测试框架，旨在剥离并测量不同记忆层的泄露。其核心是一个 **“删除级联”评估流程**：
        1. 注入事实和图像。
        2. 执行不同粒度的删除策略（如仅删文本、删文本+标记图像、内容感知删除）。
        3. 使用不同的**探针（Probes）** 测试事实的可恢复性。核心探针是使用保留在其他事实下的用户图像，询问 VLM 是否能重建已删除的事实。
        4. 使用由三个主流大模型（GPT-4.1, Claude Sonnet 4, Gemini 2.5 Flash）组成的**集成评判器（LLM Judge Ensemble）**，以多数投票制判断 VLM 的回答是否泄露了被遗忘的事实。

### 3. 实验设计

*   **基准对比方法**：评估了 5 种不同遗忘能力的系统架构：

    *   **Naive**：仅删除文本条目。
    *   **Mem0 (生产级)**：语义搜索并删除相关文本记忆。
    *   **Letta (模拟)**：智能体自主决定删除内容。
    *   **Oracle (溯源级联)**：文本、图像、图边等所有可寻址和链接的节点均被删除。

### 6. 论文的主要结论与发现
*   **多通道泄露**：事实级遗忘通过多个渠道同时失败。仅删除文本后，直接探测泄露率低于 1%，但保留的**相关文本**可使泄露率达 18.3%，保留的**图像**使泄露率达 12.0%。
*   **视觉通道的独特性与隐蔽性**：47% 的图像泄露案例无法通过纯文本审核发现，说明视觉通道是一个独立且更难审计的泄露源。
*   **当前方案不足**：
    *   **溯源级联删除**：可消除大部分“链接节点”泄露（将 48.7% 的直接图像恢复降至 12.0%），但无法解决跨事实视觉关联带来的残差。
    *   **内容感知语义删除**：通过 VLM 审计并移除与遗忘事实相关的保留图像，可将图像残差进一步降至 **2.0%**，但仍无法完全消除。
*   **现象普遍性**：该泄露现象在多个 VLM、生产级记忆系统（Mem0）和真实照片中均稳定复现。

### 7. 优点
*   **问题定义新颖且深刻**：将软件工程中的“内存泄漏”概念映射到 AI 记忆层的信息安全与隐私上，精准定义了“事实级遗忘”与“信息泄露”的标准。
*   **方法论系统且严谨**：提出的 **IPG 分类法**是分析复杂记忆系统遗忘问题的有力心智模型。整套 **MemLeak 基准测试**通过结构化的探针和多模型集成评判，将模糊的泄露问题转化为可量化的科学指标，实验和验证体系设计周密（如双盲、负对照、人工验证）。
*   **理论与实践价值高**：研究结果对系统构建者（需构建多模态审计）、学术研究者（指出新的研究方向）和政策制定者（验证合规不能仅依靠删除日志）均有直接的指导意义。

### 8. 不足与局限
*   **用户数据的生态效度**：基准测试主要基于合成画像和 AI 生成的图像。虽然用 Unsplash 真实照片进行了验证，但这与真实用户的、包含高度相关个人生活照的照片集在跨事实视觉关联的强度上可能仍存在差距。
*   **内容感知删除的效用代价**：为将图像泄露降至 2.0%，内容感知删除需要移除 21% 的保留图像，这可能带来显著的用户体验和记忆效用损失，论文仅标记了此问题而未量化其对下游记忆召回任务的影响。
*   **生产级系统性能的混杂因素**：端到端的 Mem0 系统泄露率（16.3%）结合了视觉泄露和文本检索残留，无法完全解耦，从而无法精确诊断其失效模式。

*   **测量工具的局限**：集成评判器在处理争议性案例（如 1/3 票）时的准确率有限，与人类的共识度较低（37.5%），可能遗漏或误判某些微妙泄露。
