## 摘要
智能体记忆系统越来越多地与RAG和全上下文基线进行比较，但报告的提升常常将记忆方法的改变与语言模型、嵌入模型或检索管线的改变混为一谈，导致实际测量目标不明确。我们提出MemDelta，这是一个受控评估协议，在LongMemEval-S（500个问题、50多个会话、三个模型家族）上每次只改变一个组件。得出四项发现：(1) 逐字RAG与全上下文GPT-4o-mini性能相当（47.2%对49.8%，p=0.34），但不同模型上排名反转：Gemini从全上下文中获得+14个百分点，而Sonnet从RAG中获得+31个百分点，部分原因是它拒绝了63%的全上下文查询；(2) 在相同管线中仅更换嵌入模型，在n=500时准确率改变+6.2个百分点（p=0.004），Mem0比MiniLM-RAG高出11个百分点，但比云RAG低1.2个百分点，因此一个变量就颠倒了结论；(3) 智能体自记忆（42%）表现不如基本检索（47%）；(4) 在6类问题中的2类上（n=88），Mem0以50倍成本匹配了云RAG（72.7%对73.9%，p=1.0），表明增益是狭窄的而非普遍的。我们建议记忆评估在比较中固定嵌入模型、按模型家族分层，并在将增益归因于架构之前报告写入路径成本。

## Abstract
Agent memory systems are increasingly evaluated against RAG and full-context baselines, but reported gains often mix changes in the memory method with changes in the language model, embedding model, or retrieval pipeline, making it unclear what is actually being measured. We present MemDelta, a controlled evaluation protocol that varies one component at a time on LongMemEval-S (500 questions, 50+ sessions, three model families). Four findings emerge: (1) verbatim RAG matches full-context GPT-4o-mini (47.2% vs. 49.8%, p = 0.34), but the ranking reverses across models: Gemini gains +14pp from full context, while Sonnet gains +31pp from RAG, partly because it refuses 63% of full-context queries; (2) swapping only the embedding model in an identical pipeline shifts accuracy by +6.2pp at n = 500 (p = 0.004), and Mem0 beats MiniLM-RAG by +11pp but loses to cloud-RAG by 1.2pp, so one variable flips the conclusion; (3) agent self-memory (42%) underperforms basic retrieval (47%); (4) on 2 of 6 question types (n = 88), Mem0 matches cloud RAG (72.7% vs. 73.9%, p = 1.0) at 50x the cost, suggesting narrow rather than general gains. We recommend memory evaluations fix embedding models across comparisons, stratify by model family, and report write-path cost before attributing gains to architecture.

### 6. 论文的主要结论与发现
- **检索质量**：从无记忆（2.2%）到逐字 RAG（47.2%）提升巨大（+45pp），RAG 与全上下文 GPT‑4o‑mini 性能相当（49.8%），差异不显著（p=0.34）。
- **模型行为是强混杂**：同一对 S1 vs S4 在不同模型上方向完全相反（Gemini 全上下文胜 +14pp，Sonnet RAG 胜 +31pp），Sonnet 在 115K 全上下文中拒绝回答 63% 的查询，而使用 RAG 却可正确回答。
- **嵌入模型的强大影响**：仅更换嵌入模型（MiniLM→云嵌入），在完全相同的 RAG 管线下准确率由 47.2% 升至 53.4%（+6.2pp，p=0.004），足以颠覆 Mem0 相较基线的“胜负”。
- **代理自记忆不如简单检索**：S2 自记忆（42%）低于 S4（47.2%），且在 4K token 预算下多会话问题仅得 3.3%，表明压缩式写入是高风险赌注。
- **Mem0 获益狭窄且成本高昂**：在可比嵌入下，Mem0（72.7%）与云 RAG（73.9%）无显著差异（p=1.0，2 类问题），但写入路径成本高出 50 倍以上，且仅在单会话用户类型上持平。
- **建议**：评估必须报告与“逐字 RAG+具名嵌入”的基线对比，用随机检索分离检索相关性，测试多个模型族，披露嵌入模型并做敏感度分析，并公布写入路径成本。

### 7. 优点
- **严格受控**：“一次只变一个因素”的配对设计，有效排除了多变量混杂，让归因变得清晰。
- **揭示隐匿因素**：通过嵌入交换实验、跨模型族比较，展示嵌入选择和模型长上下文行为如何产生与架构更改相当的增益。
- **成本与准确率并重**：不仅比较答案正确率，还审计写入路径的开销，体现工程实用性。
- **实用建议明确**：提出 6 条可操作评估准则，为社区规范化记忆评估提供直接参考。
- **分析细致**：按问题类型分层剖析，揭示了策略优势的非均匀性（如 RAG 擅长单会话，全上下文擅长知识更新）。

### 8. 不足与局限
- **Mem0 覆盖不全**：Mem0 仅在 2 类问题（88 个实例）上完成对比，未能覆盖时序推理和知识更新这类可能最受益于提取式记忆的题型，结论不宜外推。

- **数据泛化风险**：全部实验基于 LongMemEval‑S 的合成对话，逐字重合度高，可能有利于逐字检索，对真实用户场景的泛化性未经验证。
