## 摘要
基于大型语言模型（LLM）的智能体可以通过多轮与环境交互来解决复杂的程序性任务，但这种能力通常依赖于大模型、长上下文和重复的推理调用。这使得增强记忆的先进智能体难以部署在资源受限的设备上。我们提出了DuoMem，这是一个双空间蒸馏框架，它将程序性问题解决能力从大型教师模型迁移到紧凑的学生模型上。DuoMem在两个互补空间中进行蒸馏：（1）上下文空间蒸馏，用教师生成的高质量程序性记忆替换学生生成的记忆，并添加到学生输入的前端；（2）参数空间蒸馏，在成功的教师轨迹上微调轻量级LoRA适配器。在具有挑战性的具身决策基准ALFWorld上评估，DuoMem将40亿参数模型的任务成功率从4.3%提升到77.9%，缩小了与720亿参数教师模型（87.1%）之间的大部分差距，同时增加了不到1000万个可训练参数和仅几兆字节的预计算教师记忆。此外，DuoMem增强的40亿模型完成任务的挂钟时间比720亿教师模型快3倍以上，使其能够用于实时边缘部署，而这对于教师模型来说是具有挑战性的。对8个从20亿到720亿参数规模的模型进行的广泛消融实验揭示了两种蒸馏轴都提供了互补的贡献。

## Abstract
Large Language Model (LLM)-based agents can solve complex procedural tasks by interacting with environments over multiple turns, but this ability typically depends on large models, long contexts, and repeated inference calls. This makes advanced memory-augmented agents difficult to deploy on resource-constrained devices. We introduce DuoMem, a dual-space distillation framework that transfers procedural problem-solving ability from a large teacher model to compact student models. DuoMem distils in two complementary spaces: (1)context-space distillation, which replaces student-generated memories with higher-quality teacher-generated procedural memories prepended to the student's input, and (2)parameter-space distillation, which fine-tunes lightweight LoRA adapters on successful teacher trajectories. Evaluated on ALFWorld, a challenging embodied decision-making benchmark, DuoMem boosts a 4B-parameter model from 4.3% to 77.9% task success rate, closing most of the gap to a 72B teacher model (87.1%), while adding fewer than 10M trainable parameters and only a few megabytes of pre-computed teacher memories. ### 2. 论文提出的方法论

DuoMem 的核心思想是在两个互补的“空间”中进行知识蒸馏，共同增强学生模型。

*   **核心流程**：
    1.  **离线阶段**：使用一个强大的“教师模型”（72B参数）在训练任务上进行推理，产生两种关键数据：
        *   **高质量的程序性记忆脚本**：将成功解决任务的轨迹总结为简洁的文本指南。
        *   **完整的成功交互轨迹**：记录了从观察到最终成功的每一步动作序列。
    2.  **在线推理阶段**：针对一个新任务，检索最相关的教师记忆脚本，并将其与任务描述一同输入给经过参数蒸馏的学生模型。

*   **关键技术细节**：
    *   **1. 上下文空间蒸馏 (Context-Space Distillation, CD)**：
        *   **机制**：在推理时，不再让学生模型自己从历史轨迹中总结低质量的记忆，而是**直接将教师模型离线生成的、质量更高的程序性记忆脚本添加到学生模型的输入上下文最前端**。
        *   **实现**：所有教师记忆通过语义嵌入（如 `text-embedding-3-small`）向量化并存储。面对新任务时，根据任务描述的语义相似度，检索出前 *k* 个最相关的记忆脚本。
        *   **特点**：这是一种**无训练成本**的蒸馏方式，教师记忆只需生成一次，存储开销极小（数兆字节），可作为即插即用的知识库。
    *   **2. 参数空间蒸馏 (Parameter-Space Distillation, LoRA)**：
        *   **机制**：在教师模型成功的交互轨迹上，通过行为克隆的方式对学生的决策策略进行微调。
        *   **实现**：在学生模型上插入轻量级的 **低秩适应 (LoRA)** 模块，并冻结原始模型参数。训练目标是最大化在给定历史上下文时生成教师动作序列的概率。
        *   **训练策略**：研究发现，使用“完整交互历史”作为输入来预测每一步动作是最优的，因为它能让模型学习到完整的程序性动态，而不仅仅是局部模式。训练只使用**成功的轨迹**，以避免学习到失败行为。

*   **组合公式**：最终的 DuoMem 智能体动作生成可以概括为：
    `a_t = A_{θ_S + Δθ}(x_{CD}, o_{≤t}, a_{<t})`
    其中，`A` 代表智能体，`θ_S` 为学生原始参数，`Δθ` 为 LoRA 模块参数，`x_{CD}` 为嵌入了教师记忆的上下文。

### 3. 实验设计

*   **数据集/场景**：实验在 **ALFWorld** 基准上进行。这是一个基于文本的具身智能环境，要求智能体通过多步规划和交互完成家务任务（如拾取、加热、清洁物品等），并包含140个任务的验证集和134个任务的测试集。

*   **基准对比和方法**：
    *   **无记忆 (No Memory)**：基础学生模型直接执行任务。
    *   **程序记忆 (MemP)**：学生模型使用自己生成的记忆脚本和一条参考轨迹。
    *   **仅上下文蒸馏 (MemP + CD)**：在 MemP 基础上，将学生记忆替换为教师生成的高质量记忆。
    *   **仅参数蒸馏 (MemP + LoRA)**：在 MemP 基础上，对学生模型进行 LoRA 微调。
    *   **DuoMem (MemP + CD + LoRA)**：同时应用上下文和参数空间蒸馏。
    *   **教师模型**：一个720亿参数的强大模型，作为性能上限参考。实验中使用了 Qwen2.5-72B-Instruct 作为教师。

*   **评测指标**：
    *   **任务成功率**：在允许的最大步数内完成任务的百分比。
    *   **平均交互步数**：完成任务所需的平均交互轮次（越少越好）。
    *   **挂钟时间**：完成任务的平均耗时（秒）。

### 4. 资源与算力

*   **硬件与推理成本**：论文明确指出，所有实验的挂钟时间是在 2x NVIDIA A100 (80GB) GPU 上运行得到的。

### 6. 论文的主要结论与发现

*   DuoMem 能够**显著且变革性地提升小模型**的程序性任务解决能力。4B 模型的任务成功率从 4.3% 跃升至 77.9%，缩小了与 72B 教师模型（87.1%）之间 89% 的差距。
*   上下文空间蒸馏（CD）和参数空间蒸馏（LoRA）**贡献互补，且组合效果远超任一单独方法**，表现出协同效应。
*   DuoMem 在提升性能的同时，**带来了巨大的效率优势**。相比 72B 教师，4B 学生模型完成任务的挂钟时间快了 3 倍以上，为实时边缘部署提供了可能。
*   CD 作为一种**无训练、低成本**的知识迁移方法，对小模型和“非思考”模式的模型增益尤为明显，是一种极佳的部署权衡方案。
*   LoRA 微调在学生与教师能力差距巨大时最有效。对于已有一定基础能力的中等模型（如 8B），收益可能减少。

### 7. 优点

*   **方法创新**：首次提出在上下文空间和参数空间进行双重蒸馏来迁移程序记忆，两种蒸馏机制分而治之，又相互促进，构思精巧。
*   **实用性强**：框架高度关注实际部署问题，充分考虑了内存、存储、延迟等端侧设备的核心约束，不仅追求准确率，更实现了高性价比的解决方案。
*   **实验扎实**：在多个模型和配置上进行详尽的消融实验，结论可靠，为不同场景下的应用提供了清晰的指导原则。
*   **即插即用性**：上下文空间蒸馏无需任何训练即可为小模型带来稳定提升，极具吸引力。

### 8. 不足与局限

*   **环境单一性**：评估仅在 ALFWorld 这一模拟具身环境中进行。尽管作者声称其方法具有任务不可知性，但其在网页导航、代码生成等更复杂、更多样化的程序性基准上的泛化能力尚未得到验证。
*   **单一教师依赖**：所有实验均使用单一的 Qwen2.5-72B 作为教师。教师的偏见或不安全行为可能通过蒸馏管道传播给学生模型。
*   **训练任务依赖性**：LoRA 微调需要访问一个与测试任务相似分布的训练任务集，这限制了其在完全开放领域或不断变化的环境中的直接应用。
*   **实时部署评估**：虽然讨论了延迟，但推理时间是在服务器级 A100 GPU 上测量的，未在真正的移动 NPU 等边缘硬件上进行端到端的部署分析。
