## 摘要
序列演化的LLM记忆使智能体能够复用过往经验，但现有系统通常不加甄别地部署每个本地生成的记忆更新，而不检查其是否改善未来行为。因此，有助于当前任务的更新可能会覆盖有用知识、引入过于具体的规则，或使最终记忆偏向近期示例。我们提出Janus，一个插件式记忆控制器，用于决定是接受候选记忆更新还是保留原有记忆。为高效做出这一决策，Janus使用记忆动量触发器识别记忆更新轨迹中的可疑偏差，并在由覆盖性、边界性及新鲜任务构成的紧凑混合评估集上比较新旧记忆，而非重演完整历史。Janus与具体方法无关，可封装现有更新器而不改变其更新规则。在六个数据集、两种骨干LLM和两种记忆更新器上，Janus相比对应基础更新器将平均准确率提升了+2.7至+4.6个百分点。

## Abstract
Sequentially evolving LLM memory enables agents to reuse past experience, but existing systems usually deploy each locally generated memory update without checking whether it improves future behavior. As a result, updates that help the current task may overwrite useful knowledge, introduce over-specific rules, or bias the final memory toward recent examples. We propose Janus, a plug-in memory controller that decides whether to accept a candidate memory update or retain the previous memory. To make this decision efficient, Janus uses a Memory Momentum Trigger to identify suspicious deviations in the memory-update trajectory, and compares old and new memories on a compact hybrid evaluation set of coverage, boundary, and fresh tasks instead of replaying the full history. Janus is method-agnostic and wraps existing updaters without changing their update rules. Across six datasets, two backbone LLMs, and two memory updaters, Janus improves average accuracy by +2.7 to +4.6 points over the corresponding base updaters.

### 2. 方法论

论文提出了**Janus**，一个“即插即用”的记忆控制器，它封装在现有的记忆更新器外部，通过两个核心设计来高效地实现部署决策。

*   **核心思想**：将记忆更新视为一个需要全局评估的部署决策，而非一个必然有益的本地操作。Janus在“何时比较”和“用什么比较”两个关键问题上进行了创新。

*   **关键技术细节**：
    *   **记忆动量触发器 (Memory Momentum Trigger, MMT)**：用于解决“**何时比较**”的问题。它并非在每个任务后或按固定间隔比较新旧记忆，而是追踪记忆更新的向量轨迹。
        *   **流程**：
            1.  计算当前候选更新与上一记忆状态之间的向量差 \(\mathbf{z}_t = \phi(\mathbf{cM}_t) - \phi(\mathbf{M}_{t-1})\)，其中\(\phi\)是文本编码器。
            2.  维护更新方向的指数移动平均（动量）\(\mathbf{m}_t = \beta\mathbf{m}_{t-1} + (1-\beta)\mathbf{z}_t\)。
            3.  当候选更新的方向 \(\mathbf{z}_t\) 与历史动量 \(\mathbf{m}_{t-1}\) 的余弦相似度低于阈值 \( \tau \) 时（即 `cos(z_t, m_{t-1}) < τ`），触发一次新旧记忆的显式比较。这表明记忆更新轨迹发生了显著偏离，可能带来有益的新知识或有害的扭曲。
        *   若未触发，则直接接受候选更新，节省计算开销。

    *   **混合触发时评估集 (Hybrid Trigger-Time Evaluation Set)**：用于解决“**用什么比较**”的问题。当触发比较时，Janus在一个紧凑的混合任务集上评估新旧记忆，而非在整个历史任务集上重演。该评估集 \(\mathbf{E}_t\) 由三部分组成：
        1.  **覆盖性集合 (\(\mathbf{S}^{\text{cov}}_t\))**：通过聚类所有已见任务的嵌入并选择最接近质心的任务来构建，旨在代表已见任务的整体分布。
        2.  **边界性集合 (\(\mathbf{S}^{\text{bdry}}_t\))**：专门存储那些对记忆变化敏感的任务（“翻转任务”），即在这些任务上，使用新旧记忆会导致模型正确性发生改变。这直接聚焦于决策的关键点。
        3.  **新鲜集合 (\(\mathbf{F}_t\))**：包含自上次触发以来新遇到的任务，避免评估退化为对固定回放池的闭环过拟合。
    *   **最终决策**：在混合评估集 \(\mathbf{E}_t\) 上，分别评估使用 \(\mathbf{M}_{t-1}\) 和 \(\mathbf{cM}_t\) 的LLM表现，并部署性能更高的那个记忆状态。
