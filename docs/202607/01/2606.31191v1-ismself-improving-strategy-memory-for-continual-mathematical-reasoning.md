## 摘要
我们提出智能模式记忆（ISM），一种自我进化的记忆增强系统，能够在具有严格片段重置的持续学习场景下，为冻结的大语言模型提升数学推理能力。ISM维护一个紧凑的、自我精炼的策略模式库，这些模式从成功和失败的片段中学习，并借助符号化工具检查中间步骤和验证答案。在不更新模型参数的情况下，ISM在MATH-Hard和OlympiadBench上优于被动检索和反思基线，使用的模式数量分别比最强被动基线少64%和86%。这些结果表明，小规模、主动维护且经过验证的策略记忆能够在严格的片段隔离条件下支持可靠的持续数学推理。

## Abstract
We propose Intelligent Schema Memory (ISM), a self-evolving memory-augmented system that improves mathematical reasoning for a frozen LLM under continual learning with hard episodic resets. ISM maintains a compact, self-refined bank of strategy schemas learned from both successful and failed episodes, with symbolic tools that check intermediate steps and certify answers.Without updating model parameters, ISM outperforms passive, retrieval, and reflection baselines on MATH-Hard and OlympiadBench, using 64% and 86% fewer schemas respectively than the strongest passive baseline. These results show that small, actively maintained, and verified strategy memories can support reliable continual mathematical reasoning under strict episodic isolation.

### 1. 论文的核心问题与整体含义

*   **研究动机**：大语言模型在解决孤立的数学问题时表现出色，但在一个持续到来的、领域不断变化的问题流中，模型每次推理都是独立的（硬性重置）。上一轮获得的“经验”无法直接传递给下一轮，导致模型无法从过往的成功或失败中持续积累能力。这引出了一个根本性问题：如何让一个参数完全冻结的大语言模型，在不进行任何训练的情况下，自我进化其数学推理能力？
*   **整体含义**：论文旨在探索一种外部化、可控的持续学习路径。其核心思想是，模型的持续改进不应发生在模型内部参数上，而应完全寄托于一个外部维护的、可验证的、自我演进的知识库（即策略记忆）。数学推理因其清晰的逻辑规则和可验证性，成为研究这一动态过程的理想测试平台。

### 2. 论文提出的方法论

ISM 系统的核心是一个在严格验证下自我演进的策略模式记忆库。它由五个主要组件构成，形成一个闭环。

*   **核心思想**：构建一个紧凑的“策略模式（Schema）”记忆库。每个模式分为“内容”和“特征钩子”两部分，实现“做什么”与“何时用”的解耦。记忆库通过七种自我改进机制，从成功和失败的经验中对称地学习，并利用符号化验证来确保所有更新的正确性。
*   **关键技术细节与流程**：
    1.  **双表示模式设计**：
        *   **内容部分**：包含策略描述、解题模板和启发式方法，用于注入到LLM的提示中，内容保持相对稳定。
        *   **特征钩子部分**：包含操作符类型、结构模式、启发式签名等检索信号，以及持续在线更新的嵌入中心点和成功率。
    2.  **两阶段检索**：
        *   **第一阶段（硬过滤）**：根据问题特征提取的操作符类型，直接滤除不相关的模式族。
        *   **第二阶段（软评分）**：对剩余候选模式进行加权打分，权重综合了结构、启发式、数量、嵌入相似度以及先验成功率。根据分数高低，分为直接使用、搭配通用模式使用、或仅使用通用模式三个桶。
    3.  **记忆控制器与自我改进循环**：
        *   这是系统的核心，包含七种在独立周期上运行的机制：
            *   **质量与生命周期管理**：**自我审计** 评估模式健康状态；**自我修正** 重写表现差的模式内容；**自我合并** 合并高相似度的重复模式；**自我修剪** 移除无用或持续失败的模式；**自我晋升/降级** 根据成功率动态调整检索分数。
            *   **对称的成功/失败学习**：**自我强化** 从成功案例中提炼正面启发式并追加到模式内容中；**自我反模式** 从失败案例中提取应避免的错误并记录在模式内容中。
    4.  **符号化验证门控**：所有记忆更新（包括钩子更新、合并、强化、合成等）在生效前，都必须通过符号化计算工具的验证，从根本上防止记忆库被错误泛化“污染”。
    5.  **有条件模式演化**：当系统在特定操作符上连续失败达到阈值时，演化门会触发一个LLM来综合一个新的“通用家族模式”，以填补能力空白。

### 3. 实验设计

*   **数据集与场景**：
    *   **基准测试**：在**MATH-Hard**（来自MATH数据集最难的5级问题）和**OlympiadBench**（奥林匹克竞赛级别问题）两个基准上进行评估。

*   **对比方法**：论文将ISM与五个基线方法进行了对比，覆盖了主要的方法类别：

    3.  **RAG-over-Examples**：存储所有过往问题作为样本，通过嵌入相似度检索最相关的样本。
